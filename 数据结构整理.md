## 序列与区间维护

### bitset

`bitset` 用一个整数块维护多个二进制位，适合布尔集合、状态压缩、位运算优化和交并补统计。目录中的手写 `FastBitset` 支持单点 `set/reset/flip/test`、整体置位/清零/翻转、`count/any/none`、寻找第一个 0/1、按位 `and/or/xor`、左右移、切片等操作。

核心价值是把若干个布尔状态压进机器字，用位运算把逐元素操作优化成按块操作。若机器字长为 $w$，很多批量操作可从 $O(n)$ 降到 $O(n / w)$。

### 树状数组

树状数组维护前缀可合并信息，常见操作是单点修改、前缀查询，也可以通过差分维护区间加与区间和。

- 线性建树：按父节点 `x + lowbit(x)` 从小到大更新，或由前缀和直接得到每个树状数组节点的管辖区间值，时间 $O(n)$。
- 树状数组倍增：在非负前缀和上找最小位置，例如最小 $x$ 使前缀和大于某值，单次 $O(\log n)$。
- 区间加、区间和：把原数组转化到差分数组，用两个树状数组维护，常数小、代码短。
- 不可差分信息：例如区间最大值，单点修改和区间查询可做，但通常需要 $O(\log^2 n)$。

### 线段树

线段树是目录中展开最多的一类结构，核心性质是任一区间可拆成 $O(\log n)$ 个线段树节点。

- zkw 线段树：满二叉树形态、堆式存储、自底向上维护。代码短、常数小，适合单点修改和区间查询；区间修改更依赖标记永久化，懒标记下传不方便。
- 猫树：静态区间询问结构，预处理前缀/后缀信息，支持 $O(1)$ 查询。功能上比 ST 表强，可维护不可交信息，但空间和常数更大。
- 李超线段树：动态插入线段或直线，询问某个 $x$ 处的最大/最小 $y$。单点询问 $O(\log n)$，插入线段通常 $O(\log^2 n)$，插入整条直线 $O(\log n)$。
- 动态开点线段树：只在访问到区间时创建节点，适合值域很大但操作稀疏的场景。单次访问节点数 $O(\log n)$，最坏空间 $O(m\log n)$。
- 标记永久化：把区间修改标记留在完整覆盖节点上，查询时沿途累计，避免下传。
- 可持久化线段树：通过复制被修改路径上的节点维护历史版本，未修改子树共享节点。
- 主席树：可持久化权值线段树，常用于区间第 $k$ 小、区间 rank 等。
- 吉司机线段树：历史最值、势能线段树一类技巧，原文仅简略提到。
- 线段树合并：合并两棵表示相同值域的动态开点线段树，常用于树上子树信息合并，总复杂度通常按节点数摊还。
- 线段树分裂：合并的逆过程，例如按前 $k$ 小把权值线段树拆开，单次 $O(\log n)$。
- 线段树优化建图：把“点到区间”或“区间到点”的连边拆成 $O(\log n)$ 条边，常与最短路结合。
- 线段树分治：把操作存在的时间区间挂到时间线线段树节点上，进入节点执行、退出节点撤销，经典搭配是可撤销并查集。
- 线段树上二分：在单调区间信息上找第一个满足条件的位置，通常 $O(\log n)$。

### Sqrt Tree

Sqrt Tree 是序列上的根号树，思想是分块维护块内前缀、后缀，以及块间答案。若询问落在同一块内则递归处理。

它可做到 $O(n\log\log n)$ 预处理、$O(1)$ 区间询问，空间 $O(n\log\log n)$。本质上和猫树类似，都是利用首尾散块前后缀加中间整块预处理；功能性一般为 Sqrt Tree 强于猫树强于 ST 表，但常数也相应更大。

原文还提到它可支持一定修改：单点修改 $O(\sqrt n)$；在不同查询复杂度约束下，区间赋值可做到 $O(\sqrt n\log\log n)$ 或 $O(\sqrt n)$。

### 分块

分块是根号思想的总称，原目录列出四类：

- 序列分块：按位置分块，散块暴力、整块维护摘要。
- 值域分块：按值域分块，适合频次、rank、前驱后继等问题。
- 操作分块：按操作序列分块，块内暴力或重构。
- 根号重构：周期性重建全局结构，用较低实现复杂度换取可接受复杂度。

### 莫队

莫队是基于分块排序的离线询问技巧，核心是调整询问顺序，让左右指针总移动次数较小。复杂度分析通常只统计指针移动次数，具体题目还要乘上 `add/del` 维护信息的代价。

- 普通莫队：处理静态区间询问。块长 $t = n / \sqrt m$ 时，指针移动复杂度 $O(n\sqrt m)$；当 $n,m$ 同阶时通常取 $\sqrt n$。
- 回滚莫队：用于 `add` 容易、`del` 困难，或反向的“只减不删”版本。将删除转换为块内回滚，复杂度形态为 $O(n^2 / B + mB)$。
- 带修莫队：加入时间维度，把询问表示为 $(l,r,t)$。当序列规模、询问数、修改数同阶，块长取 $n^{2/3}$，复杂度 $O(n^{5/3})$。
- 树上莫队：借助欧拉序把树上路径转为序列区间，维护点出现次数奇偶；若询问子树信息则通常直接用 dfs 序区间。
- 二次离线莫队：当指针移动贡献可前缀差分离线计算时，用二次离线降低 `add/del` 代价。
- 在线莫队：更像分块预处理技巧，预处理块到块的状态，询问时从整块答案向散块扩展。

### 珂朵莉树

珂朵莉树，也称 ODT，本质是用 `map` 等结构维护值相同的连续区间段。核心操作是：

- `split(x)`：把包含 $x$ 的区间切开。
- `assign(l,r,v)`：将区间推平为同一个值。
- `perform(l,r)`：遍历区间段并执行具体操作。

在经常区间推平的场景下，每个区间段被遍历次数可摊还，复杂度约 $O(m\log n)$；随机数据下 CF896C 给出 $O(m\log\log n)$ 的表现。它强依赖数据随机或推平操作，不能当作稳定最坏复杂度结构使用。

## 有序集合和值域结构

### 01-Trie

01-Trie 是字符集为 $\{0,1\}$ 的 Trie，常用于维护整数二进制信息，例如最大异或。

- 普通 01-Trie：插入和查询按二进制位走，空间 $O(n\log V)$。
- 可持久化 01-Trie：维护历史版本，用两个版本做差可查询区间二进制信息；空间仍为 $O(n\log V)$。
- Patricia Trie：压缩不分叉的链，节点数降到 $O(n)$，时间复杂度不变。
- 压位 01-Trie：把每层儿子状态用机器字压缩，树高 $O(\log_w V)$，默认维护不可重集合；空间 $O(V / w)$，插入、删除、前驱、后继、最大最小均 $O(\log_w V)$。
- 动态开点压位 01-Trie：针对值域大但元素少的场景，避免直接开完整值域空间，但子节点映射会带来额外复杂度或常数。

### vEB

vEB 是对值域做根号递归的结构，功能与压位 Trie 接近：插入、删除、前驱、后继、最大值、最小值。

每个节点把值域分成 $O(\sqrt V)$ 块，并用一个 summary 结构维护哪些子块非空。树高 $O(\log\log V)$，空间 $O(V)$。插入、删除、前驱、后继为 $O(\log\log V)$，最大值和最小值可 $O(1)$ 获取。

### 堆

- 对顶堆：用一个小根堆维护前 $k$ 大，一个大根堆维护剩余元素。适合动态全局第 $k$ 大，复杂度取决于相邻询问的 $k$ 变化量：$O(n\log n+\sum \lvert k_i-k_{i-1}\rvert\log n)$。
- 可并堆：支持高效合并两个堆，典型实现有配对堆、左偏树等。配对堆常数与均摊表现较好，左偏树支持可持久化。GNU PBDS 提供 pairing heap、binomial heap 等实现。

### 平衡树

平衡树维护有序集合，支持插入、删除、rank、第 $k$ 小、前驱、后继等。

- Treap：二叉查找树加随机堆权，靠旋转维护平衡，期望树高 $O(\log n)$；常数小，可持久化相对方便。
- Fhq-Treap：无旋 Treap，通过 split/merge 维护。适合区间作为子树的操作，如序列翻转、区间移动、拼接，也支持持久化；常数通常比有旋 Treap 大。
- Splay：通过伸展操作把访问节点旋到根，均摊 $O(\log n)$。适合维护序列区间操作，例如把区间夹成一棵子树后打懒标记。
- 替罪羊树：用平衡因子判断是否重构子树，重构由中序展开和二分建树完成。插入、rank、k-th 方便；删除通常惰性处理，前驱后继要注意被删除节点影响。
- 序列平衡树：把序列下标视作键值，中序遍历就是原序列。Fhq-Treap 与 Splay 特别适合区间翻转、区间移动、区间插入等操作。

平衡树比较：

| 类型 | 常数 | 区间翻转 | 持久化 |
| --- | --- | --- | --- |
| Treap | 小 | 不擅长 | 支持 |
| Fhq-Treap | 较大 | 支持 | 支持 |
| Splay | 最大 | 支持 | 不擅长 |
| 替罪羊树 | 小 | 不擅长 | 不擅长 |

### 线性基

线性基是异或意义下的极大线性无关组，常用于维护集合可异或出的所有值。它把原集合的 $2^n$ 种选择压缩到最多 $2^m$ 种，其中 $m$ 是值域二进制位数。

- 高斯消元构建：把整数看成 01 向量矩阵，从高位消元，使用 bitset 可做到 $O(m^2n / w)$。
- 贪心插入构建：在线插入元素，从高位到低位尝试消去，空间 $O(w)$，支持回退。
- 常见查询：能否表示某值、最大异或值、最小非零值、第 $k$ 小/大可表示值。
- 解的构造：若要还原由原集合哪些数异或得到，可用 `set` 或 `bitset` 维护每个基向量来源。

## 树上结构与技巧

### dfs 序

dfs 序记录树上节点首次访问时间。核心性质是：一个子树在 dfs 序上是一段连续区间。因此子树问题常可转化为序列区间问题，再交给树状数组、线段树等维护。

### 重链剖分

重链剖分按子树大小选重儿子，使任意树上路径经过不超过 $O(\log n)$ 条重链。若 dfs 时优先遍历重儿子，则每条重链上的节点 dfs 序连续，路径问题可拆成 $O(\log n)$ 个序列区间。

它可同时维护路径和子树信息：路径依赖重链拆分，子树依赖 dfs 序连续。

### 长链剖分

长链剖分按子树最大深度选重儿子。它常用于树上 $k$ 级祖先和与深度相关的 DP 优化。

- 树上 $k$ 级祖先：预处理倍增祖先，以及每条长链链顶上下若干节点，可做到单次 $O(1)$ 查询。
- 长链剖分优化 DP：适合状态与深度相关、可以复用长儿子数组空间的树形 DP。

### 猫猫虫剖分

这是一种调整节点编号顺序的剖分方式：先递归重儿子，再给轻儿子编号，再递归轻儿子。它保留类似重链剖分的维护能力，并额外让一个点的邻接点可拆成较少连续区间。

### 点分治与点分树

点分治用于树上路径问题。以重心为划分点，路径分为经过重心和不经过重心两类；经过重心的路径在重心处合并，不经过重心的路径递归到子树处理。由于每次重心划分后子树规模至多减半，递归层数 $O(\log n)$，总复杂度通常为 $O(n\log nT(n))$。

动态点分治，也称点分树，把点分治递归过程中的重心连成一棵树。每个点作为端点时，只需在点分树祖先处合并信息，适合强制在线的树上路径查询。

### 树上启发式合并与 dsu on tree

树上启发式合并是在 DFS 回溯时把儿子子树的信息合并到当前节点，合并两个数据结构时把小的暴力并入大的，复杂度一般为 $O(n\log nT(n))$。

dsu on tree 基于轻重链剖分思想，保留重儿子信息，清空轻儿子信息。每个节点被暴力遍历的次数与根到它路径上的轻边数有关，因此总遍历次数 $O(n\log n)$，复杂度 $O(n\log nT(n))$。

### 动态树与 LCT

动态树用于维护会改变形态的森林。常见动态树有 Link-Cut Tree、Euler Tour Tree、Top Tree；原文主要整理 LCT。

LCT 用 Splay 维护动态树上的链，支持路径信息维护以及 `link/cut` 修改树形态。核心操作包括：

- `access(x)`：打通从当前根到 $x$ 的偏好路径。
- `make_root(x)`：把 $x$ 变成所在树的根。
- `find_root(x)`：找所在树根。
- `split(x,y)`：把 $x,y$ 路径表示成一棵 Splay。
- `link(x,y)`：连边。
- `cut(x,y)`：删边。

LCT 维护的是森林，不维护一般无向图。离线情况下，有些 LCT 问题可用树链剖分或“线段树分治 + 可撤销并查集”替代。

### 树套树

树套树用于维护多维信息，典型形式是外层线段树每个节点维护一棵内层数据结构。外层线段树负责把区间拆成 $O(\log n)$ 个节点，真正的信息由内层结构维护。

- 线段树套线段树：常见为线段树套权值线段树，时空可达 $O(n\log^2 n)$。
- 线段树套平衡树：时间 $O(n\log^2 n)$，空间 $O(n\log n)$。
- 分块套线段树/平衡树：外层用分块，整块走内层结构，散块暴力。
- 单点修改区间 k-th：线段树套平衡树通常转化为区间 rank 后二分，$O(\log^3 n)$；线段树套权值线段树可多棵一起递归，做到 $O(\log^2 n)$。

## 集合维护

### 并查集

并查集维护若干不交集合，常用于连通性。路径压缩与按秩合并结合后，总复杂度 $O(n\alpha(n))$，可近似视为线性。

- 启发式合并：把小集合暴力并入大集合，并维护每个节点当前根，可做到任意时刻 $O(1)$ 判断连通性，但总复杂度 $O(n\log n)$ 且常数较大。
- 可撤销并查集：不用路径压缩，只用按秩合并，并把修改记录入栈，按逆序撤销。常与线段树分治搭配。
- 带权并查集：维护节点到父亲的边权，路径压缩时同步累计权值，适合差分约束、食物链等关系维护。
- 扩展域并查集：把每个点拆成多个身份节点，通过合并不同身份表示关系，也称种类并查集。
- 可持久化并查集：用可持久化数组维护 `fa` 和 `size`，支持历史版本；不使用路径压缩，复杂度与空间通常 $O(n\log n)$。
- 赋值并查集：维护“把集合中所有值 $x$ 修改成 $y$，查询初始值当前是什么”的合并映射问题。

## 离线分治技巧

### CDQ 分治

CDQ 分治常用于偏序问题。二维偏序可按一维排序，另一维在分治过程中用数据结构维护跨左右区间的贡献。

基本分为三类贡献：左区间内部、右区间内部、左区间到右区间。最后通过归并保持有序性。扩展到 $N$ 维偏序时可嵌套 CDQ，复杂度形如 $O(n\log^{N-1}nT(n))$，但高维时常数和对数次方会很重。

### 整体二分

整体二分用于多次可二分答案的离线询问。它不是对每个询问单独二分，而是在答案值域上整体递归，把询问按答案落在左半还是右半划分。

核心优化是多组询问共享一次 `check` 贡献。复杂度通常为 $O(n\log mT(n))$，其中 $T(n)$ 是维护或检查数据结构的代价。它常替代复杂在线结构，例如离线区间第 $k$ 大等问题。

### 线段树分治

线段树分治也可归入离线技巧。把某个操作存在的时间区间挂到时间线线段树上，递归时进入节点执行操作，退出节点撤销操作，叶子即某个时间点的答案。

它要求操作可撤销，典型搭配是可撤销并查集，用于动态连通性、边按时间出现消失等问题。

## 选型速查

| 需求 | 优先考虑 |
| --- | --- |
| 静态 RMQ，信息可交 | ST 表或猫树 |
| 静态区间询问，信息不可交，要求 $O(1)$ 查询 | 猫树、Sqrt Tree |
| 区间加区间和，追求短代码小常数 | 树状数组双差分 |
| 大值域稀疏点更新/查询 | 动态开点线段树 |
