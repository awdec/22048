<h1><center>第一节 文件、目录与打开文件</center></h1>

::: info 例 1
为什么 open 成功后，read 通常只需文件描述符而不再提供完整路径？
:::

::: details 解析
open 已完成路径解析和权限检查，并在进程文件描述符表中建立到系统打开文件表项的映射。read 可通过描述符直接定位打开文件状态。
:::

::: info 例 2
两个进程分别打开同一文件，是否一定共享同一个文件偏移量？
:::

::: details 解析
不一定。独立 open 通常建立不同打开文件表项，各自有偏移量；若通过进程复制或描述符复制共享同一打开文件表项，则可共享偏移量。
:::

::: info 例 3
删除文件名后，正在读取该文件的进程是否一定立即失败？
:::

::: details 解析
不一定。删除目录项只是减少链接计数；只要文件仍被打开，内核可继续保留 inode 和数据块，直到链接计数和打开引用都归零。
:::
