<h1><center>第一章综合训练</center></h1>

::: info 综合题
用户程序调用 read 读取磁盘文件，简述可能发生的状态和模式切换。
:::

::: details 解析
1. 用户程序执行封装 read 的库函数并发起系统调用。
2. 陷入指令使处理器由用户态进入内核态。
3. 内核检查参数、文件描述符和缓存；若数据不在缓存，启动 I/O。
4. 当前进程阻塞，调度器可能切换到另一进程。
5. 磁盘完成后产生外中断，处理器进入内核态执行中断处理。
6. 原进程被唤醒进入就绪态，之后被调度运行并返回用户态。
:::

## 答题技巧

- 先区分用户态/内核态，再区分进程状态。
- “外设完成”对应外中断，“当前指令出错”对应异常。
- 系统调用是接口，陷入指令是实现进入内核的机制。
