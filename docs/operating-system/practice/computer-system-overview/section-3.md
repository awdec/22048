<h1><center>第三节 系统调用与系统结构</center></h1>

::: info 例 1
执行普通库函数一定会切换到内核态吗？
:::

::: details 解析
不一定。纯计算型库函数可完全在用户态执行；只有库函数内部发起系统调用时才会进入内核态。
:::

::: info 例 2
比较宏内核和微内核的主要优缺点。
:::

::: details 解析
宏内核把主要服务置于内核态，调用路径短、性能高，但内核大、故障隔离较弱。微内核只保留基本机制，服务多在用户态，可靠且易扩展，但消息通信和模式切换开销较大。
:::

::: info 例 3
调度器选择“时间片轮转”属于机制还是策略？保存和恢复上下文呢？
:::

::: details 解析
选择时间片轮转属于策略，回答“选谁、何时切换”；保存和恢复上下文属于机制，提供“怎样切换”的能力。
:::

::: info 例 4
外核与微内核在资源管理上的主要区别是什么？
:::

::: details 解析
微内核把文件、设备等服务移到用户态，但仍由操作系统服务进程实施主要管理策略；外核进一步只在内核中安全地分配、回收和保护资源，把更多管理策略交给用户态库操作系统。
:::

::: info 例 5
计算机启动时，是操作系统内核先运行还是引导程序先运行？
:::

::: details 解析
固件先执行并装入引导程序，引导程序再把内核装入内存并移交控制权。内核完成初始化后才启动用户态服务。虚拟机中的客户操作系统也要经历相应的虚拟启动过程。
:::
