<h1><center>第一节 I/O 控制与驱动</center></h1>

::: info 例 1
比较中断驱动 I/O 与 DMA 中 CPU 被打断的频率。
:::

::: details 解析
中断驱动通常每传输一个字或较小单位就中断一次；DMA 由控制器成块传输，CPU 只在初始化和整块完成时参与，因此中断频率更低。
:::

::: info 例 2
DMA 传输期间 CPU 能否继续执行指令？
:::

::: details 解析
可以，但 DMA 与 CPU 会竞争内存总线。DMA 可采用周期窃取等方式取得总线，CPU 在部分存储周期可能等待。
:::

::: info 例 3
设备独立性软件和设备驱动程序哪一个更依赖具体硬件？
:::

::: details 解析
设备驱动程序更依赖具体控制器和设备。设备独立性软件向上提供统一接口并完成逻辑设备名映射、保护和缓冲等通用功能。
:::

::: info 例 4
非阻塞 I/O 与阻塞 I/O 的返回时机有何不同？
:::

::: details 解析
阻塞 I/O 在操作暂时不能完成时使调用进程等待；非阻塞 I/O 立即返回当前结果或“暂不可用”状态，调用者可继续执行并稍后重试。
:::

::: info 例 5
设备驱动程序接口为什么要由操作系统统一规定？
:::

::: details 解析
统一接口让上层使用一致的打开、关闭、读、写和控制操作调用不同设备驱动，同时把寄存器、控制时序和中断处理等硬件细节限制在驱动内部。
:::
