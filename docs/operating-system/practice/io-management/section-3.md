<h1><center>第三节 磁盘调度</center></h1>

::: info 例 1
磁头当前位于磁道 $50$，请求序列为 $55,58,39,18$。按 FCFS 计算总移动磁道数。
:::

::: details 解析
$$
|55-50|+|58-55|+|39-58|+|18-39|
=5+3+19+21=48.
$$
:::

::: info 例 2
对上题采用 SSTF，求服务顺序和总移动量。
:::

::: details 解析
从 $50$ 开始，最近请求依次为：
$$
55\rightarrow58\rightarrow39\rightarrow18.
$$
恰与本题 FCFS 相同，总移动量仍为 $48$。SSTF 并不保证每组请求都优于 FCFS，只是每一步局部选择最近请求。
:::

::: info 例 3
SCAN 与 LOOK 的关键区别是什么？
:::

::: details 解析
SCAN 沿当前方向移动到磁盘物理端点再反向；LOOK 只移动到该方向上最远的待服务请求就反向，避免无请求区间的移动。
:::

::: info 例 4
SSD 是否仍需按 SSTF 优化机械寻道距离？
:::

::: details 解析
通常不需要。SSD 没有机械磁头和旋转延迟，调度重点转向并行性、写合并、垃圾回收和磨损均衡。
:::
