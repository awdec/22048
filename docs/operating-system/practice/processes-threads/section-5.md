<h1><center>第二章综合训练</center></h1>

::: info 综合题
系统有三类资源，总量为 $(3,3,2)$。某时刻可用资源为 $(1,1,0)$，两个未完成进程的尚需资源分别为 $P_1:(1,0,0)$、$P_2:(0,2,1)$。判断当前状态是否安全。
:::

::: details 解析
当前可用 $(1,1,0)$ 能满足 $P_1$，令 $P_1$ 完成。假设它释放的已分配资源为 $(1,0,1)$，可用量变为 $(2,1,1)$，仍不能满足 $P_2$ 对第二类资源的需求 $2$。

仅凭题中信息还缺少各进程已分配量，不能断言安全。银行家算法必须同时知道 Available、Allocation 和 Need；这正是本题考查点。
:::

::: info 综合题 2
为什么信号量值为负时，其绝对值可表示等待进程数？
:::

::: details 解析
记录型信号量每次 wait 先减 $1$，资源不足时进程入等待队列；每多一个等待进程，值再减 $1$。因此在没有额外实现差异时，$|S|$ 对应队列中的等待进程数。
:::

## 答题技巧

- 状态转换题先找“缺 CPU”还是“缺事件”。
- 调度题先画时间轴，再算周转和等待时间。
- PV 题先定义每个信号量的物理含义和初值。
- 银行家题逐行验证 `Need ≤ Work`，完成后把 Allocation 加回 Work。
