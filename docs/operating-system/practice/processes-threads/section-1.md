<h1><center>第一节 进程、线程与状态</center></h1>

::: info 例 1
进程等待磁盘 I/O 完成时处于什么状态？I/O 完成后直接进入运行态吗？
:::

::: details 解析
等待期间处于阻塞态。I/O 完成后由阻塞态变为就绪态，之后还需等待调度，不能直接进入运行态。
:::

::: info 例 2
同一进程的两个线程共享哪些资源？哪些内容各自独立？
:::

::: details 解析
共享进程地址空间、代码、全局数据和打开文件；各自拥有程序计数器、寄存器现场、线程状态和栈。
:::

::: info 例 3
为什么说 PCB 是进程存在的唯一标志？
:::

::: details 解析
操作系统通过 PCB 识别、控制和调度进程。程序段和数据段本身不能完整描述动态执行状态；PCB 被创建表示进程建立，PCB 被撤销表示进程终止。
:::
