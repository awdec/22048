<h1><center>第四节 通信与死锁</center></h1>

::: info 例 1
系统处于不安全状态，是否说明已经死锁？
:::

::: details 解析
不说明。不安全状态表示无法找到保证所有进程完成的安全序列，之后可能死锁；死锁状态一定不安全，但不安全状态未必已经死锁。
:::

::: info 例 2
每类资源都只有一个实例时，资源分配图中存在环说明什么？
:::

::: details 解析
存在环是死锁的充要条件。若某类资源有多个实例，存在环只能说明可能死锁。
:::

::: info 例 3
规定所有进程必须按资源编号递增顺序申请资源，破坏了哪个死锁必要条件？
:::

::: details 解析
破坏循环等待条件。统一的全序使资源请求边不可能形成首尾相接的循环。
:::

::: info 例 4
共享内存与消息传递各有什么同步特点？
:::

::: details 解析
共享内存通信速度快，但通信进程必须借助锁、信号量等自行保证同步；消息传递由 send/receive 和内核队列提供较明确的同步边界，但涉及系统调用和数据复制。
:::

::: info 例 5
进程间通信中的信号适合传送大块数据吗？
:::

::: details 解析
不适合。信号主要用于通知异步事件及其类型，携带的信息量很小；大量数据应使用共享内存、消息传递或管道，并另行处理同步问题。
:::
