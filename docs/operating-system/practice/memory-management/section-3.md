<h1><center>第三节 虚拟内存与请求分页</center></h1>

::: info 例 1
访问某合法页时 TLB 未命中，但页表项存在位为 $1$，是否发生缺页？
:::

::: details 解析
不发生。TLB 未命中只表示快表中没有该映射，访问内存页表后仍能得到页框号；存在位为 $0$ 才会缺页。
:::

::: info 例 2
换出的页面修改位为 $0$，是否必须写回磁盘？
:::

::: details 解析
通常不必。页面装入后未被修改，磁盘已有相同副本，可直接覆盖页框；修改位为 $1$ 才需先写回。
:::

::: info 例 3
缺页处理结束后，为什么要重新执行引发缺页的指令？
:::

::: details 解析
该指令第一次执行时所需页面不在内存，访存操作尚未完成。页面调入并更新映射后，需要从原指令重新开始，才能得到正确结果。
:::

::: info 例 4
进程把文件映射到虚拟地址空间后，首次访问尚未装入内存的映射页会发生什么？
:::

::: details 解析
处理器产生缺页异常。内核检查访问合法性和映射关系，把文件中对应的数据块读入页框，更新页表后重新执行引发缺页的指令。
:::
