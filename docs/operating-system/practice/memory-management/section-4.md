<h1><center>第四节 页面置换与抖动</center></h1>

::: info 例 1
访问序列为 $1,2,3,1,4,2$，分配 $3$ 个初始为空的页框。用 FIFO 计算缺页次数。
:::

::: details 解析
- 访问 $1,2,3$：各缺页，页框为 $[1,2,3]$。
- 访问 $1$：命中，FIFO 顺序不变。
- 访问 $4$：缺页，淘汰最早进入的 $1$，页框为 $[4,2,3]$。
- 访问 $2$：命中。

共缺页 $4$ 次。
:::

::: info 例 2
对同一访问序列，为什么 OPT 的缺页次数不会多于其他算法？
:::

::: details 解析
OPT 每次淘汰未来最晚使用的页，使当前选择对后续访问最有利，因此给出理论上的最少缺页次数。但运行时无法预知完整未来序列。
:::

::: info 例 3
给进程增加页框一定会降低 FIFO 的缺页数吗？
:::

::: details 解析
不一定。FIFO 可能出现 Belady 异常，页框增加反而缺页更多。OPT 和 LRU 属于栈算法，不会出现该异常。
:::

::: info 例 4
系统发现处理器利用率很低、磁盘换页频繁，此时再增加多道程序度是否合适？
:::

::: details 解析
通常不合适。这是抖动迹象，增加进程会进一步减少每个进程的驻留集。应减少多道程序度或按工作集、缺页率增加关键进程页框。
:::
