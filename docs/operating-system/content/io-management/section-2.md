<h1><center>第二节 缓冲、设备分配与 SPOOLing</center></h1>

## 1. 缓冲的作用

- 缓和 CPU 与设备速度不匹配。
- 减少对 CPU 的中断频率。
- 提高 CPU 与设备之间的并行性。
- 解决数据粒度不匹配。

## 2. 缓冲方式

### 2.1 单缓冲

设备把数据送入缓冲区，进程再从缓冲区处理。设备输入与用户处理可以部分重叠。

### 2.2 双缓冲

两个缓冲区交替使用，一个由设备填充，另一个供进程处理，可进一步提高并行度。

### 2.3 循环缓冲与缓冲池

循环缓冲由多个缓冲区首尾相接；缓冲池统一管理空缓冲、输入满缓冲和输出满缓冲，适合多个进程与设备共享。

## 3. 高速缓存与缓冲的区别

- 缓冲主要解决速度和数据粒度不匹配，数据最终要交给接收方。
- 高速缓存保存数据副本，利用局部性减少对慢速层次的访问。

## 4. 设备分配

设备分配需要记录设备状态、占用者和等待队列。独占设备一次只能分配给一个进程，共享设备可以并发接受多个请求；系统还可用设备独立性软件完成逻辑设备名到物理设备的映射。

设备使用完毕后应回收占用标志并唤醒等待进程。分配策略要兼顾安全性、公平性和设备利用率。

## 5. SPOOLing

假脱机技术利用磁盘模拟独占设备，把独占设备改造为可由多个进程共享的虚拟设备。

以打印为例：用户进程把输出写入磁盘输出井，打印守护进程再按顺序送往打印机。

组成包括：

- 输入井与输出井。
- 输入、输出缓冲区。
- 输入、输出进程。

::: warning 易错点
SPOOLing 的核心外存区域是输入井和输出井；内存中的输入、输出缓冲区只负责暂存传输数据。
:::
