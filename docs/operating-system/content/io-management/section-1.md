<h1><center>第一节 I/O 系统、控制方式与设备驱动</center></h1>

## 1. I/O 设备分类

- 按使用特性：人机交互设备、存储设备、网络通信设备。
- 按信息交换单位：字符设备、块设备。
- 按共享属性：独占设备、共享设备、虚拟设备。

块设备通常支持按块随机访问，字符设备以字符流方式顺序传输。

## 2. I/O 控制器

设备控制器位于 CPU 与设备之间，通常包含：

- 数据寄存器。
- 状态寄存器。
- 控制寄存器。
- 与 CPU、设备连接的接口逻辑。

CPU 可通过独立编址或内存映射 I/O 访问控制器寄存器。

## 3. I/O 控制方式

### 3.1 程序直接控制

CPU 不断查询设备状态，忙等待，利用率低。

### 3.2 中断驱动

CPU 发出命令后执行其他程序，设备完成一个数据单位后中断 CPU。减少轮询，但每次传输仍需 CPU 参与。

### 3.3 DMA

DMA 控制器在内存与设备之间成块传输数据。CPU 负责初始化 DMA 参数，块传输完成后再被中断。

## 4. I/O 软件层次

自上而下通常为：

1. 用户层 I/O 软件。
2. 设备独立性软件。
3. 设备驱动程序。
4. 中断处理程序。
5. 硬件设备。

设备独立性软件完成逻辑设备名到物理设备的映射、统一接口、缓冲和错误处理。

## 5. 输入/输出应用程序接口

- 字符设备接口按字节流读写，通常不支持任意位置访问。
- 块设备接口以固定大小的数据块为单位，可支持随机访问。
- 网络设备接口通过套接字等抽象收发网络数据。
- 阻塞 I/O 在操作完成前使调用进程等待；非阻塞 I/O 在暂时不能完成时立即返回，调用者可稍后重试。

## 6. 设备驱动程序及其接口

驱动程序与具体硬件相关，把上层抽象 I/O 请求转换为控制器命令，设置寄存器并启动设备，通常运行在内核态。

操作系统通过统一的驱动程序接口调用设备初始化、打开、关闭、读、写和控制等操作。驱动程序还负责处理中断、报告设备状态和错误，并向上层隐藏设备寄存器及控制时序。

::: warning 易错点
DMA 并非完全不需要 CPU：CPU 要初始化传输，DMA 竞争系统总线，完成后还要通过中断通知 CPU。
:::
