<h1><center>第五章总结：输入/输出管理</center></h1>

## I/O 控制方式

| 方式 | CPU 参与程度 | 数据传输单位 |
| --- | --- | --- |
| 程序查询 | 很高，忙等待 | 字/字符 |
| 中断驱动 | 每次传输完成参与 | 字/字符 |
| DMA | 初始化与块完成时参与 | 数据块 |

## 缓冲与虚拟设备

- 单缓冲可让输入与处理部分重叠。
- 双缓冲可让设备和进程更充分并行。
- SPOOLing 用磁盘井把独占设备虚拟为共享设备。

## 磁盘调度

| 算法 | 优点 | 风险/特点 |
| --- | --- | --- |
| FCFS | 公平简单 | 平均距离可能较大 |
| SSTF | 平均距离较短 | 可能饥饿 |
| SCAN | 性能与公平兼顾 | 到端点后反向 |
| C-SCAN | 等待时间较均匀 | 单向服务 |
| LOOK/C-LOOK | 减少无效移动 | 只到最远请求位置 |

::: tip 一句话总结
I/O 管理通过控制器、驱动和中断屏蔽硬件差异，通过缓冲提高并行度，再通过设备与磁盘调度协调多个请求。
:::
