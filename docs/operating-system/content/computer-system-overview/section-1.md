<h1><center>第一节 操作系统的概念、功能与特征</center></h1>

## 1. 操作系统的概念

操作系统是控制和管理计算机硬件与软件资源、合理组织计算机工作流程，并为用户和应用程序提供接口的系统软件。

它位于硬件与应用程序之间：

$$
\text{用户/应用程序}\rightarrow\text{操作系统}\rightarrow\text{硬件}
$$

## 2. 操作系统的功能

- **处理机管理**：进程控制、调度、同步、通信与死锁处理。
- **存储器管理**：内存分配、地址转换、保护与虚拟内存。
- **文件管理**：文件、目录、磁盘空间和访问权限。
- **设备管理**：设备分配、驱动、中断、缓冲与磁盘调度。
- **接口服务**：命令接口、程序接口和图形用户接口。

## 3. 四个基本特征

### 3.1 并发

多个事件在同一时间间隔内发生。单处理器上多个进程宏观同时推进、微观交替执行，属于并发。

### 3.2 共享

系统资源可供多个并发进程共同使用：

- 互斥共享：同一时刻只允许一个进程访问。
- 同时共享：允许一段时间内被多个进程交替或并行访问。

### 3.3 虚拟

通过某种技术把一个物理实体映射为若干逻辑实体，如虚拟处理器和虚拟内存。

### 3.4 异步

并发进程以不可预知的速度向前推进，但操作系统必须保证执行结果仍具有确定性。

::: tip 核心关系
并发与共享互为存在条件；没有并发就无需共享管理，没有资源共享也难以实现真正的并发协作。
:::

## 4. 操作系统的发展

| 类型 | 主要目标 | 典型特点 |
| --- | --- | --- |
| 手工操作 | 直接使用硬件 | 人机矛盾突出 |
| 批处理系统 | 提高吞吐量 | 作业成批、缺少交互 |
| 分时系统 | 改善交互性 | 时间片轮转、多个终端 |
| 实时系统 | 保证及时响应 | 截止时间和可靠性要求高 |

::: warning 易错点
并发是“同一时间间隔”，并行是“同一时刻”。单核处理器可以并发，但不能在同一时刻真正执行两个用户进程。
:::
