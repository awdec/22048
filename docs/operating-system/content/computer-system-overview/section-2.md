<h1><center>第二节 运行环境与中断、异常</center></h1>

## 1. 处理器运行模式

处理器通常设置两种运行模式：

- **用户态（目态）**：执行普通应用程序，只能使用非特权指令。
- **内核态（管态）**：执行操作系统内核，可使用全部指令。

涉及资源管理和系统安全的指令属于特权指令，如修改中断屏蔽字、设置时钟、执行 I/O 指令和切换页表基址。

## 2. 中断与异常

中断和异常使处理器暂停当前程序，转去执行相应处理程序，完成后再决定是否返回。

| 类型 | 来源 | 同步性 | 示例 |
| --- | --- | --- | --- |
| 外中断 | CPU 外部 | 异步 | 时钟、I/O 完成、硬件故障 |
| 异常/内中断 | 当前指令内部 | 同步 | 除零、缺页、非法指令、系统调用 |

## 3. 异常的常见分类

- **故障（Fault）**：通常可修复，处理后重新执行当前指令，如缺页故障。
- **陷阱（Trap）**：有意触发，处理后执行下一条指令，如系统调用。
- **终止（Abort）**：严重错误，通常无法恢复当前程序。

## 4. 中断处理过程

1. 关中断，防止保存现场时被再次打断。
2. 保存程序计数器、程序状态字等硬件现场。
3. 根据中断向量找到处理程序入口。
4. 保存通用寄存器，执行中断服务程序。
5. 恢复现场并执行中断返回。

其中部分现场由硬件自动保存，其余由操作系统软件保存。

## 5. 用户态与内核态的切换

用户态进入内核态的原因包括：

- 执行系统调用。
- 发生异常。
- 响应外部中断。

内核态返回用户态通常通过中断返回指令完成。

::: warning 易错点
“访管指令/陷入指令”在用户态执行，用于主动请求系统服务；真正的特权操作由进入内核态后的系统调用处理程序完成。
:::
