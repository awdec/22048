<h1><center>第三节 进程同步与互斥</center></h1>

## 1. 临界资源与临界区

一次只允许一个进程使用的资源称为临界资源。访问临界资源的代码称为临界区，其结构通常为：

1. 进入区：检查并申请访问权。
2. 临界区：访问临界资源。
3. 退出区：释放访问权。
4. 剩余区：其他代码。

同步强调进程间的先后制约，互斥强调对临界资源的排他访问。

## 2. 临界区管理原则

- 空闲让进。
- 忙则等待。
- 有限等待。
- 让权等待。

“让权等待”是理想原则，忙等待算法可能不满足，但仍可能正确实现互斥。

## 3. 软件与硬件实现方法

### 3.1 软件方法

软件方法只使用共享变量实现互斥。以两个进程的 Peterson 算法为例，每个进程先声明自己想进入临界区，再把机会让给对方；若对方也想进入且当前轮到对方，自己忙等待。

Peterson 算法满足互斥、空闲让进和有限等待，但依赖对共享变量读写顺序的严格假设，且忙等待会消耗处理器时间。

### 3.2 硬件方法

- 中断屏蔽：在单处理器内核临界区可暂时禁止中断，但不适合用户程序，也不能单独解决多处理器互斥。
- Test-and-Set、Swap 等原子指令：原子地读改写共享变量，可构造自旋锁。

硬件方法简单且适用于多处理器，但竞争期间可能持续忙等待。

## 4. 锁

锁具有“加锁—访问临界区—解锁”的基本结构。互斥锁在锁不可用时可让线程阻塞；自旋锁则循环检查锁，适合等待时间很短且线程不宜睡眠的场景。

加锁和解锁必须成对出现；持锁后再申请其他资源时要特别注意死锁。

## 5. 信号量

整型信号量通过原子操作 wait 和 signal（也记作 P、V）访问。

```text
wait(S):
    S = S - 1
    if S < 0: 阻塞当前进程

signal(S):
    S = S + 1
    if S <= 0: 唤醒一个等待进程
```

- 互斥信号量通常初始化为 $1$。
- 资源信号量初始化为可用资源数量。
- 表示前驱关系的同步信号量通常初始化为 $0$。

## 6. 经典同步问题

### 6.1 生产者—消费者

有界缓冲区大小为 $n$：

```text
empty = n, full = 0, mutex = 1

producer: wait(empty) → wait(mutex) → 放入 → signal(mutex) → signal(full)
consumer: wait(full)  → wait(mutex) → 取出 → signal(mutex) → signal(empty)
```

资源信号量的 wait 必须放在互斥锁之外，否则可能占锁后等待资源而死锁。

### 6.2 读者—写者

允许多个读者同时读，但写者与其他读者或写者互斥。需要计数器记录读者数，并用互斥量保护计数器更新。

### 6.3 哲学家进餐

若每人都先拿左筷子再等右筷子，可能死锁。可限制最多 $n-1$ 人同时拿筷子，或规定拿取顺序。

## 7. 条件变量与管程

管程把共享数据、操作过程和初始化封装起来，任一时刻只允许一个进程在管程内活动。

条件变量用于等待某个条件成立：`wait` 使调用线程释放管程互斥权并进入等待队列；`signal` 唤醒一个等待线程。条件变量本身不保存资源数量，因此不能把它与计数信号量混为一谈。

::: warning 易错点
P/V 操作本身是原语；互斥信号量保护临界区，资源信号量描述剩余资源，二者的初值和操作顺序不可混用。
:::
