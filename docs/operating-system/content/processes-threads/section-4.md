<h1><center>第四节 进程通信与死锁</center></h1>

## 1. 进程通信

低级通信通过少量状态信息协调，如信号量；高级通信可传递大量数据。

- **共享内存**：通信进程映射同一内存区，速度快，但需自行同步。
- **消息传递**：通过 send/receive 交换消息，可直接或间接寻址。
- **管道**：内核中的字节流缓冲区，普通管道通常用于有亲缘关系的进程。
- **信号**：内核向进程通知异步事件；进程可使用默认动作、忽略或执行已注册的信号处理程序。信号主要传递事件类型，不适合传送大量数据。

## 2. 死锁的概念

多个进程因竞争资源而相互等待，若无外力干预将永远无法推进，称为死锁。

死锁与饥饿不同：死锁常涉及一组进程形成循环等待；饥饿是某进程长期得不到服务，其他进程仍可推进。

## 3. 死锁的四个必要条件

1. 互斥条件。
2. 不剥夺条件。
3. 请求并保持条件。
4. 循环等待条件。

四者同时成立才可能发生死锁；破坏任意一个即可预防死锁。

## 4. 死锁处理策略

### 4.1 预防死锁

通过制度破坏必要条件，如一次性申请全部资源、允许剥夺、按统一编号顺序申请资源。限制强，资源利用率可能较低。

### 4.2 避免死锁

在每次分配前检查系统是否仍处于安全状态。银行家算法需要最大需求、已分配和可用资源信息。

安全状态表示存在安全序列，使所有进程都能依次完成。安全状态一定无死锁；不安全状态不等于已经死锁，但可能进入死锁。

### 4.3 检测与解除

允许死锁发生，定期用资源分配图或检测算法发现死锁，再通过撤销进程、剥夺资源或回退解除。
