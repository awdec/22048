<h1><center>第二章总结：进程与线程</center></h1>

## 状态转换速查

| 转换 | 原因 |
| --- | --- |
| 就绪 → 运行 | 被调度 |
| 运行 → 就绪 | 时间片到或被抢占 |
| 运行 → 阻塞 | 主动等待事件或资源 |
| 阻塞 → 就绪 | 等待事件完成 |

## 调度算法速查

| 算法 | 是否抢占 | 主要特点 |
| --- | --- | --- |
| FCFS | 否 | 简单，短作业可能等待很久 |
| SJF | 否 | 平均等待时间较小，可能饥饿 |
| SRTF | 是 | 最短剩余时间优先 |
| HRRN | 否 | 兼顾等待时间与服务时间 |
| RR | 是 | 适合分时系统 |

## 同步与死锁

- 互斥量初值通常为 $1$。
- 同步信号量初值通常由前驱是否完成决定，常为 $0$。
- 安全状态一定无死锁，不安全状态只是存在死锁风险。
- 预防破坏必要条件，避免维持安全状态，检测允许发生后再处理。
- 软件方法、原子硬件指令和锁均可实现互斥；条件变量用于等待条件成立。
- 共享内存、消息传递、管道和信号是大纲要求的进程间通信方式。

::: tip 一句话总结
本章以“状态—调度—同步—死锁”为主线：处理机决定谁运行，信号量决定如何协作，死锁理论决定资源请求能否继续满足。
:::
