<h1><center>第一节 文件、目录与打开文件</center></h1>

## 1. 文件的概念

文件是以计算机硬盘为载体、由操作系统管理的一组相关信息的集合。文件属性通常包括名称、标识符、类型、位置、大小、保护信息和时间信息。

基本操作包括创建、删除、打开、关闭、读、写、定位和截断。

## 2. 文件控制块 FCB

目录项通常保存文件控制块或指向 FCB 的索引。FCB 包含文件属性和物理位置等元数据，是系统管理文件的数据结构。

UNIX 类文件系统把文件名与其他元数据分离：目录项保存“文件名 + inode 号”，inode 保存权限、大小和数据块地址。

## 3. 目录结构

- 单级目录：简单，但文件名易冲突。
- 两级目录：每个用户一个目录，可解决不同用户同名问题。
- 树形目录：支持分层组织和绝对、相对路径。
- 无环图目录：允许共享文件或目录，不允许环。
- 通用图目录：允许环，遍历和回收更复杂。

## 4. 路径

- 绝对路径从根目录开始。
- 相对路径从当前工作目录开始。

设置当前目录可以减少每次路径解析从根开始的开销。

## 5. 打开文件

执行 open 后，操作系统把文件元数据装入内存打开文件表并返回文件描述符。之后 read/write 使用描述符，避免每次重新查找目录。

系统通常维护：

- 系统级打开文件表：保存打开计数、位置和 inode 等共享信息。
- 进程级文件描述符表：保存本进程描述符到系统表项的映射。

::: warning 易错点
删除一个目录项不一定立即删除文件数据；存在硬链接或文件仍被进程打开时，系统仍可能保留 inode 和数据块。
:::
