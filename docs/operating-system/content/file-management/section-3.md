<h1><center>第三节 文件系统、空间管理与保护</center></h1>

## 1. 文件系统层次

典型文件系统自上而下包括：

1. 用户调用接口。
2. 逻辑文件系统：目录、权限与元数据。
3. 文件组织模块：逻辑块到物理块映射。
4. 基本文件系统：发出通用块读写请求。
5. I/O 控制层：驱动程序和中断处理。
6. 存储设备。

## 2. 文件系统布局

磁盘分区通常包含：

- 引导块。
- 超级块/卷控制块。
- 空闲空间管理信息。
- inode 或 FCB 区。
- 目录与文件数据区。

## 3. 空闲空间管理

### 3.1 空闲表法

记录连续空闲区的起始块号和长度，适合连续分配。

### 3.2 空闲链表法

把空闲块或空闲区链接起来，实现简单，但随机取得大量连续块较慢。

### 3.3 位示图

用一个二进制位表示一个磁盘块是否空闲。查找、分配和回收方便，位示图大小为：

$$
\frac{\text{磁盘块数}}{8}\text{ 字节}.
$$

### 3.4 成组链接法

把空闲块号分组保存，兼具空闲表和链表特点，适合大型文件系统。

## 4. 文件共享

- **硬链接**：多个目录项指向同一 inode；通常不能跨文件系统。
- **符号链接**：新建一个保存目标路径的特殊文件；可以跨文件系统，但目标删除后可能悬空。

## 5. 文件保护

- 口令或加密保护。
- 访问控制矩阵。
- 访问控制表 ACL：按对象记录主体权限。
- 能力表：按主体记录可访问对象及权限。

## 6. 虚拟文件系统

虚拟文件系统 VFS 在系统调用与具体文件系统之间提供统一接口。它用统一的文件、目录项、索引节点和超级块等抽象屏蔽不同文件系统的实现差异，使应用能够用相同的 `open`、`read`、`write` 等操作访问不同类型的文件系统。

## 7. 文件系统挂载

挂载把一个文件系统的根目录接入已有目录树的某个挂载点。挂载后，访问该目录就会进入被挂载文件系统；卸载前应保证相关文件不再被使用并处理需要写回的数据。

::: tip 识别方法
ACL 的视角是“这个文件允许谁做什么”，能力表的视角是“这个用户能对哪些对象做什么”。
:::
