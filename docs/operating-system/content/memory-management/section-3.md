<h1><center>第三节 虚拟内存与请求分页</center></h1>

## 1. 局部性原理

- **时间局部性**：最近访问的数据或指令很可能再次访问。
- **空间局部性**：访问某地址后，很可能访问其附近地址。

虚拟内存基于局部性，只把当前需要的部分页面装入内存。

## 2. 虚拟内存的特征

- 多次性：作业分多次装入内存。
- 对换性：页面可在内存与外存之间换入换出。
- 虚拟性：逻辑地址空间可大于物理内存。

## 3. 请求分页页表项

除页框号外，常包含：

- 有效位/存在位：页面是否在内存。
- 访问位：近期是否被访问。
- 修改位：换出时是否需要写回外存。
- 保护位：读、写、执行权限。

## 4. 缺页处理

1. 处理器访问页面，发现存在位为 $0$。
2. 产生缺页异常，转入内核。
3. 检查地址合法性。
4. 寻找空闲页框；没有则选择牺牲页。
5. 若牺牲页已修改，先写回外存。
6. 从外存读入所需页面。
7. 更新页表和 TLB。
8. 重新执行引发缺页的指令。

缺页是可恢复的故障，不是普通 I/O 外中断；页面调入完成又会通过 I/O 中断通知处理器。

## 5. 页框分配、回收与置换范围

- 固定分配：每个进程页框数固定。
- 可变分配：运行中可调整页框数。
- 局部置换：只替换本进程页面。
- 全局置换：可从其他进程占用的页框中选择。

固定分配通常搭配局部置换；可变分配可搭配全局或局部置换。

内存空闲页框低于阈值时，系统可在后台回收页框。未修改的文件页可直接回收；已修改页面通常要先写回外存。访问位和修改位可帮助选择回收对象。

## 6. 内存映射文件

内存映射把文件的一部分关联到进程虚拟地址空间，进程可像访问内存一样读写文件内容。首次访问尚未装入的映射页时发生缺页，内核把相应文件块调入页框；修改后的映射页可按映射方式写回文件。

多个进程映射同一文件区域可以实现共享；解除映射时，操作系统撤销地址范围与文件之间的关联。

::: tip 核心流程
请求分页题要区分 TLB 未命中、页表命中和缺页三种情况，它们的代价完全不同。
:::
