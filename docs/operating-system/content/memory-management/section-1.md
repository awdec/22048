<h1><center>第一节 内存管理基础与连续分配</center></h1>

## 1. 内存管理的任务

- 内存空间的分配与回收。
- 逻辑地址到物理地址的转换。
- 内存保护，防止越界访问。
- 内存扩充，用有限内存运行更大程序。
- 进程间共享。

## 2. 地址重定位

源程序经编译、链接和装入后进入内存。

- **绝对装入**：编译时产生绝对地址，只适合固定位置。
- **静态重定位**：装入时一次性修改地址，运行期间不能移动。
- **动态重定位**：运行时由硬件地址转换机构完成，可在内存中移动。

动态重定位常使用基址寄存器和界限寄存器：

$$
\text{物理地址}=\text{基址}+\text{逻辑地址}.
$$

若逻辑地址超过界限，则产生越界异常。

## 3. 链接方式

- 静态链接：装入前连接成完整程序。
- 装入时动态链接：装入模块时完成链接。
- 运行时动态链接：真正调用模块时才装入并链接，便于共享动态库。

## 4. 连续分配方式

### 4.1 单一连续分配

内存分为系统区和用户区，用户区一次只装入一个进程。实现简单，但利用率低。

### 4.2 固定分区分配

预先划分固定大小分区，会产生内部碎片。

### 4.3 动态分区分配

按进程实际需要建立分区，主要产生外部碎片。可通过紧凑技术合并零散空闲区，但需动态重定位支持。

## 5. 动态分区算法

| 算法 | 选择方式 | 特点 |
| --- | --- | --- |
| 首次适应 FF | 地址递增首次满足 | 综合性能较好 |
| 最佳适应 BF | 最小可满足分区 | 易留下许多小碎片 |
| 最坏适应 WF | 最大空闲分区 | 保留较大剩余块 |
| 邻近适应 NF | 从上次位置继续 | 低地址、高地址分布较均匀 |

::: warning 易错点
固定分区主要产生内部碎片，动态分区主要产生外部碎片；分页有内部碎片，分段可能产生外部碎片。
:::
