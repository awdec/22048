<h1><center>第四节 页面置换与抖动</center></h1>

## 1. 最佳置换 OPT

淘汰未来最长时间不再访问的页面。它能得到最少缺页次数，但需要预知未来，只能作为理论最优基准。

## 2. 先进先出 FIFO

淘汰最早进入内存的页面，实现简单，但可能出现 Belady 异常：分配更多页框反而产生更多缺页。

## 3. 最近最少使用 LRU

淘汰最长时间未被访问的页面，利用过去预测未来，性能较好，但精确实现需要时间戳或栈等硬件支持。

LRU 属于栈算法，不会出现 Belady 异常。

## 4. 时钟 CLOCK

使用访问位和循环指针：

1. 指针指向页面访问位为 $0$，则淘汰。
2. 访问位为 $1$，清零并向后扫描。

改进型 CLOCK 同时考虑访问位和修改位，优先淘汰“未访问且未修改”的页面，减少写回开销。

## 5. 工作集

某段时间窗口内进程实际访问的页面集合称为工作集。驻留集是分配给进程的物理页框集合。

若驻留集长期小于工作集，进程会频繁缺页。

## 6. 抖动

系统把大部分时间用于页面换入换出，处理器利用率反而下降，称为抖动。原因通常是同时运行的进程过多、每个进程页框不足。

缓解方法：

- 减少多道程序度。
- 按工作集分配页框。
- 使用缺页率控制动态调整驻留集。

## 7. 缺页率计算

若访问序列长度为 $N$、缺页次数为 $F$：

$$
\text{缺页率}=\frac{F}{N}.
$$

计算时要明确初始页框是否为空、第一次装入是否计缺页以及采用的置换算法。

::: warning 易错点
OPT 和 LRU 不出现 Belady 异常，FIFO 可能出现；CLOCK 是 LRU 的近似实现，不等同于精确 LRU。
:::
