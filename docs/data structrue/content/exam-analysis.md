<h1><center>408 数据结构考点分析</center></h1>

## 1. 408 中的数据结构分值

408 全称通常为“计算机学科专业基础综合”，满分为：

$$
150\text{ 分}
$$

近年 408 的科目分值结构通常为：

| 科目 | 分值 | 占比 |
| --- | ---: | ---: |
| 数据结构 | $45$ 分 | $30\%$ |
| 计算机组成原理 | $45$ 分 | $30\%$ |
| 操作系统 | $35$ 分 | $23.3\%$ |
| 计算机网络 | $25$ 分 | $16.7\%$ |
| 合计 | $150$ 分 | $100\%$ |

::: tip 说明
具体分值与题型以当年考试大纲和当年真题为准。数据结构长期处于 408 中的高分值科目之一，通常与计算机组成原理同为最高占比。
:::

## 2. 408 试卷题型结构

408 常见题型由两部分组成：

| 题型 | 常见分值 |
| --- | ---: |
| 单项选择题 | $80$ 分 |
| 综合应用题 | $70$ 分 |
| 合计 | $150$ 分 |

数据结构的 $45$ 分通常会同时出现在：

- 单项选择题
- 综合应用题

常见理解方式：

| 部分 | 数据结构常见考查 |
| --- | --- |
| 选择题 | 概念、性质、复杂度、算法过程、结构特点 |
| 综合应用题 | 算法设计、树/图/查找/排序综合、代码阅读与补全 |

::: warning 注意
数据结构内部每年各章节的具体题量不固定，不要机械认为某一章必考几分。复习时应按“高频 + 高难 + 可综合出题”的优先级安排。
:::

## 3. 数据结构内部重点层级

### 第一层：必须掌握

这些内容属于基础中的基础，选择题和综合题都可能出现。

| 模块 | 重点内容 | 常见难点 |
| --- | --- | --- |
| 绪论 | 逻辑结构、存储结构、复杂度 | 复杂度分析 |
| 线性表 | 顺序表、链表、栈、队列 | 插入删除、指针操作 |
| 树与二叉树 | 遍历、性质、存储、二叉排序树 | 遍历序列还原、算法设计 |
| 图 | 存储、遍历、最短路径、最小生成树 | 算法过程和复杂度 |
| 查找 | 折半查找、散列、BST、AVL、红黑树、B 树 | 平均查找长度、调整过程 |
| 排序 | 插入、交换、选择、归并、基数排序 | 过程模拟、稳定性、复杂度 |

### 第二层：高频难点

这些内容容易出综合题，需要能手写过程或算法。

| 模块 | 高频难点 |
| --- | --- |
| 链表 | 头插、尾插、逆置、删除指定结点、合并 |
| 栈 | 括号匹配、表达式求值、递归调用 |
| 队列 | 循环队列、队空队满、层次遍历 |
| 二叉树 | 前中后序遍历、层序遍历、线索二叉树 |
| AVL | LL、RR、LR、RL 调整 |
| 红黑树 | 性质判断、插入调整 |
| B 树 / B+ 树 | 插入、分裂、删除、合并 |
| 图 | DFS、BFS、Dijkstra、Floyd、Prim、Kruskal |
| 排序 | 快排、堆排、归并、稳定性判断 |

### 第三层：容易丢分

这些内容看起来不难，但非常容易因为细节失分。

| 易错点 | 说明 |
| --- | --- |
| 位序和下标混淆 | 408 常用位序从 $1$ 开始，数组下标常从 $0$ 开始 |
| 复杂度只看一层循环 | 要看总执行次数，不是只看代码缩进 |
| 链表断链 | 修改指针时要先保存后继 |
| 循环队列判满 | 要看题目采用牺牲单元、`size` 还是 `tag` |
| KMP 的 `next` 定义 | 不同教材定义可能不同 |
| 树的结点数公式 | 度、叶结点、高度等公式条件不同 |
| 图算法过程 | 边权、访问状态、候选集合容易写错 |
| 排序稳定性 | 不能只看时间复杂度 |

## 5. 重点难点说明

### 线性表

线性表是算法设计题的常见载体。

重点掌握：

- 顺序表插入、删除、查找。
- 单链表头插法、尾插法。
- 单链表逆置。
- 双指针思想。
- 栈和队列的应用。

难点通常不在概念，而在代码边界：

- 空表
- 只有一个结点
- 删除头结点
- 删除尾结点
- 指针是否断链

### 树

树是 408 数据结构中最重要的模块之一。

重点掌握：

- 二叉树性质。
- 遍历序列。
- 二叉树存储。
- 线索二叉树。
- 哈夫曼树。
- 树、森林与二叉树的转换。

难点在于：

- 能不能根据遍历序列还原树。
- 能不能正确使用二叉树性质公式。
- 能不能区分树、森林和二叉树的转换规则。

### 图

图题综合性强，算法过程多。

重点掌握：

- 邻接矩阵和邻接表。
- DFS 与 BFS。
- 最小生成树。
- 最短路径。
- 拓扑排序。
- 关键路径。

难点在于过程模拟：

- 当前访问了哪些顶点。
- 候选边或候选顶点如何更新。
- 距离数组如何变化。
- 前驱路径如何记录。

### 查找与排序

查找与排序通常既考选择题，也可能考综合应用。

重点掌握：

- 折半查找判定树。
- 平均查找长度 ASL。
- 散列表冲突处理。
- 二叉排序树和 AVL 调整。
- 红黑树性质和插入调整。
- B 树、B+ 树插入删除过程。
- 各类排序算法过程。
- 时间复杂度、空间复杂度、稳定性。

::: tip 重点
排序题不要只背复杂度，还要能手动模拟每一趟排序结果。
:::

## 6. 复习策略

### 第一轮

目标是把概念和基本操作过一遍。

重点：

- 理解每种结构的定义。
- 会画结构图。
- 会写基本操作。
- 会分析简单复杂度。

### 第二轮

目标是形成题型意识。

重点：

- 链表算法设计。
- 栈和队列应用。
- 树的遍历与构造。
- 图算法过程。
- 查找和排序综合。

### 第三轮

目标是面向真题查漏补缺。

重点：

- 总结错题。
- 背熟复杂度表。
- 熟悉高频算法模板。
- 训练综合应用题书写。

## 7. 数据结构拿分建议

数据结构要想拿高分，应做到：

1. 概念题少丢分。
2. 复杂度分析稳定。
3. 链表和树的代码能写清楚。
4. 图算法和排序过程能手算。
5. 综合题能写出规范步骤。

408 数据结构的目标不只是“看懂”，而是：

$$
\text{能判断}
\quad
\text{能模拟}
\quad
\text{能分析}
\quad
\text{能写算法}
$$

## 8. 考前速记

1. 数据结构约占 408 的 $45$ 分。
2. 数据结构是 408 中最高占比科目之一。
3. 线性表、树、图、查找、排序是复习主干。
4. 链表、树、图容易出综合应用题。
5. 复杂度、稳定性、ASL、指针操作是常见失分点。
6. 算法题要注意边界条件和时间复杂度。
