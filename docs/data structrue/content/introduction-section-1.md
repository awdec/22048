<h1><center>第一节 基本概念</center></h1>

数据结构研究的是数据元素之间的关系，以及这些关系在计算机中的表示和实现方式。

程序设计中常说：

$$
程序 = 数据结构 + 算法
$$

数据结构偏重“数据如何组织”，算法偏重“如何处理数据”。选择合适的数据结构，往往能直接影响算法的时间复杂度和空间复杂度。

## 1. 基本术语

### 数据

数据是信息的载体，是能够被计算机识别、存储和处理的符号集合。

例如整数、字符、图像、声音、学生信息、网页链接等都可以看作数据。

### 数据元素

数据元素是数据的基本单位，通常作为一个整体进行处理。

例如学生表中的一名学生、线性表中的一个结点、图中的一个顶点，都可以看作数据元素。

### 数据项

数据项是组成数据元素的、有独立含义的最小单位。

例如一个学生数据元素可以包含：

- 学号
- 姓名
- 年龄
- 成绩

其中每一项都是数据项。

### 数据对象

数据对象是具有相同性质的数据元素的集合，是数据的一个子集。

例如所有整数构成一个整数数据对象，所有学生记录构成一个学生数据对象。

### 数据结构

数据结构是相互之间存在一种或多种特定关系的数据元素的集合。

也就是说，数据结构不仅关心“有哪些数据”，还关心“这些数据之间有什么关系”。

## 2. 数据结构三要素

数据结构通常由三个方面组成：

- 逻辑结构
- 存储结构
- 数据的运算

三者之间的关系可以理解为：

逻辑结构描述数据元素之间的逻辑关系；存储结构描述这种逻辑关系在计算机内存中的表示；数据的运算描述对这些数据可以执行哪些操作。

## 3. 逻辑结构

逻辑结构是从逻辑关系上描述数据元素之间的关系，与数据在计算机中的具体存储方式无关。

常见逻辑结构有四类。

### 集合结构

集合结构中的数据元素除了“属于同一个集合”之外，没有其他明显关系。

特点：

- 元素之间关系最弱。
- 不强调先后次序。
- 不强调层次或连接关系。

例如一个班级中所有学生的集合。

### 线性结构

线性结构中的数据元素之间存在一对一关系。

除第一个元素外，每个元素有且仅有一个直接前驱；除最后一个元素外，每个元素有且仅有一个直接后继。

常见例子：

- 线性表
- 栈
- 队列
- 串

### 树形结构

树形结构中的数据元素之间存在一对多关系。

除根结点外，每个结点有且仅有一个前驱；一个结点可以有零个或多个后继。

常见例子：

- 树
- 二叉树
- 堆
- B 树

### 图状结构

图状结构中的数据元素之间存在多对多关系。

一个顶点可以和多个顶点相关联，结构最复杂。

常见例子：

- 无向图
- 有向图
- 网

## 4. 存储结构

存储结构也叫物理结构，是逻辑结构在计算机存储器中的表示。

逻辑结构面向问题，存储结构面向实现。同一种逻辑结构可以采用不同的存储结构实现。

例如线性表可以用顺序表实现，也可以用链表实现。

### 顺序存储

顺序存储用一组地址连续的存储单元依次存放数据元素。

特点：

- 逻辑上相邻的元素，物理位置也相邻。
- 可以随机访问。
- 插入和删除时可能需要移动大量元素。

典型例子：顺序表。

### 链式存储

链式存储用任意的存储单元存放数据元素，元素之间的逻辑关系通过指针表示。

特点：

- 不要求物理地址连续。
- 插入和删除较灵活。
- 不能直接随机访问，通常需要从头开始查找。
- 需要额外空间保存指针。

典型例子：单链表、双链表、循环链表。

### 索引存储

索引存储在存储数据元素的同时，建立额外的索引表。

索引表中的每一项通常记录关键字和对应元素的地址。

特点：

- 查找速度较快。
- 需要额外空间保存索引。
- 维护索引会增加插入、删除的代价。

典型例子：数据库索引、查找表索引。

### 散列存储

散列存储又称哈希存储，通过散列函数根据关键字直接计算元素的存储地址。

特点：

- 平均查找效率高。
- 可能产生冲突，需要设计冲突处理方法。
- 存储位置与关键字之间由散列函数决定。

典型例子：哈希表。

## 5. 数据的运算

数据的运算是定义在逻辑结构上的操作。

常见运算包括：

- 创建
- 销毁
- 插入
- 删除
- 查找
- 修改
- 遍历
- 排序

运算的定义只说明“做什么”，不一定说明“怎么做”。具体实现方式取决于采用的存储结构。

例如线性表的插入操作，在顺序表中通常需要移动元素；在链表中通常需要修改指针。

## 6. 抽象数据类型

抽象数据类型简称 ADT，是指一个数学模型以及定义在该模型上的一组操作。

它强调数据的逻辑特性和操作接口，不关心底层如何存储和实现。

ADT 通常包含三部分：

- 数据对象
- 数据关系
- 基本操作

例如线性表可以抽象为：

- 数据对象：具有相同类型的若干数据元素。
- 数据关系：元素之间存在一对一的线性关系。
- 基本操作：初始化、插入、删除、查找、求长度、判空等。

## 7. 逻辑结构与存储结构的关系

逻辑结构和存储结构不是一一对应关系。

| 逻辑结构 | 常见存储方式 |
| --- | --- |
| 线性表 | 顺序存储、链式存储 |
| 树 | 顺序存储、链式存储 |
| 图 | 邻接矩阵、邻接表、十字链表、邻接多重表 |
| 查找表 | 顺序表、链表、索引表、散列表 |

做题时要先判断题目讨论的是逻辑结构还是存储结构。

例如：

- “线性表”是逻辑结构。
- “顺序表”是线性表的顺序存储实现。
- “链表”是线性表的链式存储实现。
- “栈”和“队列”是受限的线性表。

## 8. 常见易混点

### 数据元素与数据项

数据元素是处理的基本单位，数据项是组成数据元素的最小单位。

学生记录是数据元素；学号、姓名、成绩是数据项。

### 数据对象与数据结构

数据对象强调“同类元素的集合”，数据结构强调“元素之间存在关系”。

只有数据元素之间存在特定关系时，才进一步讨论数据结构。

### 逻辑结构与物理结构

逻辑结构与存储位置无关，物理结构与内存表示有关。

例如线性表不一定连续存储；只有顺序表才要求连续存储。

### 运算定义与运算实现

运算定义是逻辑层面的接口，运算实现是存储层面的代码。

同一个操作在不同存储结构下，时间复杂度可能不同。

例如按位查找：

- 顺序表可以通过数组下标直接访问，时间复杂度为 $O(1)$。
- 链表通常需要从头结点开始逐个访问，时间复杂度为 $O(n)$。

## 9. 小结

学习数据结构时，可以按下面的顺序理解一个结构：

1. 它的数据元素是什么。
2. 元素之间是什么逻辑关系。
3. 可以执行哪些基本操作。
4. 适合用什么存储结构实现。
5. 各种操作的时间复杂度和空间复杂度是多少。

这条线索后面学习线性表、栈、队列、树、图、查找和排序时都会反复使用。
