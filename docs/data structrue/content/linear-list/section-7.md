<h1><center>线性表-数组和特殊矩阵</center></h1>

## 6. 数组

数组是由相同类型的数据元素构成的有限序列。

数组中的每个元素占用相同大小的存储空间，并且可以通过下标直接访问。

数组的特点：

- 元素类型相同。
- 存储空间连续。
- 支持随机访问。
- 一旦定义，维数和每一维长度通常固定。

数组可以看作线性表的推广。

一维数组是线性结构；二维数组可以看作“每个元素又是一维数组”的线性结构。

## 6.1 一维数组的地址计算

设一维数组 `A[0...n-1]` 的首地址为 `LOC(A[0])`，每个元素占用 `L` 个存储单元。

则第 `i` 个元素的地址为：

$$
LOC(A[i]) = LOC(A[0]) + i \times L
$$

若采用 1 开始的下标 `A[1...n]`，则：

$$
LOC(A[i]) = LOC(A[1]) + (i - 1) \times L
$$

408 做题时，要先判断题目下标是从 `0` 开始还是从 `1` 开始。

## 6.2 二维数组的存储

二维数组在逻辑上有行和列，但内存仍然是一维连续空间。

常见存储方式有两种：

- 行优先存储：先存第 1 行，再存第 2 行。
- 列优先存储：先存第 1 列，再存第 2 列。

C 语言中的二维数组采用行优先存储。

## 6.3 行优先存储地址计算

设二维数组 `A[m][n]` 有 `m` 行、`n` 列，下标从 `0` 开始，每个元素占用 `L` 个存储单元。

则元素 `A[i][j]` 前面有：

- `i` 整行元素，共 `i * n` 个。
- 当前行中 `j` 个元素。

因此：

$$
LOC(A[i][j]) = LOC(A[0][0]) + (i \times n + j) \times L
$$

若下标从 `1` 开始，即 `A[1...m][1...n]`：

$$
LOC(A[i][j]) = LOC(A[1][1]) + ((i - 1) \times n + (j - 1)) \times L
$$

### 行优先下标映射

实现逻辑：把二维下标映射为一维数组下标。

::: details 点击展开代码

```c
int RowMajorIndex(int i, int j, int cols) {
    return i * cols + j;
}

int RowMajorIndexFromOne(int i, int j, int cols) {
    return (i - 1) * cols + (j - 1);
}
```

:::

## 6.4 列优先存储地址计算

设二维数组 `A[m][n]` 有 `m` 行、`n` 列，下标从 `0` 开始。

列优先存储中，元素 `A[i][j]` 前面有：

- `j` 整列元素，共 `j * m` 个。
- 当前列中 `i` 个元素。

因此：

$$
LOC(A[i][j]) = LOC(A[0][0]) + (j \times m + i) \times L
$$

若下标从 `1` 开始：

$$
LOC(A[i][j]) = LOC(A[1][1]) + ((j - 1) \times m + (i - 1)) \times L
$$

### 列优先下标映射

::: details 点击展开代码

```c
int ColumnMajorIndex(int i, int j, int rows) {
    return j * rows + i;
}

int ColumnMajorIndexFromOne(int i, int j, int rows) {
    return (j - 1) * rows + (i - 1);
}
```

:::

## 6.5 多维数组地址计算

多维数组的地址计算本质上仍是下标映射。

以三维数组 `A[d1][d2][d3]` 为例，若下标从 `0` 开始并采用行优先存储，则：

$$
LOC(A[i][j][k]) = LOC(A[0][0][0]) + ((i \times d_2 + j) \times d_3 + k) \times L
$$

可以理解为：

- 第 1 维每加 `1`，跳过 `d2 * d3` 个元素。
- 第 2 维每加 `1`，跳过 `d3` 个元素。
- 第 3 维每加 `1`，跳过 `1` 个元素。

## 7. 特殊矩阵

矩阵可以用二维数组存储。

普通 `n * n` 矩阵需要存储 `n^2` 个元素。

但某些矩阵中存在大量相同元素或零元素，可以只存储必要元素，从而节省空间。这种方法称为压缩存储。

408 中常见特殊矩阵包括：

- 对称矩阵
- 三角矩阵
- 三对角矩阵
- 稀疏矩阵

## 7.1 对称矩阵

若矩阵满足：

$$
a_{ij} = a_{ji}
$$

则称为对称矩阵。

对称矩阵关于主对角线对称，只需要存储上三角或下三角部分。

对于 `n * n` 对称矩阵，需要存储的元素个数为：

$$
\frac{n(n + 1)}{2}
$$

### 下三角按行压缩

若只存储下三角部分，且按行优先存储：

```txt
a11
a21 a22
a31 a32 a33
...
an1 an2 ... ann
```

设下标从 `1` 开始，压缩数组 `B` 也按 `1` 开始编号。

若 `i >= j`，则：

$$
k = \frac{i(i - 1)}{2} + j
$$

若 `i < j`，由于对称性：

$$
a_{ij} = a_{ji}
$$

因此：

$$
k = \frac{j(j - 1)}{2} + i
$$

若 C 语言数组 `B` 从 `0` 开始，则实际下标为 `k - 1`。

### 对称矩阵下标映射

::: details 点击展开代码

```c
int SymmetricIndex(int i, int j) {
    int k;

    if (i >= j) {
        k = i * (i - 1) / 2 + j;
    } else {
        k = j * (j - 1) / 2 + i;
    }

    return k - 1;
}
```

:::

## 7.2 三角矩阵

三角矩阵分为上三角矩阵和下三角矩阵。

### 下三角矩阵

下三角矩阵中，主对角线上方的元素通常都是同一个常数 `c`，常见情况是 `0`。

只需要存储下三角部分，再额外存储常数 `c`。

需要存储的有效元素个数为：

$$
\frac{n(n + 1)}{2}
$$

若把常数 `c` 也存入压缩数组最后一个位置，则总存储单元数为：

$$
\frac{n(n + 1)}{2} + 1
$$

下标从 `1` 开始时：

- 若 `i >= j`，使用下三角映射。
- 若 `i < j`，访问常数 `c`。

### 下三角矩阵下标映射

::: details 点击展开代码

```c
int LowerTriangularIndex(int i, int j, int n) {
    if (i >= j) {
        int k = i * (i - 1) / 2 + j;
        return k - 1;
    }

    return n * (n + 1) / 2;
}
```

:::

### 上三角矩阵

上三角矩阵中，主对角线下方的元素通常都是同一个常数 `c`。

若按行优先存储上三角部分：

```txt
a11 a12 ... a1n
    a22 ... a2n
        ...
            ann
```

设下标从 `1` 开始，压缩数组 `B` 也从 `1` 开始编号。

第 `i` 行之前共有：

$$
(n) + (n - 1) + \cdots + (n - i + 2)
$$

个元素。

因此当 `i <= j` 时：

$$
k = \frac{(i - 1)(2n - i + 2)}{2} + (j - i + 1)
$$

若 `i > j`，访问常数 `c`。

### 上三角矩阵下标映射

::: details 点击展开代码

```c
int UpperTriangularIndex(int i, int j, int n) {
    if (i <= j) {
        int k = (i - 1) * (2 * n - i + 2) / 2 + (j - i + 1);
        return k - 1;
    }

    return n * (n + 1) / 2;
}
```

:::

## 7.3 三对角矩阵

三对角矩阵又称带状矩阵。

对于 `n * n` 矩阵，若非零元素只可能出现在主对角线及其上下两条对角线上，即：

$$
|i - j| \le 1
$$

则称为三对角矩阵。

三对角矩阵中非零元素个数最多为：

$$
3n - 2
$$

因为：

- 主对角线有 `n` 个元素。
- 上对角线有 `n - 1` 个元素。
- 下对角线有 `n - 1` 个元素。

总数为：

$$
n + (n - 1) + (n - 1) = 3n - 2
$$

### 三对角矩阵压缩存储

按行优先存储三对角线上的元素：

```txt
a11 a12
a21 a22 a23
a32 a33 a34
...
a(n)(n-1) a(n)(n)
```

设下标从 `1` 开始，压缩数组 `B` 从 `1` 开始编号。

对于非零元素 `aij`，满足 `|i - j| <= 1`。

第 `i` 行之前的非零元素个数为：

- 第 1 行有 `2` 个。
- 第 2 行到第 `i - 1` 行，每行有 `3` 个。

因此当 `2 <= i <= n - 1` 时，第 `i` 行第一个非零元素在压缩数组中的位置为：

$$
2 + 3(i - 2) + 1 = 3i - 3
$$

该行第一个非零元素是 `a(i)(i-1)`。

所以一般映射公式为：

$$
k = 2i + j - 2
$$

这个公式对第一行和最后一行的非零元素也成立。

若 C 语言数组 `B` 从 `0` 开始，则实际下标为：

$$
k - 1 = 2i + j - 3
$$

### 三对角矩阵下标映射

::: details 点击展开代码

```c
int TriDiagonalIndex(int i, int j) {
    if (i - j > 1 || j - i > 1) {
        return -1;
    }

    return 2 * i + j - 3;
}
```

:::

## 7.4 稀疏矩阵

若矩阵中大多数元素为 `0`，只有少数非零元素，则称为稀疏矩阵。

稀疏矩阵通常不适合直接使用二维数组存储，因为会浪费大量空间。

常见压缩存储方式：

- 三元组顺序表
- 十字链表

408 中最常考的是三元组顺序表。

## 7.5 三元组顺序表

三元组用来描述一个非零元素：

```txt
(row, col, value)
```

分别表示非零元素所在行、所在列和元素值。

例如矩阵：

```txt
0 0 5
0 7 0
9 0 0
```

可以表示为：

```txt
(1, 3, 5)
(2, 2, 7)
(3, 1, 9)
```

### 三元组定义

::: details 点击展开代码

```c
#define MaxTerms 100
#define MaxCols 100

typedef int ElemType;

typedef struct {
    int row;
    int col;
    ElemType value;
} Triple;

typedef struct {
    Triple data[MaxTerms];
    int rows;
    int cols;
    int nums;
} SparseMatrix;
```

:::

## 7.6 稀疏矩阵转置

稀疏矩阵转置是 408 的常见考点。

普通转置需要把所有元素的行列互换。

对于三元组，只需要：

- 新矩阵行数等于原矩阵列数。
- 新矩阵列数等于原矩阵行数。
- 每个三元组的 `row` 与 `col` 互换。

但如果要保持三元组按行优先有序，需要按原矩阵列号从小到大扫描。

### 普通转置

实现逻辑：依次枚举原矩阵列号，把该列中的非零元素按顺序写入转置矩阵。

::: details 点击展开代码

```c
void TransposeSparseMatrix(SparseMatrix A, SparseMatrix *B) {
    B->rows = A.cols;
    B->cols = A.rows;
    B->nums = A.nums;

    int index = 0;

    for (int col = 1; col <= A.cols; col++) {
        for (int i = 0; i < A.nums; i++) {
            if (A.data[i].col == col) {
                B->data[index].row = A.data[i].col;
                B->data[index].col = A.data[i].row;
                B->data[index].value = A.data[i].value;
                index++;
            }
        }
    }
}
```

:::

普通转置时间复杂度为：

$$
O(cols \times nums)
$$

其中 `cols` 为原矩阵列数，`nums` 为非零元素个数。

### 快速转置

快速转置通过预先统计每一列的非零元素个数，直接计算转置后每一行第一个元素的位置。

核心数组：

- `num[col]`：原矩阵第 `col` 列中非零元素个数。
- `pos[col]`：原矩阵第 `col` 列第一个非零元素在转置矩阵三元组表中的起始位置。

::: details 点击展开代码

```c
void FastTransposeSparseMatrix(SparseMatrix A, SparseMatrix *B) {
    int num[MaxCols + 1] = {0};
    int pos[MaxCols + 1] = {0};

    B->rows = A.cols;
    B->cols = A.rows;
    B->nums = A.nums;

    if (A.nums == 0) {
        return;
    }

    for (int i = 0; i < A.nums; i++) {
        num[A.data[i].col]++;
    }

    pos[1] = 0;

    for (int col = 2; col <= A.cols; col++) {
        pos[col] = pos[col - 1] + num[col - 1];
    }

    for (int i = 0; i < A.nums; i++) {
        int col = A.data[i].col;
        int index = pos[col];

        B->data[index].row = A.data[i].col;
        B->data[index].col = A.data[i].row;
        B->data[index].value = A.data[i].value;

        pos[col]++;
    }
}
```

:::

快速转置时间复杂度为：

$$
O(cols + nums)
$$

## 8. 408 常考点

### 8.1 地址计算先看下标起点

数组地址计算题最容易错在下标起点。

题目可能使用：

- `A[0...n-1]`
- `A[1...n]`
- `A[l...u]`

若下标范围是 `A[l...u]`，则第 `i` 个元素前面有 `i - l` 个元素。

### 8.2 行优先与列优先

C 语言采用行优先存储。

做题时若题目没有说明语言，而明确说“按列优先存储”，就必须使用列优先公式。

### 8.3 对称矩阵只存一半

对称矩阵只需要存储上三角或下三角，包括主对角线。

存储元素个数为：

$$
\frac{n(n + 1)}{2}
$$

### 8.4 三角矩阵多存一个常数

三角矩阵中另一半元素通常是同一个常数。

若把这个常数也存入压缩数组，则需要：

$$
\frac{n(n + 1)}{2} + 1
$$

个存储单元。

### 8.5 三对角矩阵公式

三对角矩阵中，非零元素满足：

$$
|i - j| \le 1
$$

按行优先压缩时，若下标从 `1` 开始，压缩数组从 `1` 开始，则：

$$
k = 2i + j - 2
$$

若压缩数组从 `0` 开始，则：

$$
k = 2i + j - 3
$$

### 8.6 稀疏矩阵转置

三元组顺序表的普通转置按列扫描，时间复杂度为：

$$
O(cols \times nums)
$$

快速转置通过统计每列非零元素个数，时间复杂度为：

$$
O(cols + nums)
$$

## 9. 小结

| 内容 | 关键点 |
| --- | --- |
| 一维数组 | 首地址 + 偏移量 |
| 二维数组 | 行优先或列优先 |
| 对称矩阵 | 只存上三角或下三角 |
| 三角矩阵 | 三角部分 + 一个常数 |
| 三对角矩阵 | 只存三条对角线，最多 `3n - 2` 个元素 |
| 稀疏矩阵 | 三元组顺序表、快速转置 |

数组与特殊矩阵的核心是“下标映射”。

做题时先确定逻辑下标，再确定压缩数组下标，最后再代入地址公式。
