<h1><center>第四节 B 树和 B+ 树</center></h1>

## 1. B 树

B 树是一种多路平衡查找树，常用于磁盘、数据库、文件系统等外存查找场景。

二叉排序树、AVL 树每个结点最多只有两个孩子。若数据规模很大，树高仍然可能较高，查找时需要多次访问结点。

B 树的思想是：一个结点中存放多个关键字，并拥有多棵子树，从而降低树的高度，减少外存访问次数。

## 2. m 阶 B 树的定义

一棵 m 阶 B 树满足以下性质：

- 每个结点至多有 `m` 棵子树。
- 每个结点至多有 `m - 1` 个关键字。
- 若根结点不是叶结点，则根结点至少有 `2` 棵子树。
- 除根结点外，所有非叶结点至少有 `ceil(m / 2)` 棵子树。
- 除根结点外，所有结点至少有 `ceil(m / 2) - 1` 个关键字。
- 有 `k` 棵子树的非叶结点，恰好有 `k - 1` 个关键字。
- 所有叶结点都出现在同一层次上。
- 结点内关键字有序，子树中的关键字范围由结点中的关键字分隔。

对于一个含有关键字：

$$
K_1 < K_2 < \cdots < K_n
$$

的结点，它有 `n + 1` 棵子树：

$$
P_0, P_1, \cdots, P_n
$$

其关键字范围满足：

- `P0` 中所有关键字小于 `K1`。
- `Pi` 中所有关键字大于 `Ki` 且小于 `K(i+1)`。
- `Pn` 中所有关键字大于 `Kn`。

## 3. 结点关键字个数范围

设 B 树阶数为 `m`。

非根结点的子树个数范围：

$$
\lceil m / 2 \rceil \le 子树个数 \le m
$$

非根结点的关键字个数范围：

$$
\lceil m / 2 \rceil - 1 \le 关键字个数 \le m - 1
$$

根结点比较特殊：

- 若根结点是叶结点，可以只有 `0` 个或多个关键字。
- 若根结点不是叶结点，至少有 `2` 棵子树，至少有 `1` 个关键字。

例如 5 阶 B 树：

| 项目 | 范围 |
| --- | --- |
| 每个结点最多子树数 | 5 |
| 每个结点最多关键字数 | 4 |
| 非根结点最少子树数 | 3 |
| 非根结点最少关键字数 | 2 |

## 4. 查找

B 树查找类似二叉排序树查找，只是每个结点中有多个关键字。

查找过程：

1. 在当前结点内顺序查找或折半查找目标关键字。
2. 若找到，则查找成功。
3. 若未找到，则根据关键字范围选择对应子树继续查找。
4. 若查找到叶结点仍失败，则查找失败。

一次查找经过的结点数不超过树的高度。

在外存查找中，通常把一个结点设计成一个磁盘块。每访问一个结点，相当于一次磁盘 I/O。因此 B 树通过增加分支数降低树高，从而减少 I/O 次数。

## 5. 插入

B 树插入总是在叶结点中进行。

插入过程：

1. 按查找规则找到应该插入的叶结点。
2. 将新关键字插入该叶结点，保持结点内关键字有序。
3. 若插入后关键字个数不超过 `m - 1`，则插入结束。
4. 若关键字个数达到 `m`，说明结点溢出，需要分裂。

### 5.1 分裂

当 m 阶 B 树某结点插入后有 `m` 个关键字时，需要分裂。

分裂方法：

1. 取中间关键字上升到父结点。
2. 中间关键字左侧关键字形成左结点。
3. 中间关键字右侧关键字形成右结点。
4. 若父结点也溢出，则继续向上分裂。
5. 若根结点溢出，则生成新的根结点，树高加一。

对于 `m` 个关键字：

$$
K_1, K_2, \cdots, K_m
$$

通常令第 `ceil(m / 2)` 个关键字上升。

以 5 阶 B 树为例，一个结点最多有 4 个关键字。若插入后出现 5 个关键字：

```txt
[10 20 30 40 50]
```

中间关键字 `30` 上升：

```txt
        [30]
       /    \
[10 20]    [40 50]
```

## 6. 删除

B 树删除比插入更复杂。删除后需要保证每个非根结点的关键字个数不少于：

$$
\lceil m / 2 \rceil - 1
$$

删除分为两类情况。

### 6.1 删除叶结点中的关键字

若要删除的关键字在叶结点中：

1. 直接删除该关键字。
2. 若删除后关键字个数仍不少于下限，则结束。
3. 若低于下限，则需要向兄弟借关键字或与兄弟合并。

### 6.2 删除非叶结点中的关键字

若要删除的关键字在非叶结点中，不能直接删除后破坏子树分隔关系。

常用方法：

- 用该关键字的前驱关键字替代它，再删除前驱关键字。
- 或用该关键字的后继关键字替代它，再删除后继关键字。

前驱或后继一定在叶结点中，因此最终会转化为删除叶结点关键字的问题。

## 7. 删除后的调整

删除后若某结点关键字个数低于下限，需要调整。

### 7.1 向兄弟借关键字

若相邻兄弟结点关键字个数大于下限，可以向兄弟借一个关键字。

向左兄弟借：

1. 父结点中分隔两个兄弟的关键字下移到当前结点。
2. 左兄弟中最大关键字上移到父结点。

向右兄弟借：

1. 父结点中分隔两个兄弟的关键字下移到当前结点。
2. 右兄弟中最小关键字上移到父结点。

借关键字后，当前结点关键字个数恢复合法，B 树性质保持不变。

### 7.2 与兄弟合并

若相邻兄弟结点也只有最少关键字，无法借关键字，则需要合并。

合并方法：

1. 将当前结点、父结点中的分隔关键字、相邻兄弟结点合并成一个结点。
2. 父结点失去一个关键字和一棵子树。
3. 若父结点因此低于下限，则继续向上调整。

若根结点关键字个数变为 `0`，则让合并后的唯一孩子成为新的根，树高减一。

## 8. B 树高度

设 B 树阶数为 `m`，含有 `n` 个关键字。

B 树中每个结点最多有 `m` 棵子树，因此树高较低。

408 中常见考法是估算高度范围。

### 8.1 最大高度

要让高度最大，应让每个结点尽可能少地保存关键字和子树。

设：

$$
t = \lceil m / 2 \rceil
$$

除根结点外，每个非叶结点至少有 `t` 棵子树。

若树高为 `h`，根结点至少有 `2` 棵子树，则第 `h` 层至少有：

$$
2t^{h - 2}
$$

个结点。

因此常可通过最少结点数推高度上界。

### 8.2 最小高度

要让高度最小，应让每个结点尽可能多地保存关键字和子树。

每个结点最多有 `m` 棵子树，最多有 `m - 1` 个关键字。

因此高度约为：

$$
O(\log_m n)
$$

## 9. 408 常考点

### 9.1 m 阶的含义

m 阶 B 树中的 `m` 表示一个结点最多有 `m` 棵子树，而不是最多有 `m` 个关键字。

所以：

- 最多子树数为 `m`。
- 最多关键字数为 `m - 1`。

### 9.2 关键字个数与孩子个数

B 树中，非叶结点若有 `k` 个关键字，则一定有 `k + 1` 棵子树。

反过来，若有 `k` 棵子树，则一定有 `k - 1` 个关键字。

### 9.3 所有叶结点同层

B 树是平衡的多路查找树，所有叶结点都在同一层。

这保证了查找路径长度相近，不会出现普通二叉排序树那样严重退化的情况。

### 9.4 插入一定发生在叶结点

B 树插入新关键字时，先查找失败位置，新关键字一定插入到叶结点。

插入后若结点溢出，则分裂，并可能一路向上调整。

### 9.5 删除可能一路向上调整

删除后若结点关键字个数不足，可能需要借关键字或合并。

合并会使父结点关键字个数减少，因此可能继续向上调整，甚至使树高减一。

## 10. 小结

| 项目 | B 树 |
| --- | --- |
| 类型 | 多路平衡查找树 |
| 常见用途 | 外存查找、数据库索引、文件系统 |
| 结点关键字 | 每个结点都可以保存关键字和对应记录 |
| 查找路径 | 可能在非叶结点成功 |
| 树高 | $O(\log_m n)$ |
| 插入 | 在叶结点插入，溢出则分裂 |
| 删除 | 可能借关键字或合并 |

## 11. B+ 树

B+ 树是 B 树的一种变形，也是一种多路平衡查找树。

它广泛用于数据库索引和文件系统索引。

B+ 树与 B 树最大的不同是：

- B 树中每个结点都可以保存关键字和对应记录。
- B+ 树中非叶结点通常只保存索引关键字，所有记录都保存在叶结点中。
- B+ 树的叶结点按关键字大小顺序链接，便于顺序查找和范围查询。

## 12. m 阶 B+ 树的定义

一棵 m 阶 B+ 树通常满足：

- 每个结点最多有 `m` 棵子树。
- 非根内部结点至少有 `ceil(m / 2)` 棵子树。
- 根结点若不是叶结点，至少有 `2` 棵子树。
- 所有叶结点都在同一层。
- 非叶结点只起索引作用。
- 所有数据记录都存放在叶结点中。
- 叶结点按关键字大小顺序链接。

在 408 常用定义中，B+ 树非叶结点中有 `k` 棵子树时，通常有 `k` 个关键字。

这些关键字用于指示子树中的最大关键字或分界关键字，具体取决于教材约定。做题时要以题目给出的定义为准。

## 13. B+ 树的结构特点

### 13.1 非叶结点是索引

B+ 树的非叶结点不保存完整记录，只保存索引关键字和子树指针。

因此非叶结点可以容纳更多索引项，使树的分支数更大，树高更低。

### 13.2 叶结点保存所有记录

所有关键字和对应记录都出现在叶结点中。

因此一次成功查找通常都要走到叶结点。

这与 B 树不同。B 树查找时，如果关键字在内部结点命中，就可以直接成功。

### 13.3 叶结点顺序链接

B+ 树的叶结点通常用指针按关键字大小顺序链接起来。

这样可以高效支持：

- 顺序查找
- 范围查询
- 区间扫描

例如查找 `[20, 50]` 范围内的记录时，先定位到第一个不小于 `20` 的叶结点位置，然后沿叶结点链表向后扫描即可。

## 14. B+ 树的查找、插入和删除

### 14.1 查找

B+ 树查找过程：

1. 从根结点开始。
2. 在当前非叶结点中查找目标关键字应当进入的子树。
3. 沿对应指针进入下一层。
4. 重复直到叶结点。
5. 在叶结点中查找目标关键字。

在 B+ 树中，即使目标关键字在非叶结点中出现，也不能直接停止。因为非叶结点中的关键字只是索引，真正的数据记录在叶结点中。

### 14.2 插入

B+ 树插入一定发生在叶结点。

插入过程：

1. 查找到应插入的叶结点。
2. 将新关键字和记录插入叶结点，保持有序。
3. 若叶结点未溢出，则插入结束。
4. 若叶结点溢出，则分裂叶结点，并将分裂后的索引关键字插入父结点。
5. 若父结点也溢出，则继续向上分裂。
6. 若根结点溢出，则生成新的根结点，树高加一。

### 14.3 删除

B+ 树删除记录时，一定从叶结点删除。

删除过程：

1. 查找到目标关键字所在的叶结点。
2. 删除对应关键字和记录。
3. 若叶结点关键字个数仍满足下限，则删除结束。
4. 若低于下限，则尝试向相邻兄弟借关键字。
5. 若兄弟无法借，则与兄弟合并。
6. 合并可能导致父结点关键字减少，从而继续向上调整。

删除时还要注意更新父结点中的索引关键字。

若某个叶结点的最小关键字发生变化，父结点中用于索引该叶结点的关键字可能也需要同步修改。

## 15. B 树与 B+ 树比较

| 项目 | B 树 | B+ 树 |
| --- | --- | --- |
| 数据记录位置 | 可以在任意结点 | 只在叶结点 |
| 非叶结点作用 | 保存关键字和记录，也可索引 | 只作为索引 |
| 查找成功位置 | 可能在非叶结点 | 一定在叶结点 |
| 叶结点关系 | 通常不要求链接 | 通常按序链接 |
| 顺序查找 | 相对不方便 | 很方便 |
| 范围查询 | 效率较低 | 效率高 |
| 树高 | 较低 | 通常更低或相近 |
| 应用 | 多路查找 | 数据库索引更常见 |

## 16. 408 常考点补充

### 16.1 B+ 树查找一定到叶结点

即使目标关键字在非叶结点中出现，也只是索引命中，不能直接返回记录。

真正记录在叶结点。

### 16.2 非叶结点关键字数量

408 中常见说法：

- B 树：有 `k` 棵子树的结点有 `k - 1` 个关键字。
- B+ 树：有 `k` 棵子树的非叶结点通常有 `k` 个关键字。

但不同教材对 B+ 树索引关键字含义有差异，遇到具体题目时要以题干定义为准。

### 16.3 叶结点链表支持范围查询

B+ 树比 B 树更适合顺序查找和范围查找。

原因不是它“更平衡”，而是所有数据都在叶结点，并且叶结点按序链接。

## 17. 本节小结

| 项目 | B 树 | B+ 树 |
| --- | --- | --- |
| 类型 | 多路平衡查找树 | 多路平衡查找树 |
| 记录位置 | 任意结点 | 全部在叶结点 |
| 查找成功 | 可在非叶结点成功 | 一定到叶结点 |
| 范围查询 | 相对不方便 | 很方便 |
| 常见用途 | 外存查找 | 数据库索引、文件系统索引 |

B 树和 B+ 树的核心都是降低树高，减少外存访问次数；区别在于 B+ 树把数据集中放到叶结点，并用叶结点链表支持范围查询。
