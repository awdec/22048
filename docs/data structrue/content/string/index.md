<h1><center>第三章 字符串</center></h1>

## 1. 本章定位

字符串简称串，是由零个或多个字符组成的有限序列。

串在逻辑结构上属于线性结构，但它的数据元素通常是字符，因此在存储、比较和模式匹配上有一些专门问题。

本章重点不是“字符串是什么”，而是：

1. 串的基本概念与术语。
2. 串的存储方式。
3. 串的模式匹配。
4. 朴素匹配算法与 KMP 算法。

::: tip 重点
408 中字符串最核心的考点是模式匹配，尤其是 KMP 算法中的 `next` 数组与匹配过程。
:::

## 3. 学习主线

本章建议按下面顺序复习：

$$
\text{串的概念}
\longrightarrow
\text{串的存储}
\longrightarrow
\text{朴素模式匹配}
\longrightarrow
\text{KMP 算法}
\longrightarrow
\text{next 数组}
$$

学习时要特别注意：

- 位序从 `1` 开始还是从 `0` 开始。
- 主串和模式串不能混淆。
- 匹配失败时，朴素算法移动主串，KMP 算法移动模式串。
- `next` 数组的定义要以题目或教材约定为准。
