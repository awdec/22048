<h1><center>第二节 本章总结</center></h1>

## 1. 本章核心

字符串是特殊的线性表。

普通线性表中的数据元素可以是任意类型，而字符串中的数据元素通常是字符。

字符串一般记作：

$$
S="a_1a_2\cdots a_n"
$$

其中 $n$ 是串长。

当 $n=0$ 时，称为空串。

::: tip 重点
字符串这一章的核心不是普通插入删除，而是“模式匹配”，即在主串中查找模式串出现的位置。
:::

## 2. 基本概念总结

| 概念 | 含义 |
| --- | --- |
| 串 | 零个或多个字符组成的有限序列 |
| 空串 | 不含任何字符的串，长度为 `0` |
| 空格串 | 由一个或多个空格组成的串，长度不为 `0` |
| 子串 | 串中任意个连续字符组成的序列 |
| 主串 | 包含某个子串的串 |
| 模式串 | 模式匹配中被查找的串 |
| 串长 | 串中字符个数 |

容易混淆的是空串和空格串：

| 串 | 含义 | 长度 |
| --- | --- | --- |
| `""` | 空串 | `0` |
| `" "` | 空格串 | `1` |

## 3. 串和线性表的关系

串可以看作特殊的线性表。

相同点：

- 都是线性结构。
- 元素之间存在一对一关系。
- 都可以顺序存储或链式存储。

不同点：

- 线性表的数据元素可以是任意类型。
- 串的数据元素通常是字符。
- 串更关注整体操作，如求子串、比较、连接、模式匹配。

## 4. 串的存储方式

常见存储方式：

| 存储方式 | 特点 |
| --- | --- |
| 定长顺序存储 | 空间固定，可能溢出 |
| 堆分配存储 | 动态分配空间，长度灵活 |
| 块链存储 | 用链表块保存字符 |

408 中常见的是顺序存储，尤其是模式匹配算法中主串和模式串通常按数组理解。

## 5. 朴素模式匹配

朴素模式匹配的思想：

1. 从主串某个位置开始。
2. 将模式串从头开始逐个字符比较。
3. 如果匹配失败，主串起始位置后移一位。
4. 模式串重新从第一个字符开始比较。

设主串长度为 $n$，模式串长度为 $m$。

最坏时间复杂度：

$$
O(nm)
$$

::: warning 易错
朴素匹配失败后，主串指针通常要回溯，模式串指针也要重新从头开始。
:::

## 6. KMP 模式匹配

KMP 算法的核心思想：

匹配失败时，不回溯主串指针，而是利用模式串自身的前后缀信息移动模式串。

也就是说：

$$
\text{主串指针不回退，模式串指针按 next 数组跳转}
$$

KMP 的时间复杂度：

$$
O(n+m)
$$

其中：

- $n$ 是主串长度。
- $m$ 是模式串长度。

## 7. next 数组总结

`next` 数组用于记录模式串在失配时应该跳转到的位置。

它本质上反映的是：

$$
\text{模式串当前前缀与后缀的最长相等长度}
$$

理解时抓住一句话：

::: tip 核心
KMP 不是跳过主串，而是根据已经匹配过的信息，避免模式串从头重来。
:::

不同教材中 `next` 数组可能有不同写法：

- 有的从 `0` 开始。
- 有的从 `1` 开始。
- 有的使用 `next`。
- 有的使用 `nextval`。

做题时要先看题目给出的定义。

## 8. 朴素匹配与 KMP 对比

| 项目 | 朴素模式匹配 | KMP |
| --- | --- | --- |
| 主串指针 | 可能回溯 | 不回溯 |
| 模式串指针 | 失败后回到开头 | 根据 `next` 跳转 |
| 利用已匹配信息 | 不利用 | 利用 |
| 最坏复杂度 | $O(nm)$ | $O(n+m)$ |
| 重点 | 模拟过程 | `next` 数组 |

## 9. 解题方法总结

### 模式匹配题

1. 先区分主串和模式串。
2. 明确下标从 `0` 还是从 `1` 开始。
3. 若是朴素匹配，逐轮模拟比较。
4. 若是 KMP，先求 `next` 数组。
5. 匹配失败时，根据 `next` 更新模式串位置。

### next 数组题

1. 写出模式串每个位置之前的子串。
2. 找最长相等真前缀和真后缀。
3. 按题目约定填入 `next`。
4. 检查首位的特殊规定。

### 复杂度题

| 算法 | 最坏时间复杂度 |
| --- | --- |
| 朴素模式匹配 | $O(nm)$ |
| KMP | $O(n+m)$ |

## 10. 易错点总结

| 易错点 | 正确理解 |
| --- | --- |
| 把空串和空格串混淆 | 空串长度为 `0`，空格串长度不为 `0` |
| 把子序列当子串 | 子串必须连续 |
| 主串和模式串混淆 | 主串被查找，模式串被匹配 |
| KMP 中主串指针回退 | KMP 主串指针不回退 |
| 直接套 next 数组 | 先确认题目采用的定义 |
| 忽略下标起点 | 先判断从 `0` 还是从 `1` 开始 |

## 11. 考前背诵版

1. 串是由零个或多个字符组成的有限序列。
2. 空串长度为 `0`，空格串长度不为 `0`。
3. 子串必须连续。
4. 模式匹配是在主串中查找模式串。
5. 朴素模式匹配最坏时间复杂度为 $O(nm)$。
6. KMP 算法主串指针不回退。
7. KMP 时间复杂度为 $O(n+m)$。
8. `next` 数组反映模式串前后缀信息。
9. `next` 的具体值要以题目定义为准。
