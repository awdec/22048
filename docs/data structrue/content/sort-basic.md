<h1><center>第一节 排序的基本概念</center></h1>

## 1. 排序

排序是指按照关键字大小，将一组记录重新排列成有序序列。

若按关键字从小到大排列，称为升序排序。

若按关键字从大到小排列，称为降序排序。

408 中默认讨论的是按关键字升序排列，除非题目特别说明。

## 2. 排序码

用于排序的数据项称为排序码，也常称为关键字。

若记录只有一个关键字，则排序规则比较直接。

若记录有多个关键字，题目通常会指定主关键字或排序优先级。

## 3. 内部排序和外部排序

### 3.1 内部排序

若待排序记录全部存放在内存中，排序过程不需要访问外存，则称为内部排序。

408 数据结构中主要考内部排序。

常见内部排序：

- 插入排序
- 交换排序
- 选择排序
- 归并排序
- 基数排序

### 3.2 外部排序

若待排序记录数量很大，内存不能一次容纳全部记录，需要借助外存，则称为外部排序。

外部排序的核心思想通常是：

1. 先在内存中形成若干有序初始归并段。
2. 再对这些归并段进行多路归并。

外部排序重点关注外存读写次数。

## 4. 稳定性

若序列中存在两个关键字相同的记录：

$$
K_i = K_j
$$

且排序前 $R_i$ 在 $R_j$ 前面。

若排序后 $R_i$ 仍在 $R_j$ 前面，则称该排序算法是稳定的。

否则称为不稳定的。

稳定性只讨论关键字相同的记录之间相对次序是否改变。

稳定排序常见有：

- 直接插入排序
- 折半插入排序
- 冒泡排序
- 归并排序
- 基数排序
- 计数排序

不稳定排序常见有：

- 希尔排序
- 快速排序
- 简单选择排序
- 堆排序

## 5. 排序算法分类

### 5.1 按基本思想分类

| 类型 | 典型算法 |
| --- | --- |
| 插入排序 | 直接插入排序、折半插入排序、希尔排序 |
| 交换排序 | 冒泡排序、快速排序 |
| 选择排序 | 简单选择排序、堆排序 |
| 归并排序 | 二路归并排序 |
| 分配类排序 | 基数排序、计数排序 |

### 5.2 按是否基于比较分类

基于比较的排序通过关键字比较决定记录顺序。

例如：

- 直接插入排序
- 冒泡排序
- 快速排序
- 简单选择排序
- 堆排序
- 归并排序

不完全基于比较的排序通常利用关键字范围或关键字位数。

例如：

- 基数排序
- 计数排序

## 6. 评价指标

评价一个排序算法，408 中常看四个指标：

- 时间复杂度
- 空间复杂度
- 稳定性
- 初始序列对性能的影响

时间复杂度要区分：

- 最好情况
- 平均情况
- 最坏情况

空间复杂度要看是否需要额外辅助数组、递归栈或桶。

## 7. 408 常考点

### 7.1 稳定性不是看复杂度

稳定性和时间复杂度无直接关系。

例如归并排序是 $O(n\log n)$ 且稳定。

堆排序也是 $O(n\log n)$，但不稳定。

### 7.2 初始序列会影响某些算法

直接插入排序、冒泡排序、快速排序都明显受初始序列影响。

堆排序和归并排序的时间复杂度通常比较稳定。

### 7.3 过程模拟很重要

408 经常给出一个序列，要求判断经过若干趟排序后的结果。

做这类题要明确“一趟”的含义：

- 插入排序的一趟是插入一个元素。
- 冒泡排序的一趟通常确定一个最大或最小元素。
- 快速排序的一趟是一次划分。
- 选择排序的一趟是选择一个最小或最大元素。
- 归并排序的一趟是归并段长度扩大一倍。

## 8. 小结

| 内容 | 重点 |
| --- | --- |
| 排序 | 按关键字重排记录 |
| 内部排序 | 数据全部在内存中 |
| 外部排序 | 需要访问外存 |
| 稳定性 | 相等关键字相对次序不变 |
| 时间复杂度 | 最好、平均、最坏 |
| 空间复杂度 | 辅助空间或递归栈 |

排序的基本概念要抓住一句话：过程要会模拟，性质要会比较。
