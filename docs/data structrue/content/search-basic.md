<h1><center>第一节 查找的基本概念</center></h1>

## 1. 查找

查找是指在数据集合中，根据给定关键字寻找满足条件的数据元素。

若找到关键字等于给定值的元素，则称查找成功。

若不存在这样的元素，则称查找失败。

查找操作通常关注两个问题：

- 能否找到目标元素。
- 平均需要比较多少次。

## 2. 查找表

由同一类型的数据元素或记录构成的集合称为查找表。

查找表中的元素通常包含多个数据项。

用于标识元素的数据项称为关键字。

### 2.1 主关键字

若关键字可以唯一标识一个记录，则称为主关键字。

例如学生表中的学号通常可以作为主关键字。

### 2.2 次关键字

若关键字不能唯一标识一个记录，则称为次关键字。

例如学生表中的姓名可能重复，因此通常只能作为次关键字。

## 3. 静态查找表和动态查找表

### 3.1 静态查找表

只进行查找操作，不改变表中元素的查找表称为静态查找表。

典型查找方法：

- 顺序查找
- 折半查找
- 分块查找

静态查找表不涉及插入和删除，因此重点通常是查找过程和平均查找长度。

### 3.2 动态查找表

在查找过程中还需要插入或删除元素的查找表称为动态查找表。

典型结构：

- 二叉排序树
- 平衡二叉树
- 红黑树
- B 树
- B+ 树
- 散列表

动态查找表不仅要考虑查找效率，还要考虑插入、删除后结构是否仍然满足要求。

## 4. 平均查找长度

平均查找长度简称 ASL。

ASL 表示为了确定查找成功或失败，平均需要比较关键字的次数。

一般形式为：

$$
ASL = \sum_{i=1}^{n} p_i c_i
$$

其中：

- $p_i$ 表示查找第 $i$ 个元素的概率。
- $c_i$ 表示查找第 $i$ 个元素需要比较的次数。

若每个元素被查找的概率相同，则：

$$
p_i = \frac{1}{n}
$$

因此：

$$
ASL = \frac{1}{n}\sum_{i=1}^{n} c_i
$$

## 5. 查找成功和查找失败

### 5.1 查找成功 ASL

查找成功 ASL 统计的是目标元素存在时的平均比较次数。

它通常和表中已有元素的位置、树中结点的层次、散列表中成功查找路径有关。

### 5.2 查找失败 ASL

查找失败 ASL 统计的是目标元素不存在时的平均比较次数。

它通常和失败位置有关。

例如：

- 折半查找中，失败位置对应判定树的外部结点。
- 散列表中，失败查找要看探测序列直到空位置。

408 题中，查找失败 ASL 常比查找成功 ASL 更容易出错。

## 6. 常见查找方法

| 方法 | 类型 | 适用条件 | 重点 |
| --- | --- | --- | --- |
| 顺序查找 | 静态查找 | 无特殊要求 | 哨兵、ASL |
| 折半查找 | 静态查找 | 有序顺序表 | 判定树 |
| 分块查找 | 静态查找 | 块间有序 | 索引表 |
| 二叉排序树 | 动态查找 | 关键字可比较 | 树高 |
| AVL 树 | 动态查找 | 保持平衡 | 旋转 |
| 红黑树 | 动态查找 | 近似平衡 | 红黑性质 |
| B 树/B+ 树 | 动态查找 | 外存查找 | 多路平衡 |
| 散列表 | 动态查找 | 散列函数 | 冲突处理 |

## 7. 408 常考点

### 7.1 ASL 是核心

查找章节最常见的计算指标是 ASL。

做题时先明确统计对象：

- 查找成功
- 查找失败
- 等概率
- 不等概率

### 7.2 查找方法有前提

折半查找必须满足：

- 顺序存储
- 关键字有序

若是链表，即使有序，也不能直接进行高效折半查找。

### 7.3 动态结构要维护性质

二叉排序树要维护左小右大的性质。

AVL 树要维护平衡条件。

红黑树要维护颜色性质和黑高一致。

B 树和 B+ 树要维护结点关键字个数范围和平衡性。

散列表要维护冲突处理规则。

## 8. 小结

| 内容 | 重点 |
| --- | --- |
| 查找表 | 数据元素集合 |
| 关键字 | 标识数据元素的数据项 |
| 静态查找 | 只查找，不插入删除 |
| 动态查找 | 查找、插入、删除都可能发生 |
| ASL | 平均比较次数 |
| 成功 ASL | 目标存在 |
| 失败 ASL | 目标不存在 |

查找的基本概念要抓住一句话：用比较次数衡量查找效率，用结构性质减少比较次数。
