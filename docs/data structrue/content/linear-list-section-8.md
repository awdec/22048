<h1><center>第八节 本章总结</center></h1>

## 1. 本章核心

线性表是最基本的线性结构，描述的是数据元素之间的一对一关系。

线性表可以表示为：

$$
L=(a_1,a_2,\cdots,a_i,a_{i+1},\cdots,a_n)
$$

其中：

- 除第一个元素外，每个元素有且仅有一个直接前驱。
- 除最后一个元素外，每个元素有且仅有一个直接后继。

本章围绕线性结构展开：

$$
\text{线性表}
\longrightarrow
\text{顺序表}
\longrightarrow
\text{链表}
\longrightarrow
\text{栈}
\longrightarrow
\text{队列}
\longrightarrow
\text{数组和特殊矩阵}
$$

::: tip 重点
线性表是逻辑结构，顺序表和链表是存储实现；栈和队列是操作受限的线性表。
:::

## 2. 线性表基本操作

线性表常见基本操作：

| 操作 | 含义 |
| --- | --- |
| 初始化 | 建立空表 |
| 销毁 | 释放线性表 |
| 清空 | 删除所有元素 |
| 求长度 | 返回元素个数 |
| 判空 | 判断是否为空表 |
| 按位查找 | 根据位序找元素 |
| 按值查找 | 根据元素值找位置 |
| 插入 | 在指定位置插入元素 |
| 删除 | 删除指定位置元素 |
| 遍历 | 依次访问所有元素 |

408 中经常考：

- 插入、删除是否合法。
- 位序从 `1` 开始还是下标从 `0` 开始。
- 操作的时间复杂度。

## 3. 顺序表总结

顺序表采用一组地址连续的存储单元保存线性表元素。

特点：

| 项目 | 结论 |
| --- | --- |
| 存储方式 | 顺序存储 |
| 逻辑相邻 | 物理位置也相邻 |
| 按位查找 | $O(1)$ |
| 按值查找 | $O(n)$ |
| 插入删除 | 平均 $O(n)$ |
| 存储密度 | 高 |
| 扩容要求 | 需要连续空间 |

顺序表适合：

- 经常按位访问。
- 插入删除较少。
- 元素个数变化不剧烈。

::: warning 易错
顺序表插入删除慢，不是因为找位置慢，而是因为需要移动大量元素。
:::

## 4. 链表总结

链表用指针表示元素之间的逻辑关系。

常见类型：

| 类型 | 特点 |
| --- | --- |
| 单链表 | 每个结点只有一个后继指针 |
| 双链表 | 同时有前驱和后继指针 |
| 循环链表 | 尾结点指向头部 |
| 静态链表 | 用数组模拟链式存储 |

链表特点：

| 项目 | 结论 |
| --- | --- |
| 存储方式 | 链式存储 |
| 是否连续 | 不要求连续空间 |
| 按位查找 | $O(n)$ |
| 按值查找 | $O(n)$ |
| 插入删除 | 找到位置后 $O(1)$ |
| 存储密度 | 较低，需要指针域 |

链表适合：

- 插入删除频繁。
- 元素个数变化较大。
- 不要求随机访问。

### 顺序表与链表对比

| 对比点 | 顺序表 | 链表 |
| --- | --- | --- |
| 存储空间 | 连续 | 可不连续 |
| 随机访问 | 支持 $O(1)$ | 不支持，$O(n)$ |
| 插入删除 | 需要移动元素 | 修改指针 |
| 存储密度 | 高 | 较低 |
| 扩容 | 不方便 | 较方便 |

## 5. 栈总结

栈是只允许在一端进行插入和删除的线性表。

特点：

$$
\text{后进先出}
$$

即：

$$
LIFO
$$

基本操作：

| 操作 | 含义 |
| --- | --- |
| InitStack | 初始化栈 |
| Push | 入栈 |
| Pop | 出栈 |
| GetTop | 读栈顶元素 |
| StackEmpty | 判断栈空 |

常见考点：

- 入栈、出栈序列判断。
- 共享栈。
- 栈在递归中的应用。
- 栈在表达式求值中的应用。
- 栈在括号匹配中的应用。

::: tip 重点
看到“最近的、最后出现的、嵌套的、递归的”，优先想到栈。
:::

## 6. 队列总结

队列是只允许一端插入、另一端删除的线性表。

特点：

$$
\text{先进先出}
$$

即：

$$
FIFO
$$

基本操作：

| 操作 | 含义 |
| --- | --- |
| InitQueue | 初始化队列 |
| EnQueue | 入队 |
| DeQueue | 出队 |
| GetHead | 读队头元素 |
| QueueEmpty | 判断队空 |

常见队列：

| 类型 | 特点 |
| --- | --- |
| 顺序队列 | 用数组实现 |
| 循环队列 | 解决假溢出 |
| 链队列 | 用链表实现 |
| 双端队列 | 两端都可插入删除 |

### 循环队列关键公式

设数组容量为 `MaxSize`。

队尾指针进一：

$$
rear=(rear+1)\bmod MaxSize
$$

队头指针进一：

$$
front=(front+1)\bmod MaxSize
$$

若牺牲一个存储单元，则：

队空：

$$
front=rear
$$

队满：

$$
(rear+1)\bmod MaxSize=front
$$

队列长度：

$$
(rear-front+MaxSize)\bmod MaxSize
$$

::: warning 易错
循环队列中 `front == rear` 可能表示队空，也可能表示队满，所以常用“牺牲一个单元”或增加 `size/tag` 来区分。
:::

## 7. 栈和队列应用

### 栈的应用

栈常用于：

- 括号匹配
- 表达式求值
- 中缀表达式转后缀表达式
- 函数调用
- 递归
- 深度优先搜索

核心原因是栈具有后进先出的特性。

### 队列的应用

队列常用于：

- 层次遍历
- 广度优先搜索
- 缓冲区
- 操作系统进程调度

核心原因是队列具有先进先出的特性。

## 8. 数组和特殊矩阵

数组是线性表的推广。

二维数组按行优先存储时：

$$
LOC(a_{ij})=LOC(a_{00})+(i\times n+j)\times L
$$

按列优先存储时：

$$
LOC(a_{ij})=LOC(a_{00})+(j\times m+i)\times L
$$

特殊矩阵压缩存储的核心思想是：

$$
\text{只存储有规律或非零的元素}
$$

常见特殊矩阵：

| 矩阵 | 存储重点 |
| --- | --- |
| 对称矩阵 | 只存上三角或下三角 |
| 三角矩阵 | 只存三角部分和常数 |
| 三对角矩阵 | 只存主对角线及上下相邻对角线 |
| 稀疏矩阵 | 只存非零元素 |

| 顺序表按位查找 | $O(1)$ | - | - | 随机访问 |
| 顺序表按值查找 | $O(n)$ | - | - | 逐个比较 |
| 顺序表 | - | 平均 $O(n)$ | 平均 $O(n)$ | 需要移动元素 |

## 10. 解题方法总结

### 线性表题

1. 先判断逻辑结构。
2. 再判断存储结构。
3. 若是顺序表，重点考虑元素移动。
4. 若是链表，重点考虑指针修改。

### 栈题

1. 判断是否符合后进先出。
2. 出栈序列题用模拟法。
3. 括号匹配题遇左括号入栈，遇右括号出栈匹配。
4. 表达式题注意运算符优先级。

### 队列题

1. 判断是否符合先进先出。
2. 循环队列题先明确队空、队满条件。
3. 注意 `front` 和 `rear` 的含义。
4. 计算队列长度时使用取模公式。

### 矩阵压缩题

1. 先判断矩阵类型。
2. 确定存储区域。
3. 明确下标从 `0` 开始还是从 `1` 开始。
4. 写出二维下标到一维下标的映射公式。

## 11. 易错点总结

| 易错点 | 正确理解 |
| --- | --- |
| 把线性表和顺序表混为一谈 | 线性表是逻辑结构，顺序表是存储结构 |
| 顺序表插入删除复杂度写成 $O(1)$ | 平均需要移动元素，是 $O(n)$ |
| 链表插入删除直接写 $O(1)$ | 找到位置后才是 $O(1)$ |
| 忘记头结点不存数据 | 头结点只是操作方便 |
| 循环队列队空队满混淆 | 先看题目采用哪种判定方法 |
| 栈和队列顺序混淆 | 栈后进先出，队列先进先出 |
| 数组地址公式下标起点混乱 | 先确认从 `0` 还是从 `1` 开始 |

## 12. 考前背诵版

1. 线性表是逻辑结构，顺序表和链表是存储结构。
2. 顺序表支持随机访问，按位查找 $O(1)$。
3. 顺序表插入删除平均 $O(n)$。
4. 链表不支持随机访问，查找 $O(n)$。
5. 链表插入删除找到位置后 $O(1)$。
6. 栈是后进先出，队列是先进先出。
7. 循环队列常用取模实现首尾相接。
8. 特殊矩阵压缩存储的核心是减少重复或无效元素。
9. 复杂度题要区分“查找位置”和“真正插入删除”两个阶段。
