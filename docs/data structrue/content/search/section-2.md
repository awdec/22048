<h1><center>第二节 顺序查找和折半查找</center></h1>

## 1. 顺序查找

顺序查找又称线性查找。

基本思想：从表的一端开始，逐个比较关键字，直到查找成功或查找完整个表。

顺序查找适用范围广：

- 顺序表可以使用。
- 链表可以使用。
- 有序表可以使用。
- 无序表也可以使用。

## 2. 一般顺序查找

设查找表为：

$$
R[0], R[1], \cdots, R[n-1]
$$

从前向后查找关键字 `key`。

::: details 点击展开代码

```c
int SearchSeq(int A[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (A[i] == key) {
            return i;
        }
    }

    return -1;
}
```

:::

若等概率查找成功，则第 $i$ 个元素需要比较 $i$ 次。

查找成功的平均查找长度为：

$$
ASL_{成功} = \frac{1 + 2 + \cdots + n}{n} = \frac{n + 1}{2}
$$

查找失败时，需要比较 $n$ 次：

$$
ASL_{失败} = n
$$

## 3. 带哨兵的顺序查找

哨兵用于减少循环中的边界判断。

常见做法是把待查关键字放在数组的某个固定位置作为哨兵。

::: details 点击展开代码

```c
int SearchSeq(int A[], int n, int key) {
    A[0] = key;
    int i = n;

    while (A[i] != key) {
        i--;
    }

    return i;
}
```

:::

这段代码通常假设有效元素存放在 `A[1]` 到 `A[n]` 中，`A[0]` 是哨兵。

若返回 $0$，说明查找失败。

哨兵不会改变数量级复杂度，但可以减少每轮循环中的判断。

## 4. 有序表的顺序查找

若查找表有序，顺序查找失败时不一定要查完整个表。

例如表按关键字递增排列，当扫描到某个元素大于 `key` 时，就可以确定查找失败。

这会影响查找失败 ASL。

408 中若考有序表顺序查找，通常会结合判定树或失败位置统计比较次数。

## 5. 折半查找

折半查找又称二分查找。

适用条件：

- 顺序存储。
- 关键字有序。

链表即使有序，也不适合直接使用折半查找，因为链表不能随机访问中间元素。

## 6. 折半查找过程

设有序表递增排列。

基本过程：

1. 令 `low` 指向查找区间下界。
2. 令 `high` 指向查找区间上界。
3. 计算中间位置 `mid`。
4. 若 `A[mid] == key`，查找成功。
5. 若 `key < A[mid]`，在左半区继续查找。
6. 若 `key > A[mid]`，在右半区继续查找。
7. 若 `low > high`，查找失败。

::: details 点击展开代码

```c
int BinarySearch(int A[], int n, int key) {
    int low = 0;
    int high = n - 1;

    while (low <= high) {
        int mid = (low + high) / 2;

        if (A[mid] == key) {
            return mid;
        } else if (key < A[mid]) {
            high = mid - 1;
        } else {
            low = mid + 1;
        }
    }

    return -1;
}
```

:::

折半查找时间复杂度为：

$$
O(\log n)
$$

## 7. 折半查找判定树

折半查找过程可以用判定树表示。

判定树中：

- 每个内部结点对应一次关键字比较。
- 结点中的关键字是当前查找区间的中间元素。
- 左子树表示继续在左半区查找。
- 右子树表示继续在右半区查找。
- 外部结点对应查找失败的位置。

折半查找判定树通常接近平衡。

若有 $n$ 个关键字，则判定树有 $n$ 个成功结点和 $n + 1$ 个失败结点。

## 8. 折半查找 ASL

查找成功 ASL 需要统计每个成功结点所在层次。

若根结点层次为 $1$，则某结点所在层次就是查找该关键字需要比较的次数。

等概率时：

$$
ASL_{成功} = \frac{\text{所有成功结点层次之和}}{n}
$$

查找失败 ASL 需要统计所有失败结点的比较次数。

等概率时：

$$
ASL_{失败} = \frac{\text{所有失败结点比较次数之和}}{n + 1}
$$

注意：失败结点不是表中已有关键字，而是查找区间为空的位置。

## 9. 顺序查找与折半查找比较

| 项目 | 顺序查找 | 折半查找 |
| --- | --- | --- |
| 存储要求 | 顺序、链式均可 | 必须顺序存储 |
| 有序要求 | 不要求有序 | 必须有序 |
| 时间复杂度 | $O(n)$ | $O(\log n)$ |
| 插入删除 | 较方便 | 维护有序性有代价 |
| 常考点 | 哨兵、ASL | 判定树、ASL |

## 10. 408 常考点

### 10.1 折半查找前提

折半查找不是只要求“有序”，还要求“顺序存储”。

有序链表不能高效随机访问中间结点，因此不适合直接折半查找。

### 10.2 mid 的取法

常见取法：

$$
mid = \lfloor (low + high) / 2 \rfloor
$$

如果题目规定向上取整或其他取法，要按题目规定模拟。

不同 `mid` 取法可能导致判定树不同。

### 10.3 查找失败 ASL

折半查找失败 ASL 要统计失败结点，不是统计成功结点。

这是 408 查找题常见失分点。

### 10.4 有序顺序查找失败可以提前停止

无序表顺序查找失败要比较所有元素。

有序表顺序查找失败可能提前停止，比较次数要按失败位置统计。

## 11. 小结

| 内容 | 重点 |
| --- | --- |
| 顺序查找 | 逐个比较 |
| 哨兵 | 减少边界判断 |
| 折半查找 | 有序顺序表 |
| 判定树 | 分析成功和失败 ASL |
| 成功 ASL | 统计成功结点层次 |
| 失败 ASL | 统计失败结点比较次数 |

顺序查找胜在条件少，折半查找胜在比较次数少。
