<h1><center>第三节 树形查找</center></h1>

## 1. 树形查找

树形查找是指用树结构组织关键字，并沿树中路径进行查找。

408 中树形查找主要包括：

- 二叉排序树
- 平衡二叉树，也称 AVL 树
- 红黑树

二叉排序树又称二叉查找树，它满足：

- 若左子树非空，则左子树上所有结点关键字都小于根结点关键字。
- 若右子树非空，则右子树上所有结点关键字都大于根结点关键字。
- 左、右子树也分别是二叉排序树。

因此，二叉排序树的中序遍历序列是递增有序序列。

二叉排序树的查找过程类似折半查找：

1. 若查找关键字等于根结点关键字，则查找成功。
2. 若查找关键字小于根结点关键字，则进入左子树继续查找。
3. 若查找关键字大于根结点关键字，则进入右子树继续查找。
4. 若查找到空子树仍未找到，则查找失败。

查找效率取决于树高。

若树高为 $h$，则查找、插入、删除的时间复杂度均为：

$$
O(h)
$$

当二叉排序树比较平衡时：

$$
h \approx O(\log n)
$$

当插入序列接近有序时，二叉排序树可能退化成单支树，此时：

$$
h \approx O(n)
$$

所以普通二叉排序树不能保证最坏情况下的查找效率。

## 2. 平衡二叉树

平衡二叉树又称 AVL 树，是一种特殊的二叉排序树。

它首先满足二叉排序树的性质：

- 若左子树非空，则左子树上所有结点关键字都小于根结点关键字。
- 若右子树非空，则右子树上所有结点关键字都大于根结点关键字。
- 左、右子树也分别是二叉排序树。

在此基础上，AVL 树还要求每个结点的左子树和右子树高度差的绝对值不超过 `1`。

也就是说，对任意结点都有：

$$
|height(left) - height(right)| \le 1
$$

AVL 树通过在插入和删除后进行旋转调整，使树的高度始终保持在 $O(\log n)$ 级别，从而保证查找、插入、删除的时间复杂度均为 $O(\log n)$。

## 3. 平衡因子

平衡因子简称 BF，定义为：

$$
BF = 左子树高度 - 右子树高度
$$

在 AVL 树中，任意结点的平衡因子只能是：

- `-1`
- `0`
- `1`

若某个结点的平衡因子变为 `2` 或 `-2`，说明以该结点为根的子树失衡，需要通过旋转重新平衡。

## 4. 失衡类型

插入或删除结点后，可能出现四种失衡类型。

设最小失衡子树的根结点为 `A`。

### 4.1 LL 型

LL 型表示新结点插入在 `A` 的左孩子的左子树中。

失衡特征：

- `A` 的平衡因子为 `+2`。
- `A` 的左孩子 `B` 的平衡因子为 `+1` 或 `0`。

调整方法：对 `A` 做一次右旋。

```txt
        A                 B
       /                 / \
      B        =>       C   A
     /
    C
```

### 4.2 RR 型

RR 型表示新结点插入在 `A` 的右孩子的右子树中。

失衡特征：

- `A` 的平衡因子为 `-2`。
- `A` 的右孩子 `B` 的平衡因子为 `-1` 或 `0`。

调整方法：对 `A` 做一次左旋。

```txt
    A                     B
     \                   / \
      B       =>        A   C
       \
        C
```

### 4.3 LR 型

LR 型表示新结点插入在 `A` 的左孩子的右子树中。

失衡特征：

- `A` 的平衡因子为 `+2`。
- `A` 的左孩子 `B` 的平衡因子为 `-1`。

调整方法：

1. 先对 `A` 的左孩子 `B` 做左旋。
2. 再对 `A` 做右旋。

```txt
        A                 A                 C
       /                 /                 / \
      B        =>       C        =>       B   A
       \               /
        C             B
```

### 4.4 RL 型

RL 型表示新结点插入在 `A` 的右孩子的左子树中。

失衡特征：

- `A` 的平衡因子为 `-2`。
- `A` 的右孩子 `B` 的平衡因子为 `+1`。

调整方法：

1. 先对 `A` 的右孩子 `B` 做右旋。
2. 再对 `A` 做左旋。

```txt
    A                   A                     C
     \                   \                   / \
      B       =>          C       =>        A   B
     /                     \
    C                       B
```

## 5. 旋转操作

旋转的目标是恢复平衡，同时保持二叉排序树的中序序列不变。

也就是说，旋转前后，中序遍历结果相同，关键字仍然有序。

### 5.1 右旋

右旋用于修复 LL 型失衡。

设 `A` 为失衡结点，`B` 为 `A` 的左孩子：

```txt
        A                 B
       / \               / \
      B   Ar    =>      Bl  A
     / \                   / \
    Bl  Br                Br  Ar
```

旋转步骤：

1. `B` 上升为新的根。
2. `A` 下降为 `B` 的右孩子。
3. `B` 原来的右子树 `Br` 成为 `A` 的左子树。

### 5.2 左旋

左旋用于修复 RR 型失衡。

设 `A` 为失衡结点，`B` 为 `A` 的右孩子：

```txt
      A                     B
     / \                   / \
    Al  B        =>       A   Br
       / \               / \
      Bl  Br            Al  Bl
```

旋转步骤：

1. `B` 上升为新的根。
2. `A` 下降为 `B` 的左孩子。
3. `B` 原来的左子树 `Bl` 成为 `A` 的右子树。

## 6. 插入操作

AVL 树的插入过程可以分为两步：

1. 按二叉排序树的规则插入新结点。
2. 从插入位置向上检查祖先结点，找到最小失衡子树并旋转调整。

插入时，一旦调整好最小失衡子树，整棵树通常即可恢复平衡。

常见判断方法：

| 失衡类型 | 判断条件 | 调整方法 |
| --- | --- | --- |
| LL | `BF(A) = 2` 且 `BF(A->left) >= 0` | 右旋 |
| RR | `BF(A) = -2` 且 `BF(A->right) <= 0` | 左旋 |
| LR | `BF(A) = 2` 且 `BF(A->left) < 0` | 左旋左孩子，再右旋 |
| RL | `BF(A) = -2` 且 `BF(A->right) > 0` | 右旋右孩子，再左旋 |

## 7. 删除操作

AVL 树的删除操作也先按照二叉排序树的规则删除结点。

删除结点分三种情况：

- 被删结点是叶子结点：直接删除。
- 被删结点只有一棵子树：用其子树替代该结点。
- 被删结点有两棵子树：用中序前驱或中序后继替代，再删除前驱或后继结点。

删除后需要从删除位置向上检查祖先结点是否失衡。

与插入不同，删除后一次旋转可能会使更高层祖先继续失衡，因此删除后的调整可能需要一路向根结点回溯。

## 8. C 语言实现

下面代码采用“结点保存高度”的方式实现 AVL 树。

约定：

- 空树高度为 `0`。
- 叶子结点高度为 `1`。
- 平衡因子为左子树高度减右子树高度。

### 8.1 公共定义

定义逻辑：每个结点保存关键字、高度、左孩子指针和右孩子指针。

::: details 点击展开代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

typedef int KeyType;

typedef struct AVLNode {
    KeyType key;
    int height;
    struct AVLNode *left;
    struct AVLNode *right;
} AVLNode, *AVLTree;
```

:::

### 8.2 基本工具函数

实现逻辑：高度由左右子树高度较大者加一得到；平衡因子由左子树高度减右子树高度得到。

::: details 点击展开代码

```c
int Max(int a, int b) {
    return a > b ? a : b;
}

int Height(AVLTree T) {
    return T == NULL ? 0 : T->height;
}

void UpdateHeight(AVLTree T) {
    if (T != NULL) {
        T->height = Max(Height(T->left), Height(T->right)) + 1;
    }
}

int BalanceFactor(AVLTree T) {
    return T == NULL ? 0 : Height(T->left) - Height(T->right);
}

AVLNode *CreateNode(KeyType key) {
    AVLNode *node = (AVLNode *)malloc(sizeof(AVLNode));
    if (node == NULL) {
        return NULL;
    }

    node->key = key;
    node->height = 1;
    node->left = NULL;
    node->right = NULL;
    return node;
}
```

:::

### 8.3 右旋

实现逻辑：左孩子上升，原根结点下降为其右孩子，再更新高度。

::: details 点击展开代码

```c
AVLTree RightRotate(AVLTree A) {
    AVLTree B = A->left;
    AVLTree Br = B->right;

    B->right = A;
    A->left = Br;

    UpdateHeight(A);
    UpdateHeight(B);

    return B;
}
```

:::

### 8.4 左旋

实现逻辑：右孩子上升，原根结点下降为其左孩子，再更新高度。

::: details 点击展开代码

```c
AVLTree LeftRotate(AVLTree A) {
    AVLTree B = A->right;
    AVLTree Bl = B->left;

    B->left = A;
    A->right = Bl;

    UpdateHeight(A);
    UpdateHeight(B);

    return B;
}
```

:::

### 8.5 平衡调整

实现逻辑：先更新当前结点高度，再根据平衡因子判断四种失衡类型。

::: details 点击展开代码

```c
AVLTree Rebalance(AVLTree T) {
    if (T == NULL) {
        return NULL;
    }

    UpdateHeight(T);

    int bf = BalanceFactor(T);

    if (bf > 1) {
        if (BalanceFactor(T->left) < 0) {
            T->left = LeftRotate(T->left);
        }

        return RightRotate(T);
    }

    if (bf < -1) {
        if (BalanceFactor(T->right) > 0) {
            T->right = RightRotate(T->right);
        }

        return LeftRotate(T);
    }

    return T;
}
```

:::

### 8.6 查找

实现逻辑：AVL 树仍然是二叉排序树，因此查找过程与普通二叉排序树相同。

::: details 点击展开代码

```c
AVLNode *SearchAVL(AVLTree T, KeyType key) {
    while (T != NULL) {
        if (key == T->key) {
            return T;
        }

        if (key < T->key) {
            T = T->left;
        } else {
            T = T->right;
        }
    }

    return NULL;
}
```

:::

### 8.7 插入

实现逻辑：先递归执行二叉排序树插入，再在回溯过程中更新高度并调整平衡。

::: details 点击展开代码

```c
AVLTree InsertAVL(AVLTree T, KeyType key) {
    if (T == NULL) {
        return CreateNode(key);
    }

    if (key < T->key) {
        T->left = InsertAVL(T->left, key);
    } else if (key > T->key) {
        T->right = InsertAVL(T->right, key);
    } else {
        return T;
    }

    return Rebalance(T);
}
```

:::

### 8.8 删除

实现逻辑：先按二叉排序树规则删除结点，再在回溯过程中逐层调整平衡。

::: details 点击展开代码

```c
AVLNode *FindMin(AVLTree T) {
    while (T != NULL && T->left != NULL) {
        T = T->left;
    }

    return T;
}

AVLTree DeleteAVL(AVLTree T, KeyType key) {
    if (T == NULL) {
        return NULL;
    }

    if (key < T->key) {
        T->left = DeleteAVL(T->left, key);
    } else if (key > T->key) {
        T->right = DeleteAVL(T->right, key);
    } else {
        if (T->left == NULL || T->right == NULL) {
            AVLNode *child = T->left != NULL ? T->left : T->right;

            free(T);
            return child;
        }

        AVLNode *successor = FindMin(T->right);
        T->key = successor->key;
        T->right = DeleteAVL(T->right, successor->key);
    }

    return Rebalance(T);
}
```

:::

### 8.9 中序遍历与销毁

实现逻辑：中序遍历 AVL 树可以得到递增序列；销毁时采用后序方式释放结点。

::: details 点击展开代码

```c
void InOrder(AVLTree T) {
    if (T == NULL) {
        return;
    }

    InOrder(T->left);
    printf("%d ", T->key);
    InOrder(T->right);
}

void DestroyAVL(AVLTree *T) {
    if (*T == NULL) {
        return;
    }

    DestroyAVL(&(*T)->left);
    DestroyAVL(&(*T)->right);
    free(*T);
    *T = NULL;
}
```

## 9. 408 常考点

### 9.1 AVL 树一定是二叉排序树

AVL 树是在二叉排序树的基础上增加平衡条件得到的。

所以 AVL 树必须同时满足：

- 二叉排序树的有序性。
- 每个结点左右子树高度差不超过 `1`。

只平衡但不中序有序的二叉树不是 AVL 树。

### 9.2 旋转不改变中序序列

AVL 树调整时虽然改变了局部结构，但不会改变中序遍历序列。

这是因为旋转本质上是在保持关键字大小关系的前提下改变父子关系。

因此旋转后仍然是二叉排序树。

### 9.3 最小失衡子树

插入结点后，从新结点向根结点回溯，第一个平衡因子绝对值大于 `1` 的结点，就是最小失衡子树的根。

408 手算题通常要求：

1. 先按二叉排序树插入。
2. 找到最小失衡子树。
3. 判断属于 LL、RR、LR、RL 哪一类。
4. 进行对应旋转。

### 9.4 插入与删除调整次数

插入操作：

- 通常只需要调整最小失衡子树。
- 一次单旋或一次双旋即可恢复整棵树平衡。

删除操作：

- 可能导致祖先结点继续失衡。
- 需要从删除位置一路向根结点回溯检查。
- 可能进行多次调整。

### 9.5 平衡因子的正负含义

本页采用：

$$
BF = 左子树高度 - 右子树高度
$$

因此：

- `BF > 0` 表示左子树更高。
- `BF < 0` 表示右子树更高。
- `BF = 0` 表示左右子树等高。

有些教材或题目可能采用相反定义，做题时要以题目说明为准。

### 9.6 AVL 树高度

AVL 树的高度为 $O(\log n)$。

若高度为 `h` 的 AVL 树最少结点数为 $N_h$，则有：

$$
N_h = N_{h - 1} + N_{h - 2} + 1
$$

因为要在满足平衡条件的前提下让结点数最少，一棵子树高度为 `h - 1`，另一棵子树高度为 `h - 2`。

这说明 AVL 树高度不会退化为 $O(n)$。

## 10. 红黑树

红黑树是一种近似平衡的二叉排序树。

它不像 AVL 树那样严格要求任意结点左右子树高度差不超过 $1$，而是通过给结点染成红色或黑色，并约束颜色分布，使树高保持在 $O(\log n)$。

红黑树首先是一棵二叉排序树，因此中序遍历序列仍然是递增有序序列。

### 10.1 红黑树的性质

红黑树满足以下性质：

1. 每个结点不是红色就是黑色。
2. 根结点是黑色。
3. 叶结点，也就是空结点 `NIL`，是黑色。
4. 若一个结点是红色，则它的两个孩子都是黑色。
5. 对任一结点，从该结点到其所有后代叶结点的简单路径上，黑色结点数相同。

性质 4 说明红黑树中不能出现连续两个红色结点。

性质 5 保证从同一结点出发到各个叶结点的黑色结点数量一致，这是红黑树保持近似平衡的关键。

### 10.2 黑高

从某结点出发，到达任一后代叶结点路径上的黑色结点数称为该结点的黑高。

通常不包括该结点本身，但包括叶结点 `NIL`。不同教材表述可能略有差异，做题时以题目定义为准。

红黑树要求同一结点到所有后代叶结点路径的黑高相同。

例如某结点的黑高为 $bh$，说明从该结点到任意后代空叶结点路径上，黑色结点数量都相等。

### 10.3 红黑树高度

红黑树不要求严格平衡，但能保证最长路径不会超过最短路径的两倍。

原因是：

- 红色结点不能连续出现。
- 每条路径上的黑色结点数相同。

因此红黑树高度满足：

$$
h \le 2\log_2(n + 1)
$$

所以查找、插入、删除的最坏时间复杂度均为：

$$
O(\log n)
$$

### 10.4 红黑树插入

红黑树插入新结点时，先按二叉排序树规则插入。

新插入结点通常染成红色。

原因是插入红色结点不会破坏黑高；若插入黑色结点，可能导致某些路径黑色结点数增加，调整更复杂。

插入后主要检查是否出现“红红冲突”，也就是父结点和新结点都是红色。

#### 10.4.1 父结点是黑色

若新结点的父结点是黑色，则红黑树性质没有被破坏，插入结束。

#### 10.4.2 父结点和叔结点都是红色

若父结点是红色，叔结点也是红色，则通常进行变色：

1. 父结点染黑。
2. 叔结点染黑。
3. 祖父结点染红。
4. 将祖父结点作为新的检查结点继续向上调整。

最后要保证根结点为黑色。

#### 10.4.3 父结点红色，叔结点黑色

若父结点是红色，叔结点是黑色，则需要旋转并变色。

常见情况和 AVL 类似，也可分成：

| 类型 | 形态 | 调整 |
| --- | --- | --- |
| LL | 新结点在祖父结点左孩子的左子树 | 右旋 |
| RR | 新结点在祖父结点右孩子的右子树 | 左旋 |
| LR | 新结点在祖父结点左孩子的右子树 | 先左旋父结点，再右旋祖父结点 |
| RL | 新结点在祖父结点右孩子的左子树 | 先右旋父结点，再左旋祖父结点 |

旋转后通常让新的子树根变黑，原祖父结点变红。

### 10.5 红黑树删除

红黑树删除也先按二叉排序树的删除规则处理。

删除后若破坏黑高或出现颜色冲突，需要通过变色和旋转调整。

408 中红黑树删除通常不作为重点复杂手算内容，更常见的是考红黑树性质、插入调整、树高界和与 AVL 树的比较。

### 10.6 红黑树与 AVL 树比较

| 项目 | AVL 树 | 红黑树 |
| --- | --- | --- |
| 平衡要求 | 严格平衡 | 近似平衡 |
| 高度 | 通常更低 | 可能略高 |
| 查找效率 | 通常略好 | 稳定为 $O(\log n)$ |
| 插入删除调整 | 可能较频繁 | 旋转次数通常较少 |
| 适合场景 | 查找更频繁 | 插入删除更频繁 |

AVL 树追求更严格的高度平衡。

红黑树在查找效率和调整代价之间折中，因此常用于工程中的动态查找结构。

### 10.7 408 常考点补充

#### 10.7.1 红黑树一定是二叉排序树

红黑树首先要满足二叉排序树的有序性。

只满足颜色性质但不满足左小右大的树，不是红黑树。

#### 10.7.2 红色结点不能连续

若某结点是红色，则它的父结点和孩子结点都必须是黑色。

这是判断红黑树是否合法时最容易检查的一条。

#### 10.7.3 每条路径黑高相同

从任一结点到其所有后代 `NIL` 结点的路径上，黑色结点数必须相同。

判断红黑树是否合法时，不能只看局部颜色，还要检查黑高。

#### 10.7.4 插入结点初始为红色

红黑树插入新结点通常先染红。

若父结点也是红色，再根据叔结点颜色进行变色或旋转。

## 11. 小结

| 操作 | 平均时间复杂度 | 最坏时间复杂度 |
| --- | --- | --- |
| 查找 | $O(\log n)$ | $O(\log n)$ |
| 插入 | $O(\log n)$ | $O(\log n)$ |
| 删除 | $O(\log n)$ | $O(\log n)$ |

AVL 树适合查找频繁、需要保持较好平衡的场景。

需要重点掌握：

- 平衡因子的定义。
- LL、RR、LR、RL 四种失衡类型。
- 单旋和双旋的过程。
- 插入后最小失衡子树的调整。
- 删除后可能需要向上连续调整。
- 红黑树的五条性质。
- 红黑树插入时的变色和旋转。
- AVL 树与红黑树的区别。
