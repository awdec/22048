<h1><center>第二节 插入排序</center></h1>

## 1. 插入排序

插入排序的基本思想是：每次将一个待排序记录插入到前面已经有序的子序列中。

插入排序包括：

- 直接插入排序
- 折半插入排序
- 希尔排序

## 2. 直接插入排序

直接插入排序将序列分成两部分：

- 前面是有序区。
- 后面是无序区。

每一趟从无序区取出第一个元素，插入到有序区的合适位置。

### 2.1 算法过程

对序列：

```txt
49 38 65 97 76 13
```

直接插入排序过程可以理解为：

```txt
[49] 38 65 97 76 13
[38 49] 65 97 76 13
[38 49 65] 97 76 13
[38 49 65 97] 76 13
[38 49 65 76 97] 13
[13 38 49 65 76 97]
```

每一趟都会让有序区长度增加 $1$。

### 2.2 代码实现

::: details 点击展开代码

```c
void InsertSort(int A[], int n) {
    for (int i = 1; i < n; i++) {
        if (A[i] < A[i - 1]) {
            int temp = A[i];
            int j;

            for (j = i - 1; j >= 0 && A[j] > temp; j--) {
                A[j + 1] = A[j];
            }

            A[j + 1] = temp;
        }
    }
}
```

:::

### 2.3 性能分析

| 情况 | 时间复杂度 |
| --- | --- |
| 最好 | $O(n)$ |
| 平均 | $O(n^2)$ |
| 最坏 | $O(n^2)$ |

空间复杂度：

$$
O(1)
$$

稳定性：稳定。

直接插入排序适合基本有序的序列。

## 3. 折半插入排序

折半插入排序是在直接插入排序基础上的改进。

它使用折半查找在有序区中寻找插入位置。

注意：折半插入排序只减少关键字比较次数，不减少元素移动次数。

### 3.1 算法过程

每趟排序：

1. 在前面有序区中用折半查找确定插入位置。
2. 将插入位置之后的元素后移。
3. 把待插入元素放入空出的位置。

### 3.2 性能分析

折半插入排序时间复杂度仍为：

$$
O(n^2)
$$

原因是移动元素的次数仍可能达到 $O(n^2)$。

空间复杂度：

$$
O(1)
$$

稳定性：稳定。

## 4. 希尔排序

希尔排序又称缩小增量排序。

它先将整个序列按某个增量分成若干子序列，分别进行直接插入排序；然后逐步缩小增量，最后当增量为 $1$ 时，再进行一次直接插入排序。

### 4.1 基本思想

若增量为 $d$，则把相隔 $d$ 的元素分为一组。

例如序列下标为：

```txt
0 1 2 3 4 5 6 7
```

当 $d = 4$ 时，分组为：

```txt
0,4
1,5
2,6
3,7
```

每组内部做直接插入排序。

### 4.2 性能分析

希尔排序时间复杂度与增量序列有关。

408 中通常记住：

- 希尔排序比直接插入排序快。
- 希尔排序空间复杂度为 $O(1)$。
- 希尔排序不稳定。

希尔排序不稳定的原因是相同关键字可能被分到不同子序列中，并在分组插入过程中改变相对次序。

## 5. 408 常考点

### 5.1 直接插入排序稳定

直接插入排序中，遇到相等关键字时不越过已有相等元素，因此稳定。

### 5.2 折半插入排序不减少移动次数

折半查找只优化插入位置的查找过程。

元素后移仍然可能很多。

### 5.3 希尔排序不稳定

希尔排序是插入排序中唯一常见的不稳定排序。

### 5.4 基本有序时直接插入排序效率高

若序列已经基本有序，直接插入排序移动次数少，效率较高。

## 6. 小结

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
| --- | --- | --- | --- |
| 直接插入排序 | $O(n^2)$ | $O(1)$ | 稳定 |
| 折半插入排序 | $O(n^2)$ | $O(1)$ | 稳定 |
| 希尔排序 | 与增量序列有关 | $O(1)$ | 不稳定 |

插入排序的核心是：前面有序，后面逐个插入。
