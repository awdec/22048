<h1><center>第三节 交换排序</center></h1>

## 1. 交换排序

交换排序的基本思想是：根据关键字比较结果交换记录位置。

408 中常见交换排序：

- 冒泡排序
- 快速排序

## 2. 冒泡排序

冒泡排序每趟通过相邻元素比较和交换，将一个最大或最小元素放到最终位置。

若按升序排列，每一趟通常把当前无序区中的最大元素交换到最后。

### 2.1 算法过程

对序列：

```txt
49 38 65 97 76 13
```

第一趟从前向后比较相邻元素：

```txt
38 49 65 76 13 97
```

最大元素 $97$ 已经到达最终位置。

之后继续对前面的无序区进行同样操作。

### 2.2 代码实现

::: details 点击展开代码

```c
void BubbleSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;

        for (int j = 0; j < n - i - 1; j++) {
            if (A[j] > A[j + 1]) {
                int temp = A[j];
                A[j] = A[j + 1];
                A[j + 1] = temp;
                flag = true;
            }
        }

        if (!flag) {
            return;
        }
    }
}
```

:::

### 2.3 性能分析

若使用 `flag` 判断一趟中是否发生交换，则最好情况为：

$$
O(n)
$$

平均和最坏情况为：

$$
O(n^2)
$$

空间复杂度：

$$
O(1)
$$

稳定性：稳定。

## 3. 快速排序

快速排序的基本思想是分治。

每一趟排序选择一个枢轴，将序列划分为两部分：

- 左边元素不大于枢轴。
- 右边元素不小于枢轴。

枢轴在这一趟后到达最终位置。

然后递归处理左右子表。

### 3.1 一趟划分

以第一个元素作为枢轴 `pivot`。

划分结束后，枢轴左侧元素都不大于它，右侧元素都不小于它。

注意：枢轴左右两边内部不一定有序。

### 3.2 代码实现

::: details 点击展开代码

```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];

    while (low < high) {
        while (low < high && A[high] >= pivot) {
            high--;
        }
        A[low] = A[high];

        while (low < high && A[low] <= pivot) {
            low++;
        }
        A[high] = A[low];
    }

    A[low] = pivot;
    return low;
}

void QuickSort(int A[], int low, int high) {
    if (low < high) {
        int pivotPos = Partition(A, low, high);
        QuickSort(A, low, pivotPos - 1);
        QuickSort(A, pivotPos + 1, high);
    }
}
```

:::

### 3.3 性能分析

快速排序平均时间复杂度为：

$$
O(n\log n)
$$

最坏时间复杂度为：

$$
O(n^2)
$$

当每次划分都极不均衡时，快速排序会退化。

平均空间复杂度来自递归栈：

$$
O(\log n)
$$

最坏空间复杂度为：

$$
O(n)
$$

稳定性：不稳定。

## 4. 冒泡排序与快速排序比较

| 项目 | 冒泡排序 | 快速排序 |
| --- | --- | --- |
| 基本思想 | 相邻交换 | 一趟划分 |
| 平均时间 | $O(n^2)$ | $O(n\log n)$ |
| 最坏时间 | $O(n^2)$ | $O(n^2)$ |
| 空间复杂度 | $O(1)$ | 平均 $O(\log n)$ |
| 稳定性 | 稳定 | 不稳定 |

## 5. 408 常考点

### 5.1 冒泡排序一趟确定一个最终位置

升序冒泡时，每趟通常把无序区最大元素放到最后。

### 5.2 快速排序一趟只确定枢轴

快速排序一趟划分后，只有枢轴到达最终位置。

左右子表内部不一定有序。

### 5.3 快速排序不稳定

快速排序划分和交换可能改变相等关键字的相对次序。

### 5.4 快速排序最坏情况

若枢轴选择不当，使每次划分都极不均衡，快速排序退化为 $O(n^2)$。

## 6. 小结

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
| --- | --- | --- | --- |
| 冒泡排序 | $O(n^2)$ | $O(1)$ | 稳定 |
| 快速排序 | 平均 $O(n\log n)$ | 平均 $O(\log n)$ | 不稳定 |

交换排序的核心是：通过交换让元素逐步靠近最终位置。
