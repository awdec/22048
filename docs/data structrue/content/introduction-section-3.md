<h1><center>第三节 本章总结</center></h1>

## 1. 本章核心

绪论主要建立数据结构的基本框架：

$$
\text{数据}
\longrightarrow
\text{数据结构}
\longrightarrow
\text{算法}
\longrightarrow
\text{复杂度分析}
$$

408 中学习数据结构，不只是背概念，更重要的是理解：

1. 数据之间有什么逻辑关系。
2. 这种关系在计算机中如何存储。
3. 对数据执行哪些基本运算。
4. 算法执行这些运算需要多少时间和空间。

::: tip 重点
数据结构解决“数据如何组织”，算法解决“问题如何处理”，复杂度解决“处理代价有多大”。
:::

## 2. 数据结构的三要素

数据结构由三个方面组成：

| 要素 | 含义 |
| --- | --- |
| 逻辑结构 | 数据元素之间的逻辑关系 |
| 存储结构 | 逻辑关系在计算机中的存储表示 |
| 数据运算 | 对数据可以执行的操作 |

三者关系：

$$
\text{逻辑结构决定抽象关系}
\quad
\text{存储结构决定实现方式}
\quad
\text{数据运算决定操作接口}
$$

例如线性表：

- 逻辑结构：一对一关系。
- 存储结构：顺序存储或链式存储。
- 数据运算：插入、删除、查找、遍历等。

## 3. 四类逻辑结构

| 逻辑结构 | 关系特点 | 典型例子 |
| --- | --- | --- |
| 集合结构 | 元素之间除同属一个集合外无其他关系 | 集合 |
| 线性结构 | 一对一关系 | 线性表、栈、队列、串 |
| 树形结构 | 一对多关系 | 树、二叉树、堆 |
| 图状结构 | 多对多关系 | 图、网 |

判断逻辑结构时，要看数据元素之间的关系，而不是看存储方式。

::: warning 易错
顺序存储不一定对应线性结构，链式存储也不一定只用于线性结构。逻辑结构和存储结构是两个层面的概念。
:::

## 4. 存储结构总结

常见存储结构：

| 存储结构 | 特点 |
| --- | --- |
| 顺序存储 | 地址连续，可随机访问 |
| 链式存储 | 用指针表示关系，便于插入删除 |
| 索引存储 | 建立索引表，提高查找效率 |
| 散列存储 | 根据关键字计算存储位置 |

存储结构会直接影响算法效率。

例如：

- 顺序表按位查找快，但插入删除慢。
- 链表插入删除方便，但按位查找慢。

## 5. 算法的五个特性

算法通常具有五个特性：

| 特性 | 含义 |
| --- | --- |
| 有穷性 | 有限步后结束 |
| 确定性 | 每一步含义明确 |
| 可行性 | 每一步可以执行 |
| 输入 | 有零个或多个输入 |
| 输出 | 有一个或多个输出 |

::: tip 记忆
算法可以没有输入，但必须有输出。
:::

## 6. 好算法的标准

评价一个算法，通常看：

1. 正确性。
2. 可读性。
3. 健壮性。
4. 时间效率。
5. 空间效率。

408 更常考的是复杂度分析：

$$
\text{时间复杂度}
\qquad
\text{空间复杂度}
$$

## 7. 时间复杂度

时间复杂度描述算法运行时间随问题规模 $n$ 增长的变化趋势。

记作：

$$
O(f(n))
$$

常见复杂度从低到高：

$$
O(1)
<
O(\log n)
<
O(n)
<
O(n\log n)
<
O(n^2)
<
O(n^3)
<
O(2^n)
<
O(n!)
$$

分析时通常只保留最高阶项，并忽略常数系数。

例如：

$$
T(n)=3n^2+2n+100
$$

则：

$$
T(n)=O(n^2)
$$

## 8. 空间复杂度

空间复杂度描述算法额外占用的存储空间随问题规模 $n$ 增长的变化趋势。

记作：

$$
S(n)=O(f(n))
$$

常见情况：

- 只使用有限个变量：$O(1)$。
- 使用长度为 $n$ 的辅助数组：$O(n)$。
- 使用递归调用栈：看递归深度。

::: warning 易错
空间复杂度通常关注“额外空间”，输入数据本身一般不计入辅助空间。
:::

## 9. 解题方法总结

### 判断概念题

概念题先抓关键词：

| 关键词 | 对应概念 |
| --- | --- |
| 数据的基本单位 | 数据元素 |
| 最小单位 | 数据项 |
| 相同性质的数据元素集合 | 数据对象 |
| 数据元素之间的关系 | 数据结构 |
| 数据之间的逻辑关系 | 逻辑结构 |
| 内存中的表示 | 存储结构 |

### 判断复杂度题

复杂度题按下面步骤：

1. 找基本操作。
2. 看基本操作执行次数与 $n$ 的关系。
3. 保留最高阶项。
4. 去掉常数系数。

常见代码结构：

| 代码结构 | 常见复杂度 |
| --- | --- |
| 单层循环 $n$ 次 | $O(n)$ |
| 双层嵌套循环 $n^2$ 次 | $O(n^2)$ |
| 每次规模减半 | $O(\log n)$ |
| 外层 $n$，内层倍增 | $O(n\log n)$ |

## 10. 易错点总结

| 易错点 | 正确理解 |
| --- | --- |
| 把数据元素和数据项混淆 | 数据元素由若干数据项组成 |
| 把逻辑结构和存储结构混淆 | 一个是抽象关系，一个是实现方式 |
| 认为顺序存储只能存线性结构 | 存储方式与逻辑结构不是一一对应 |
| 复杂度保留低阶项 | 只保留最高阶增长趋势 |
| 忽略递归栈空间 | 递归深度会影响空间复杂度 |
| 认为算法必须有输入 | 算法可以有零个输入，但必须有输出 |

## 11. 考前背诵版

1. 数据元素是数据的基本单位，数据项是最小单位。
2. 数据结构包括逻辑结构、存储结构和数据运算。
3. 逻辑结构分为集合、线性、树形、图状。
4. 存储结构常见有顺序、链式、索引、散列。
5. 算法具有有穷性、确定性、可行性、输入、输出。
6. 好算法要求正确、可读、健壮、高效、低存储。
7. 时间复杂度看运行时间随 $n$ 的增长趋势。
8. 空间复杂度看额外空间随 $n$ 的增长趋势。
9. 复杂度分析只保留最高阶项，忽略常数系数。
