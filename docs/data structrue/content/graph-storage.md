<h1><center>第二节 图结构的存储</center></h1>

## 1. 图存储的核心问题

图的存储要解决两个问题：

1. 如何保存顶点信息。
2. 如何保存顶点之间的边或弧。

408 数据结构中常考的图存储结构包括：

- 邻接矩阵
- 邻接表
- 十字链表
- 邻接多重表

不同存储结构会直接影响图算法的时间复杂度。

## 2. 邻接矩阵

邻接矩阵用一个二维数组表示顶点之间是否存在边。

设图有 $n$ 个顶点，则邻接矩阵是一个 $n \times n$ 的矩阵。

### 2.1 无权图的邻接矩阵

对于无权图，邻接矩阵可以定义为：

$$
A[i][j] =
\begin{cases}
1, & (v_i, v_j) \text{ 存在边或弧}\\
0, & (v_i, v_j) \text{ 不存在边或弧}
\end{cases}
$$

无向图的邻接矩阵一定是对称矩阵。

若无向图中没有自环，则主对角线元素均为 $0$。

### 2.2 带权图的邻接矩阵

对于带权图，邻接矩阵可以定义为：

$$
A[i][j] =
\begin{cases}
w_{ij}, & (v_i, v_j) \text{ 存在边或弧}\\
\infty, & (v_i, v_j) \text{ 不存在边或弧}
\end{cases}
$$

其中 $w_{ij}$ 表示边或弧的权值。

通常令：

$$
A[i][i] = 0
$$

### 2.3 邻接矩阵的特点

邻接矩阵的空间复杂度为：

$$
O(n^2)
$$

优点：

- 判断两个顶点之间是否有边很快。
- 适合稠密图。
- 便于实现 Floyd、Prim 等算法。

缺点：

- 对稀疏图会浪费空间。
- 查找某顶点的所有邻接点需要扫描一整行。

### 2.4 邻接矩阵定义

::: details 点击展开代码

```c
#define MaxVertexNum 100
#define INFINITY 65535

typedef char VertexType;
typedef int EdgeType;

typedef struct {
    VertexType vex[MaxVertexNum];
    EdgeType edge[MaxVertexNum][MaxVertexNum];
    int vexnum;
    int arcnum;
} MGraph;
```

:::

其中：

- `vex` 保存顶点信息。
- `edge` 保存边或弧的信息。
- `vexnum` 表示顶点数。
- `arcnum` 表示边数或弧数。

## 3. 邻接表

邻接表是顺序存储和链式存储相结合的结构。

它为每个顶点建立一个单链表，链表中保存该顶点的邻接点。

邻接表由两部分组成：

- 顶点表
- 边表

### 3.1 邻接表结构

顶点表中的每个结点通常包含：

- 顶点数据
- 指向第一条边结点的指针

边表中的每个结点通常包含：

- 邻接点下标
- 指向下一条边结点的指针
- 边权值，可选

::: details 点击展开代码

```c
#define MaxVertexNum 100

typedef char VertexType;
typedef int EdgeType;

typedef struct ArcNode {
    int adjvex;
    struct ArcNode *next;
    EdgeType weight;
} ArcNode;

typedef struct VNode {
    VertexType data;
    ArcNode *first;
} VNode, AdjList[MaxVertexNum];

typedef struct {
    AdjList vertices;
    int vexnum;
    int arcnum;
} ALGraph;
```

:::

### 3.2 无向图的邻接表

无向图中，一条边 $(u,v)$ 会在两个顶点的边表中出现：

- 在 $u$ 的边表中出现 $v$。
- 在 $v$ 的边表中出现 $u$。

因此无向图有 $e$ 条边时，邻接表中共有：

$$
2e
$$

个边表结点。

### 3.3 有向图的邻接表

有向图中，一条弧 $<u,v>$ 只在 $u$ 的出边表中出现一次。

因此有向图有 $e$ 条弧时，邻接表中共有：

$$
e
$$

个边表结点。

若需要快速查找入边，可以建立逆邻接表。

### 3.4 邻接表的特点

邻接表的空间复杂度为：

$$
O(n + e)
$$

优点：

- 适合稀疏图。
- 查找某顶点的所有邻接点方便。
- DFS 和 BFS 使用邻接表时时间复杂度为 $O(n + e)$。

缺点：

- 判断两个顶点之间是否有边，需要扫描对应链表。
- 有向图中查找某顶点入边不方便，除非建立逆邻接表。

## 4. 十字链表

十字链表主要用于有向图。

它把邻接表和逆邻接表结合起来，既方便查找出边，也方便查找入边。

### 4.1 顶点结点

顶点结点通常包含：

- 顶点数据
- 指向第一条入边的指针
- 指向第一条出边的指针

### 4.2 弧结点

弧结点通常包含：

- 弧尾顶点下标
- 弧头顶点下标
- 指向同弧尾下一条弧的指针
- 指向同弧头下一条弧的指针
- 权值或其他信息

十字链表中每条弧只存储一次。

空间复杂度为：

$$
O(n + e)
$$

## 5. 邻接多重表

邻接多重表主要用于无向图。

在普通邻接表中，无向图的一条边会存储两次。

邻接多重表让每条无向边只用一个边结点表示，同时这个边结点可以同时挂到两个端点的边链表中。

### 5.1 顶点结点

顶点结点通常包含：

- 顶点数据
- 指向第一条依附于该顶点的边的指针

### 5.2 边结点

边结点通常包含：

- 边依附的两个顶点下标
- 分别指向依附于这两个顶点的下一条边的指针
- 标记域
- 权值或其他信息

邻接多重表适合需要频繁删除边或标记边的无向图。

## 6. 存储结构比较

| 存储结构 | 适用图 | 空间复杂度 | 主要优点 | 主要缺点 |
| --- | --- | --- | --- | --- |
| 邻接矩阵 | 稠密图 | $O(n^2)$ | 判断边是否存在快 | 稀疏图浪费空间 |
| 邻接表 | 稀疏图 | $O(n + e)$ | 枚举邻接点方便 | 判断两点是否相邻较慢 |
| 十字链表 | 有向图 | $O(n + e)$ | 入边和出边都方便 | 结构较复杂 |
| 邻接多重表 | 无向图 | $O(n + e)$ | 每条边只存一次 | 结构较复杂 |

## 7. 常见操作复杂度

| 操作 | 邻接矩阵 | 邻接表 |
| --- | --- | --- |
| 判断 $v_i$ 与 $v_j$ 是否相邻 | $O(1)$ | 最坏 $O(n)$ 或 $O(e)$ |
| 求某顶点第一个邻接点 | 最坏 $O(n)$ | 通常 $O(1)$ |
| 遍历某顶点所有邻接点 | $O(n)$ | 与该顶点度数有关 |
| 遍历整个图 | $O(n^2)$ | $O(n + e)$ |

做 408 题时，看到复杂度题要先判断图采用哪种存储结构。

## 8. 408 常考点

### 8.1 无向图邻接矩阵对称

无向图的邻接矩阵一定对称。

如果无自环，主对角线为 $0$。

### 8.2 邻接表边结点数量

无向图有 $e$ 条边时，邻接表中有 $2e$ 个边表结点。

有向图有 $e$ 条弧时，邻接表中有 $e$ 个边表结点。

### 8.3 邻接矩阵适合稠密图

邻接矩阵空间固定为 $O(n^2)$。

即使边很少，也要开一个 $n \times n$ 的矩阵。

### 8.4 邻接表适合稀疏图

邻接表只保存实际存在的边或弧。

当 $e$ 远小于 $n^2$ 时，邻接表通常更节省空间。

### 8.5 有向图求入度

若使用普通邻接表存储有向图：

- 求出度方便。
- 求入度需要扫描所有边表。

若使用逆邻接表：

- 求入度方便。
- 求出度不方便。

若使用十字链表：

- 求入度和出度都比较方便。

## 9. 小结

| 内容 | 重点 |
| --- | --- |
| 邻接矩阵 | 二维数组，适合稠密图 |
| 邻接表 | 顶点表加边表，适合稀疏图 |
| 十字链表 | 有向图，兼顾入边和出边 |
| 邻接多重表 | 无向图，每条边只存一次 |
| 复杂度 | 存储结构决定遍历和查边效率 |

图结构的存储要抓住一句话：矩阵查边快，链表省空间。
