<h1><center>第二节 并查集</center></h1>

## 1. 并查集

并查集又称不相交集合，英文常写作 Disjoint Set Union，简称 DSU，也叫 Union-Find。

它用来维护若干个互不相交的集合，主要支持两类操作：

- 查找：判断某个元素属于哪个集合。
- 合并：把两个元素所在的集合合并成一个集合。

并查集只关心“两个元素是否属于同一个集合”，不关心集合中元素的线性顺序。

在 408 数据结构中，并查集常和下面内容联系：

- 集合的等价关系
- 森林和双亲表示法
- 无向图连通分量
- Kruskal 最小生成树

## 2. 基本存储结构

并查集通常用数组保存森林。

每个集合可以看作一棵树，树根代表整个集合。

设数组为 `parent`：

- `parent[x] = x`：说明 `x` 是根结点。
- `parent[x] = y`：说明 `x` 的双亲是 `y`。

例如：

```txt
parent[1] = 1
parent[2] = 1
parent[3] = 1
parent[4] = 4
parent[5] = 4
```

表示：

- $1,2,3$ 属于同一个集合，根为 $1$。
- $4,5$ 属于同一个集合，根为 $4$。

判断两个元素是否属于同一集合，只需要比较它们的根是否相同。

## 3. 初始化

初始时，每个元素单独构成一个集合。

因此每个元素都是自己的根：

$$
parent[i] = i
$$

::: details 点击展开代码

```c
#define MaxSize 100

int parent[MaxSize];

void InitSet(int n) {
    for (int i = 0; i < n; i++) {
        parent[i] = i;
    }
}
```

:::

## 4. 查找操作

查找操作用于找到元素所在集合的代表元，也就是树根。

基本思想：不断沿着双亲指针向上走，直到找到根结点。

::: details 点击展开代码

```c
int Find(int x) {
    while (parent[x] != x) {
        x = parent[x];
    }

    return x;
}
```

:::

若：

$$
Find(x) = Find(y)
$$

则 $x$ 和 $y$ 属于同一个集合。

## 5. 合并操作

合并操作用于把两个元素所在的集合合并。

步骤：

1. 分别查找两个元素所在集合的根。
2. 若根相同，说明已经在同一集合中。
3. 若根不同，把其中一棵树的根挂到另一棵树的根下面。

::: details 点击展开代码

```c
void Union(int x, int y) {
    int rootX = Find(x);
    int rootY = Find(y);

    if (rootX != rootY) {
        parent[rootY] = rootX;
    }
}
```

:::

合并后，两个集合就变成一个集合。

## 6. 路径压缩

普通查找可能形成很高的树。

例如依次合并后可能出现：

```txt
1 <- 2 <- 3 <- 4 <- 5
```

此时查找 $5$ 需要一路向上找到 $1$，效率较低。

路径压缩的思想是：在查找根的过程中，把路径上的结点直接挂到根结点下面。

压缩后：

```txt
    1
 / / \ \
2 3  4 5
```

这样后续查找会更快。

::: details 点击展开代码

```c
int Find(int x) {
    if (parent[x] != x) {
        parent[x] = Find(parent[x]);
    }

    return parent[x];
}
```

:::

路径压缩不会改变集合划分，只会改变树的形态。

## 7. 按秩合并

为了避免树变高，合并时可以让较矮的树挂到较高的树下面。

常见做法有两种：

- 按高度合并：维护每棵树的高度。
- 按规模合并：维护每个集合的结点数。

下面以按规模合并为例。

::: details 点击展开代码

```c
int parent[MaxSize];
int size[MaxSize];

void InitSet(int n) {
    for (int i = 0; i < n; i++) {
        parent[i] = i;
        size[i] = 1;
    }
}

int Find(int x) {
    if (parent[x] != x) {
        parent[x] = Find(parent[x]);
    }

    return parent[x];
}

void Union(int x, int y) {
    int rootX = Find(x);
    int rootY = Find(y);

    if (rootX == rootY) {
        return;
    }

    if (size[rootX] < size[rootY]) {
        parent[rootX] = rootY;
        size[rootY] += size[rootX];
    } else {
        parent[rootY] = rootX;
        size[rootX] += size[rootY];
    }
}
```

:::

路径压缩和按秩合并通常配合使用。

## 8. 时间复杂度

若不做优化，查找操作在最坏情况下可能退化为：

$$
O(n)
$$

采用路径压缩和按秩合并后，单次操作的均摊时间复杂度可以认为接近常数。

408 复习中通常记住：

$$
O(\alpha(n))
$$

其中 $\alpha(n)$ 是反阿克曼函数，增长极慢，实际应用中可近似看成常数。

## 9. 并查集与等价关系

等价关系满足：

- 自反性
- 对称性
- 传递性

并查集适合维护等价类。

若已知若干关系：

```txt
(a, b), (b, c), (d, e)
```

则可以依次执行：

```txt
Union(a, b)
Union(b, c)
Union(d, e)
```

最终得到两个等价类：

- $\{a,b,c\}$
- $\{d,e\}$

## 10. 并查集与图

在无向图中，并查集常用于维护连通分量。

处理每条边 $(u,v)$：

1. 若 $Find(u) \ne Find(v)$，说明 $u$ 和 $v$ 原本不连通，可以合并。
2. 若 $Find(u) = Find(v)$，说明 $u$ 和 $v$ 已经连通，这条边会形成环。

这正是 Kruskal 算法判断是否加入边的核心。

Kruskal 算法过程：

1. 将边按权值从小到大排序。
2. 依次检查每条边。
3. 若边的两个端点属于不同集合，则加入生成树并合并集合。
4. 若两个端点已在同一集合，则舍弃该边。
5. 直到选出 $n - 1$ 条边。

## 11. 408 常考点

### 11.1 并查集本质是森林

并查集不是线性结构，而是用森林表示若干集合。

每个集合对应一棵树，根结点是集合代表元。

### 11.2 Find 找的是根

`Find(x)` 的结果不是 $x$ 的直接双亲，而是 $x$ 所在集合的根。

判断两个元素是否属于同一集合，比较的是：

$$
Find(x) = Find(y)
$$

### 11.3 Union 前要先 Find

合并两个元素时，不能直接把 $x$ 挂到 $y$。

正确做法是先找到两个集合的根，再合并根：

$$
parent[Find(y)] = Find(x)
$$

### 11.4 路径压缩只优化结构

路径压缩会改变 `parent` 数组中的父子关系，但不会改变集合划分。

压缩前后，任意两个元素是否属于同一集合的结果不变。

### 11.5 与 Kruskal 的关系

Kruskal 算法用并查集判断边的两个端点是否已经连通。

若已连通，加入该边会成环，应舍弃。

若不连通，加入该边不会成环，应合并两个连通分量。

## 12. 小结

| 内容 | 重点 |
| --- | --- |
| 存储结构 | 双亲数组 |
| 集合表示 | 每个集合是一棵树 |
| 代表元 | 树根 |
| 查找 | 找到元素所在集合的根 |
| 合并 | 合并两个根所在的集合 |
| 路径压缩 | 查找时把路径结点直接挂到根下 |
| 按秩合并 | 让较小或较矮的树挂到较大或较高的树下 |
| 典型应用 | 连通分量、Kruskal 算法 |

并查集的核心口诀是：查根、比根、合根。
