<h1><center>第四章 指令系统</center></h1>

## 1. 本章定位

指令系统是软件和硬件之间的重要接口。

从程序员角度看，指令系统规定了机器能够执行哪些操作、使用哪些寄存器、支持哪些寻址方式以及指令如何编码。

从硬件角度看，CPU 必须根据指令系统完成取指、译码、取操作数、执行和写回。

本章主要回答四个问题：

1. 一条机器指令由哪些字段组成。
2. 操作数地址如何得到。
3. 高级语言程序如何被表示为机器级代码。
4. CISC 和 RISC 两种指令系统思想有什么区别。

## 3. 学习主线

本章建议按下面顺序理解：

$$
\text{指令格式}
\longrightarrow
\text{操作码和地址码}
\longrightarrow
\text{寻址方式}
\longrightarrow
\text{机器级代码}
\longrightarrow
\text{CISC 与 RISC}
$$

::: tip 重点
第四章的核心是“指令如何表达操作”。做题时要先看操作码，再看地址码和寻址方式，最后再判断操作数来自寄存器、主存、立即数还是程序计数器相关位置。
:::

## 4. 常见考点

| 考点 | 要求 |
| --- | --- |
| 指令格式 | 会分析操作码、地址码和指令长度 |
| 地址数 | 会区分零地址、一地址、二地址和三地址指令 |
| 扩展操作码 | 会根据指令格式计算可表示指令条数 |
| 寻址方式 | 会判断操作数地址如何得到 |
| 相对寻址 | 会根据 PC 和位移量计算目标地址 |
| 基址和变址 | 会区分基址寄存器和变址寄存器的作用 |
| 机器级代码 | 能理解赋值、分支、循环和数组访问的机器级形式 |
| CISC/RISC | 会比较两类指令系统的特点 |

## 5. 易错点

- 指令字长、机器字长、存储字长不是同一个概念。
- 指令中的地址码不一定直接给出操作数地址，也可能给出寄存器编号或位移量。
- 立即寻址的操作数在指令中，不需要再访问主存取操作数。
- 间接寻址需要先访问主存得到有效地址，再访问有效地址处的操作数。
- 相对寻址通常以 PC 为基准，注意 PC 此时可能已经指向下一条指令。
- RISC 不等于指令条数一定少，而是强调简单、规整、易流水化。
