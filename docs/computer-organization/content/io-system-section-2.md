<h1><center>第二节 I/O 方式</center></h1>

## 1. I/O 方式的核心问题

I/O 方式研究的是 CPU、主存和外设之间如何完成数据传送。

由于外设速度通常比 CPU 慢很多，如果 CPU 每次都等待外设完成操作，就会造成大量时间浪费。

因此，I/O 方式的核心问题是：

1. CPU 是否需要等待外设。
2. CPU 是否直接参与每个数据的传送。
3. 数据是否可以在主存和外设之间直接传输。

408 中重点掌握三种方式：

- 程序查询方式
- 程序中断方式
- DMA 方式

## 2. 程序查询方式

程序查询方式也称轮询方式。

CPU 通过程序不断读取 I/O 接口的状态寄存器，判断设备是否准备好。如果设备未准备好，CPU 继续查询；如果设备准备好，CPU 再执行数据传送。

## 3. 程序查询方式的流程

以输入设备为例：

1. CPU 向接口发送启动命令。
2. CPU 反复读取状态端口。
3. 若设备未准备好，CPU 继续查询。
4. 若设备准备好，CPU 从数据端口读取数据。
5. CPU 把数据送入寄存器或主存。
6. 继续处理后续数据。

可以概括为：

$$
\text{启动设备}
\rightarrow
\text{查询状态}
\rightarrow
\text{设备就绪}
\rightarrow
\text{传送数据}
$$

## 4. 程序查询方式的特点

优点：

- 控制简单。
- 硬件开销小。
- 不需要复杂中断控制逻辑。

缺点：

- CPU 一直轮询，利用率低。
- CPU 和外设串行工作。
- 不适合低速设备长时间等待，也不适合高速大批量传输。

适用场景：

- 简单系统。
- 少量数据传输。
- 对 CPU 利用率要求不高的场合。

## 5. 程序查询方式例子

键盘接口有一个状态位 `Ready`。

CPU 读取状态端口：

- `Ready = 0`：没有新字符，继续查询。
- `Ready = 1`：有新字符，从数据端口读取字符。

这种方式简单，但如果用户长时间不按键，CPU 会浪费大量时间反复检查状态。

## 6. 程序中断方式

程序中断方式中，CPU 启动外设后可以继续执行原程序。

当外设准备好或操作完成时，通过 I/O 接口向 CPU 发出中断请求。CPU 在满足响应条件时暂停当前程序，转去执行中断服务程序，完成数据传送或相关处理。

中断方式的核心思想：

> 外设准备好后主动通知 CPU，而不是让 CPU 一直等待。

## 7. 中断方式的基本流程

以输入设备为例：

1. CPU 向接口发出启动命令。
2. CPU 继续执行当前程序。
3. 外设准备好数据。
4. 接口设置状态位并发出中断请求。
5. CPU 在当前指令执行结束后检测并响应中断。
6. CPU 保存断点和现场。
7. CPU 转入中断服务程序。
8. 中断服务程序读取 I/O 数据。
9. 服务结束后恢复现场。
10. CPU 返回原程序继续执行。

## 8. 中断响应条件

CPU 并不是一收到中断请求就立即响应。

通常需要满足：

- CPU 开中断。
- 当前指令执行结束。
- 中断请求未被屏蔽。
- 中断优先级满足要求。
- 没有更高优先级事件正在处理。

408 中常考“中断响应一般发生在一条指令执行结束后”。

## 9. 中断处理过程

中断处理通常包括：

| 阶段 | 作用 |
| --- | --- |
| 中断请求 | 外设或接口向 CPU 请求服务 |
| 中断判优 | 多个中断同时出现时确定优先级 |
| 中断响应 | CPU 暂停现行程序，转入中断处理 |
| 保护现场 | 保存断点和必要寄存器内容 |
| 中断服务 | 执行中断服务程序 |
| 恢复现场 | 恢复被中断程序的运行环境 |
| 中断返回 | 回到原程序继续执行 |

## 10. 中断向量

中断向量用于指出中断服务程序的入口地址，或用于查找入口地址。

常见方式：

- 固定入口方式：不同中断源对应固定入口。
- 向量中断方式：由中断源提供中断类型号或向量号，CPU 据此找到服务程序入口。

向量中断比非向量中断更便于快速定位服务程序。

## 11. 中断优先级和屏蔽

多个中断源同时请求服务时，需要判定优先级。

优先级用于决定先响应哪个中断。

中断屏蔽用于暂时禁止某些中断被响应。

常见考点：

- 高优先级中断可以打断低优先级中断服务。
- 被屏蔽的中断即使发出请求，也暂时不会被 CPU 响应。
- 不可屏蔽中断通常用于严重故障或紧急事件。

## 12. 中断方式的特点

优点：

- CPU 不必长期轮询设备状态。
- CPU 和外设可以并行工作。
- 适合随机、低速、少量数据传送。

缺点：

- 每次中断都要保存现场、转入服务程序、恢复现场。
- 中断频繁时开销较大。
- 每个数据的传送通常仍需要 CPU 参与。

适用场景：

- 键盘、鼠标等低速随机输入。
- 少量数据的异步传送。
- 外设事件通知。

## 13. DMA 方式

DMA 是直接存储器存取。

DMA 方式允许外设在 DMA 控制器控制下，直接与主存交换数据，而不需要 CPU 对每个数据逐个执行输入/输出指令。

DMA 的核心思想：

> 大批量数据由 DMA 控制器搬运，CPU 只负责初始化和收尾。

## 14. DMA 控制器

DMA 控制器是实现 DMA 传送的关键部件。

它通常包含：

| 部件 | 作用 |
| --- | --- |
| 主存地址寄存器 | 保存当前传送的主存地址 |
| 字计数器 | 记录还需传送的数据个数 |
| 数据缓冲寄存器 | 暂存传送数据 |
| 控制/状态寄存器 | 保存传送方向、方式和状态 |
| DMA 请求/响应逻辑 | 与外设和 CPU 协调总线控制权 |

CPU 在 DMA 开始前需要初始化 DMA 控制器。

初始化内容通常包括：

- 主存起始地址
- 传送方向
- 传送数据长度
- 外设地址或设备号
- DMA 工作方式

## 15. DMA 传送过程

典型 DMA 过程：

1. CPU 设置 DMA 控制器参数。
2. CPU 启动 I/O 设备。
3. 外设准备好后向 DMA 控制器发出 DMA 请求。
4. DMA 控制器向 CPU 或总线仲裁器请求总线。
5. CPU 让出总线控制权。
6. DMA 控制器成为总线主设备。
7. DMA 控制器控制外设和主存之间传送数据。
8. 每传送一个数据，主存地址和计数器更新。
9. 传送完成后，DMA 控制器释放总线。
10. DMA 控制器向 CPU 发出中断，报告传送结束。

注意：DMA 传送结束后常用中断通知 CPU。

## 16. DMA 的传送方式

### 停止 CPU 访存方式

DMA 传送期间，CPU 暂停访问主存。

特点：

- 控制简单。
- DMA 传送速度快。
- CPU 受影响较大。

### 周期挪用方式

DMA 控制器在 CPU 执行过程中偶尔占用一个或几个主存周期进行传送。

特点：

- CPU 只被短暂打断。
- DMA 和 CPU 交替使用主存。
- 是常见考点。

周期挪用也称周期窃取。

### DMA 与 CPU 交替访存方式

CPU 和 DMA 控制器按固定节拍交替访问主存。

特点：

- 不需要临时申请每个存储周期。
- 适合 CPU 工作周期和主存周期可协调的系统。
- 硬件控制要求较高。

## 17. DMA 方式的特点

优点：

- CPU 不参与每个数据的传送。
- 适合高速、大批量数据传输。
- 提高系统吞吐率。

缺点：

- 需要 DMA 控制器，硬件复杂。
- DMA 需要占用总线和主存周期。
- 可能影响 CPU 访存。

适用场景：

- 磁盘与主存之间的数据块传送。
- 网卡接收或发送大量数据。
- 高速数据采集。

## 18. 三种 I/O 方式对比

| 比较项 | 程序查询 | 程序中断 | DMA |
| --- | --- | --- | --- |
| CPU 是否等待设备 | 经常等待 | 不必长期等待 | 通常不等待数据传送 |
| CPU 是否参与每个数据传送 | 是 | 是 | 否 |
| 硬件复杂度 | 最低 | 较高 | 最高 |
| CPU 利用率 | 低 | 较高 | 高 |
| 适合场景 | 简单少量数据 | 低速随机事件 | 高速大批量数据 |
| 典型例子 | 简单状态轮询 | 键盘输入 | 磁盘块传送 |

## 19. 中断和 DMA 的区别

中断方式：

- 外设通知 CPU。
- CPU 执行中断服务程序。
- 数据传送通常由 CPU 完成。

DMA 方式：

- CPU 初始化 DMA 控制器。
- DMA 控制器完成数据传送。
- 传送结束后通常用中断通知 CPU。

一句话区分：

> 中断解决“什么时候让 CPU 来处理”，DMA 解决“大批量数据不让 CPU 一个个搬”。

## 20. 例题一：选择 I/O 方式

某系统需要从磁盘读取 4 KB 数据块到主存，要求 CPU 尽量少参与数据搬运，应采用哪种 I/O 方式？

分析：

磁盘数据块较大，适合外设和主存之间直接传送，应采用 DMA 方式。

CPU 只需设置主存地址、传送长度和方向，传送完成后由 DMA 控制器中断通知 CPU。

## 21. 例题二：判断 CPU 占用

某外设每隔较长时间随机产生一个字符，CPU 不希望反复查询状态位，应采用哪种方式？

分析：

外设事件随机且数据量少，适合程序中断方式。

设备准备好后发中断，CPU 再执行中断服务程序读取字符。

## 22. 例题三：周期挪用

DMA 传送过程中，DMA 控制器偶尔占用一个主存周期，CPU 只短暂停顿。这种方式称为什么？

答案：周期挪用方式。

它的特点是 DMA 和 CPU 交替竞争主存周期，CPU 不会在整个 DMA 传送期间完全停止。

## 23. 本节考点整理

| 考点 | 掌握程度 |
| --- | --- |
| 程序查询 | 能写出轮询状态位流程 |
| 中断方式 | 能写出中断请求、响应、服务、返回流程 |
| 中断条件 | 理解当前指令结束后响应中断 |
| 中断向量 | 理解服务程序入口定位 |
| DMA 控制器 | 知道主要寄存器和作用 |
| DMA 过程 | 能描述初始化、请求、传送、结束中断 |
| DMA 传送方式 | 掌握停止 CPU、周期挪用、交替访存 |
| 三方式比较 | 能从 CPU 参与程度和适用场景比较 |

## 24. 易错提醒

- 程序中断方式不是 CPU 完全不参与数据传送，而是不需要 CPU 长期等待。
- DMA 方式不是不需要 CPU，CPU 仍需初始化 DMA 控制器并处理结束中断。
- DMA 传送要占用总线和主存周期，可能影响 CPU 运行。
- 周期挪用不是一次性抢走全部总线，而是短暂占用若干存储周期。
- 中断响应通常发生在当前指令执行结束后，不是在任意机器周期立即跳转。

## 25. 小结

I/O 方式的演进方向是减少 CPU 等待和减少 CPU 搬运数据。

程序查询最简单但浪费 CPU；中断方式让设备准备好后再通知 CPU；DMA 方式进一步让 DMA 控制器直接搬运大批量数据。408 中最常见的题型就是比较三种方式的流程、硬件开销、CPU 参与程度和适用场景。
