<h1><center>第六节 虚拟存储器</center></h1>

## 1. 虚拟存储器的作用

虚拟存储器通过硬件和操作系统配合，使程序看到的地址空间可以大于实际主存容量。

其基本思想是：

- 程序使用虚拟地址。
- 实际主存使用物理地址。
- 通过地址转换把虚拟地址映射到物理地址。
- 当前需要的页面放在主存，不常用页面保存在外存。

虚拟存储器利用局部性原理，只把程序当前活跃部分放入主存。

## 2. 虚拟地址和物理地址

CPU 执行程序时产生的地址称为虚拟地址，也称逻辑地址。

主存实际访问使用的地址称为物理地址，也称实地址。

| 地址 | 含义 |
| --- | --- |
| 虚拟地址 | 程序看到的地址，由 CPU 产生 |
| 物理地址 | 主存实际使用的地址 |

地址转换由存储管理部件完成，常见硬件是 MMU。

## 3. 页式虚拟存储器

页式虚拟存储器把虚拟地址空间划分为固定大小的页。

主存空间划分为同样大小的页框，也称物理块。

| 概念 | 含义 |
| --- | --- |
| 页 | 虚拟地址空间中的固定大小块 |
| 页框 | 物理内存中的固定大小块 |
| 页号 | 虚拟地址中表示第几页的部分 |
| 页内偏移 | 页内具体位置 |

页和页框大小相同，因此地址转换时页内偏移不变。

## 4. 虚拟地址格式

页式虚拟存储器中，虚拟地址通常分为：

$$
\text{虚拟页号}
\quad
\text{页内偏移}
$$

若页面大小为 $P$ 字节，则页内偏移位数为：

$$
\log_2 P
$$

若虚拟地址位数为 $m$，页内偏移位数为 $d$，则虚拟页号位数为：

$$
m - d
$$

物理地址通常分为：

$$
\text{物理页框号}
\quad
\text{页内偏移}
$$

页内偏移在转换前后保持不变。

## 5. 页表

页表用于记录虚拟页号到物理页框号的映射关系。

页表项通常包含：

- 物理页框号
- 有效位
- 访问位
- 修改位
- 保护位

| 字段 | 作用 |
| --- | --- |
| 有效位 | 表示该页是否在主存中 |
| 修改位 | 表示该页调入主存后是否被写过 |
| 访问位 | 用于页面替换算法 |
| 保护位 | 控制读写执行权限 |

若有效位为 $1$，说明该虚拟页在主存中，可以完成地址转换。

若有效位为 $0$，说明该页不在主存中，可能发生缺页。

## 6. 地址转换过程

页式虚拟存储器地址转换过程：

1. CPU 产生虚拟地址。
2. 根据页面大小划分虚拟页号和页内偏移。
3. 根据虚拟页号查页表。
4. 若页表项有效，得到物理页框号。
5. 将物理页框号与页内偏移拼接成物理地址。
6. 用物理地址访问主存。

可以概括为：

$$
\text{虚拟页号}
\longrightarrow
\text{页表}
\longrightarrow
\text{物理页框号}
$$

页内偏移直接复制到物理地址低位。

## 7. TLB

TLB 是 Translation Lookaside Buffer，通常称为快表。

TLB 是页表项的高速缓存，用于加快地址转换。

没有 TLB 时，一次访存可能需要：

1. 访问主存中的页表。
2. 再访问主存中的实际数据。

有 TLB 时，若 TLB 命中，可以直接得到物理页框号，减少访问页表的开销。

TLB 与页表关系：

| 项目 | TLB | 页表 |
| --- | --- | --- |
| 位置 | 高速硬件缓存 | 通常在主存中 |
| 内容 | 部分页表项 | 全部或大量页表项 |
| 速度 | 快 | 慢 |
| 作用 | 加速地址转换 | 保存完整映射关系 |

## 8. 含 TLB 的地址转换

含 TLB 的地址转换过程：

1. CPU 产生虚拟地址。
2. 用虚拟页号查 TLB。
3. 若 TLB 命中，直接得到物理页框号。
4. 若 TLB 未命中，再查页表。
5. 若页表项有效，将页表项装入 TLB。
6. 拼接物理页框号和页内偏移，访问主存。

TLB 命中时：

$$
\text{虚拟页号}
\longrightarrow
\text{TLB}
\longrightarrow
\text{物理页框号}
$$

TLB 未命中时：

$$
\text{虚拟页号}
\longrightarrow
\text{页表}
\longrightarrow
\text{物理页框号}
$$

## 9. 缺页

若页表项无效，说明需要访问的页不在主存中，会发生缺页。

缺页处理通常由操作系统完成。

基本过程：

1. CPU 发现页表项无效，产生缺页异常。
2. 操作系统从外存找到所需页面。
3. 若主存有空闲页框，将页面调入。
4. 若主存没有空闲页框，选择一个页面换出。
5. 更新页表和 TLB。
6. 重新执行引发缺页的指令。

若被换出的页面修改位为 $1$，需要先写回外存。

## 10. 页面替换

当主存没有空闲页框时，需要选择一个页面换出。

常见页面替换算法：

| 算法 | 思想 |
| --- | --- |
| FIFO | 换出最早进入主存的页面 |
| LRU | 换出最近最久未使用的页面 |
| Clock | 通过访问位近似实现 LRU |

页面替换算法属于操作系统重点内容，但在计组中需要理解其与虚拟存储器硬件字段有关，例如访问位和修改位。

## 11. 虚拟存储器与 Cache

虚拟存储器和 Cache 都利用局部性原理，但二者解决的问题不同。

| 项目 | Cache | 虚拟存储器 |
| --- | --- | --- |
| 目的 | 缓解 CPU 与主存速度差异 | 扩大程序可用地址空间 |
| 数据交换单位 | 主存块和 Cache 行 | 页面 |
| 管理主体 | 主要由硬件管理 | 硬件和操作系统共同管理 |
| 未命中代价 | 访问主存 | 访问外存，代价很高 |
| 程序员可见性 | 通常透明 | 通常也透明，但与地址空间相关 |

Cache 未命中通常由硬件处理。

缺页通常需要操作系统参与，代价远高于 Cache 未命中。

## 12. 地址转换与 Cache 的关系

实际系统中，地址转换和 Cache 访问之间存在多种组织方式。

408 通常只要求理解基本关系：

- CPU 产生虚拟地址。
- 经过地址转换得到物理地址。
- 用物理地址访问主存或 Cache。

更复杂的虚拟 Cache、物理 Cache 组织一般不作为基础重点。

## 13. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 地址划分 | 根据页大小求页号和页内偏移位数 |
| 地址转换 | 根据页表求物理地址 |
| TLB 命中 | 判断是否需要访问页表 |
| 缺页判断 | 根据有效位判断是否缺页 |
| 页表项字段 | 判断有效位、修改位、访问位作用 |
| Cache 对比 | 区分 Cache 未命中和缺页 |

## 14. 易错点

- 虚拟地址由 CPU 产生，物理地址用于访问主存。
- 页内偏移在虚拟地址和物理地址中相同。
- TLB 保存的是页表项，不是实际数据。
- TLB 命中不等于 Cache 命中。
- 缺页不是普通 Cache 未命中，通常需要操作系统处理。
- 修改位为 $1$ 的页面被换出时需要写回外存。
- 页面大小越大，页内偏移位数越多。

## 15. 本节小结

虚拟存储器的核心是地址转换。

做题时先根据页大小划分页号和页内偏移，再通过 TLB 或页表找到物理页框号，最后把物理页框号与页内偏移拼接成物理地址。
