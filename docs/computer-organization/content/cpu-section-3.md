<h1><center>第三节 数据通路的功能和基本结构</center></h1>

## 1. 数据通路的概念

数据通路是 CPU 中数据传送和加工所经过的路径。

它由寄存器、ALU、总线、多路选择器、暂存器等部件组成。

简单说：

$$
\text{数据通路负责数据怎么走}
$$

而控制器负责：

$$
\text{什么时候让数据走}
$$

## 2. 数据通路的功能

数据通路主要完成：

- 寄存器之间的数据传送。
- 寄存器和 ALU 之间的数据传送。
- CPU 和主存之间的数据传送。
- ALU 运算结果写回寄存器。
- PC 更新。
- 地址形成。

一条指令的执行过程可以看作一系列微操作在数据通路上依次发生。

## 3. 数据通路的基本部件

常见数据通路部件：

| 部件 | 作用 |
| --- | --- |
| 通用寄存器组 | 保存操作数和中间结果 |
| PC | 保存下一条指令地址 |
| IR | 保存当前指令 |
| MAR | 保存访存地址 |
| MDR | 保存访存数据 |
| ALU | 完成算术逻辑运算 |
| 暂存寄存器 | 暂存 ALU 输入或中间结果 |
| 总线 | 连接各部件，传送数据 |
| 多路选择器 | 从多个输入中选择一个送出 |

## 4. 单总线数据通路

单总线数据通路中，CPU 内部多个部件共享一条总线。

特点：

- 结构简单。
- 硬件成本低。
- 同一时刻只能有一个部件向总线发送数据。
- 微操作并行性差。
- 通常需要暂存寄存器。

例如，要完成：

$$
R1 \leftarrow R2 + R3
$$

在单总线结构中可能需要：

1. $R2$ 输出到总线，送入暂存寄存器 $Y$。
2. $R3$ 输出到总线，ALU 计算 $Y + R3$。
3. ALU 结果送入暂存寄存器 $Z$。
4. $Z$ 输出到总线，写入 $R1$。

由于总线一次只能传一个数据源，步骤较多。

## 5. 双总线和三总线数据通路

多总线数据通路可以提高并行性。

双总线结构中，ALU 可以同时从两条总线接收两个操作数。

三总线结构中，通常两条总线用于读操作数，一条总线用于写回结果。

例如：

$$
R1 \leftarrow R2 + R3
$$

在三总线结构中可以更接近一次完成：

$$
R2,\ R3
\longrightarrow
ALU
\longrightarrow
R1
$$

多总线结构特点：

- 并行性更好。
- 指令执行周期可能更少。
- 硬件成本更高。
- 控制信号更复杂。

## 6. 专用数据通路

专用数据通路为某些操作设置专门连接。

例如：

- PC 到 MAR 的专用通路。
- ALU 到 PC 的专用通路。
- MDR 到 IR 的专用通路。

特点：

- 速度快。
- 某些常用操作更高效。
- 硬件连接复杂。
- 灵活性相对低。

## 7. 数据通路中的控制信号

数据通路本身只是硬件连接，必须由控制信号驱动。

常见控制信号包括：

- 寄存器输出使能
- 寄存器写入使能
- ALU 操作选择
- 多路选择器选择信号
- 主存读写信号
- PC 更新信号

例如，要完成：

$$
R1 \leftarrow R2 + R3
$$

控制器需要发出：

1. 选择 $R2$ 作为 ALU 输入之一。
2. 选择 $R3$ 作为 ALU 输入之二。
3. 指定 ALU 做加法。
4. 允许结果写入 $R1$。

## 8. 数据通路与指令执行

不同指令使用的数据通路不同。

| 指令类型 | 数据流 |
| --- | --- |
| 算术逻辑指令 | 寄存器 $\rightarrow$ ALU $\rightarrow$ 寄存器 |
| Load 指令 | 主存 $\rightarrow$ MDR $\rightarrow$ 寄存器 |
| Store 指令 | 寄存器 $\rightarrow$ MDR $\rightarrow$ 主存 |
| 转移指令 | ALU 或地址形成部件 $\rightarrow$ PC |

分析数据通路题时，要先判断指令类型，再画出数据从哪里来、经过哪里、到哪里去。

## 9. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 数据流分析 | 判断某条指令执行时数据经过哪些部件 |
| 微操作顺序 | 写出单总线结构下的微操作序列 |
| 总线冲突 | 判断哪些微操作不能在同一时刻执行 |
| 控制信号 | 判断需要哪些寄存器读写和 ALU 控制信号 |
| 数据通路对比 | 比较单总线、多总线和专用通路特点 |

## 10. 易错点

- 数据通路不是控制器，不能主动决定何时传送数据。
- 单总线结构同一时刻不能有多个输出源同时驱动总线。
- 多总线结构并行性更好，但硬件更复杂。
- ALU 运算通常需要输入暂存寄存器或多路选择器配合。
- Store 指令是写主存，不是从主存读数据。

## 11. 本节小结

数据通路是指令执行的“道路系统”。

控制器发出信号，数据通路根据这些信号让数据在寄存器、ALU、主存之间流动。408 中要能根据指令写出数据流和微操作。
