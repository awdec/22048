<h1><center>408 计算机组成原理考点分析</center></h1>

## 1. 408 中的计算机组成原理分值

408 全称通常为“计算机学科专业基础综合”，满分为：

$$
150\text{ 分}
$$

近年 408 的科目分值结构通常为：

| 科目 | 分值 | 占比 |
| --- | ---: | ---: |
| 数据结构 | $45$ 分 | $30\%$ |
| 计算机组成原理 | $45$ 分 | $30\%$ |
| 操作系统 | $35$ 分 | $23.3\%$ |
| 计算机网络 | $25$ 分 | $16.7\%$ |
| 合计 | $150$ 分 | $100\%$ |

::: tip 说明
具体分值、题型和考查范围以当年考试大纲和当年真题为准。计算机组成原理通常与数据结构同为 408 中分值最高的科目。
:::

## 2. 408 试卷题型结构

408 常见题型由两部分组成：

| 题型 | 常见分值 |
| --- | ---: |
| 单项选择题 | $80$ 分 |
| 综合应用题 | $70$ 分 |
| 合计 | $150$ 分 |

计算机组成原理的 $45$ 分通常会同时出现在：

- 单项选择题
- 综合应用题

常见理解方式：

| 部分 | 计组常见考查 |
| --- | --- |
| 选择题 | 概念、公式、地址划分、运算过程、系统结构特点 |
| 综合应用题 | 数据表示与运算、Cache、指令系统、CPU 数据通路、流水线、中断和 DMA |

::: warning 注意
计组题目往往不是单纯背概念，而是把“二进制表示、硬件结构、执行过程、性能计算”揉在一起考。复习时要同时会算、会画、会分析过程。
:::

## 3. 计组内部重点层级

### 第一层：必须掌握

这些内容属于计算机组成原理的主干，选择题和综合题都可能出现。

| 模块 | 重点内容 | 常见难点 |
| --- | --- | --- |
| 计算机系统概述 | 冯·诺依曼结构、层次结构、性能指标 | CPI、MIPS、吞吐率、响应时间 |
| 数据的表示和运算 | 进制转换、原码、反码、补码、浮点数、定点运算 | 补码运算、溢出判断、IEEE 754 |
| 存储系统 | 存储器层次、主存、Cache、虚拟存储器 | Cache 地址划分、映射方式、命中率 |
| 指令系统 | 指令格式、寻址方式、CISC 与 RISC | 操作数地址计算、指令长度分析 |
| 中央处理器 | CPU 结构、指令执行过程、控制器、流水线 | 数据通路、控制信号、流水线性能 |
| 总线 | 总线结构、总线事务、仲裁、定时 | 同步与异步定时、带宽计算 |
| 输入输出系统 | I/O 方式、中断、DMA | 中断处理流程、DMA 与 CPU 的关系 |

### 第二层：高频难点

这些内容容易出综合应用题，需要能手算过程。

| 模块 | 高频难点 |
| --- | --- |
| 性能指标 | CPI、主频、执行时间、MIPS、吞吐率 |
| 补码运算 | 加减法、符号扩展、溢出判断 |
| 浮点数 | IEEE 754 表示、规格化、阶码和尾数 |
| Cache | 地址字段划分、直接映射、组相联、全相联 |
| Cache 性能 | 命中率、平均访问时间、替换策略、写策略 |
| 指令系统 | 寻址方式、指令格式、操作数位置 |
| CPU | 单周期、多周期、流水线、数据通路 |
| 流水线 | 理想加速比、实际执行时间、结构相关、数据相关、控制相关 |
| 中断 | 中断响应、中断处理、中断屏蔽 |
| DMA | DMA 传送方式、总线占用、与中断的区别 |

### 第三层：容易丢分

这些内容看起来熟悉，但细节很容易出错。

| 易错点 | 说明 |
| --- | --- |
| 机器数与真值混淆 | 原码、反码、补码表示的是机器数，真值是实际数值 |
| 补码范围记错 | $n$ 位补码整数范围是 $-2^{n-1}$ 到 $2^{n-1}-1$ |
| 符号扩展出错 | 补码符号扩展要补符号位，不是简单补 $0$ |
| 溢出判断混乱 | 要区分进位和溢出，两个概念不等价 |
| 浮点规格化 | 尾数规格化和阶码偏置容易算错 |
| Cache 地址划分 | 标记、组号、块内地址字段要由映射方式决定 |
| Cache 写策略混淆 | 写直达、写回、写分配、非写分配要分清 |
| 指令字长和机器字长混淆 | 指令字长、机器字长、存储字长不是同一概念 |
| CPI 与主频关系 | 主频高不一定执行时间短，还要看 CPI 和指令条数 |
| 中断和 DMA 混淆 | DMA 传输数据主体不是 CPU，但通常需要中断通知完成 |

## 5. 重点难点说明

### 数据的表示和运算

这一章是计组计算题的基础。

重点掌握：

- 二进制、八进制、十进制、十六进制之间的转换。
- 原码、反码、补码、移码的表示范围。
- 补码加减运算。
- 溢出判断。
- 定点数和浮点数表示。
- IEEE 754 浮点数。

难点在于：

- 负数补码的表示和还原。
- 符号位、数值位和进位之间的关系。
- 浮点数阶码、尾数、偏置值的处理。
- 舍入、溢出、规格化对结果的影响。

### 存储系统

存储系统是计组中最容易出综合题的模块之一。

重点掌握：

- 存储器层次结构。
- SRAM、DRAM、ROM、Flash 的特点。
- 主存容量和地址线计算。
- Cache 的基本原理。
- Cache 映射方式。
- Cache 替换策略和写策略。
- 平均访问时间。

Cache 地址划分是高频考点。一般要根据题目条件判断：

1. 主存地址位数。
2. Cache 总容量。
3. 块大小。
4. 映射方式。
5. 标记、组号、块内地址分别占多少位。

### 指令系统

指令系统重点考查“指令如何表达操作”。

重点掌握：

- 指令格式。
- 操作码和地址码。
- 定长指令和变长指令。
- 寻址方式。
- CISC 和 RISC 的基本特点。

常见题型包括：

- 根据操作码位数求最多能表示多少条指令。
- 根据地址码位数求寻址范围。
- 判断某条指令使用的寻址方式。
- 计算有效地址。

### 中央处理器

CPU 是理解整门计组的核心。

重点掌握：

- CPU 的基本组成。
- 指令执行过程。
- 取指、译码、执行、访存、写回。
- 数据通路。
- 控制器。
- 硬布线控制和微程序控制。
- 指令流水线。

难点通常在数据通路和流水线：

- 每个阶段使用哪些部件。
- 每条指令需要哪些控制信号。
- 流水线中相关如何产生。
- 如何通过转发、暂停、分支预测缓解相关。

### 输入输出系统

I/O 系统常与中断、DMA、总线联系在一起考。

重点掌握：

- 程序查询方式。
- 程序中断方式。
- DMA 方式。
- I/O 接口。
- 中断响应和中断处理过程。

三种 I/O 方式对比：

| 方式 | CPU 参与程度 | 特点 |
| --- | --- | --- |
| 程序查询 | 最高 | CPU 不断查询设备状态，效率低 |
| 程序中断 | 中 | 设备准备好后主动请求 CPU 服务 |
| DMA | 低 | 数据块传输主要由 DMA 控制器完成 |

## 6. 计组与其他科目的联系

计算机组成原理不是孤立科目，很多考点会和其他科目形成交叉。

| 关联科目 | 关联内容 |
| --- | --- |
| 数据结构 | 数据在内存中的存放、地址计算、位运算 |
| 操作系统 | 虚拟存储器、中断、I/O、进程切换的硬件基础 |
| 计算机网络 | 数据校验、编码、传输速率、带宽概念 |

其中，存储系统和操作系统联系最紧密。复习 Cache、页式存储、TLB、虚拟地址和物理地址时，要把计组和操作系统放在一起理解。

## 7. 复习策略

### 第一轮

目标是理解基本硬件结构和基础概念。

重点：

- 计算机系统层次结构。
- 机器数和数据表示。
- 存储器基本结构。
- 指令格式和寻址方式。
- CPU 基本组成。
- I/O 基本方式。

### 第二轮

目标是掌握高频计算和过程分析。

重点：

- 补码和浮点数计算。
- Cache 地址划分。
- 存储器容量计算。
- 指令地址计算。
- CPI 和执行时间计算。
- 流水线性能分析。

### 第三轮

目标是面向真题查漏补缺。

重点：

- 总结易错公式。
- 熟悉综合题书写步骤。
- 训练 Cache、流水线、指令系统类题目。
- 对比中断和 DMA。
- 把计组与操作系统的交叉点串起来。

## 8. 计算机组成原理拿分建议

计组要想稳定拿分，应做到：

1. 公式不要死背，要知道每个量代表什么。
2. 二进制、补码、浮点数计算要熟练。
3. Cache 题要先画地址字段。
4. 指令系统题要先区分操作码和地址码。
5. CPU 题要按指令执行阶段分析。
6. 流水线题要分清理想情况和实际阻塞。
7. 中断和 DMA 要能用过程语言说明区别。

408 计算机组成原理的目标不只是“记住硬件名词”，而是：

$$
\text{能表示}
\quad
\text{能计算}
\quad
\text{能划分}
\quad
\text{能分析执行过程}
$$

## 9. 考前速记

1. 计算机组成原理约占 408 的 $45$ 分。
2. 数据表示、存储系统、指令系统、CPU 是复习主干。
3. 补码范围、符号扩展、溢出判断是基础高频点。
4. Cache 题先判断映射方式，再划分地址字段。
5. 指令系统题重点看操作码、地址码、寻址方式。
6. CPU 题重点看数据通路、控制信号和流水线。
7. I/O 题重点区分程序查询、中断和 DMA。
8. 计组题经常需要“公式 + 过程 + 结构图”一起理解。
