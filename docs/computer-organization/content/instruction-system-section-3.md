<h1><center>第三节 程序的机器级代码表示</center></h1>

## 1. 本节定位

程序的机器级代码表示研究高级语言程序如何转换为机器能够执行的指令序列。

408 中通常不会要求掌握某一种复杂汇编语言的全部细节，而是重视：

- 指令如何表达变量赋值。
- 指令如何表达算术逻辑运算。
- 指令如何表达条件分支和循环。
- 指令如何计算数组和指针地址。
- 过程调用如何保存返回地址和现场。

## 2. 从高级语言到机器代码

高级语言程序需要经过编译、汇编、链接等过程，最终形成机器代码。

典型过程：

$$
\text{高级语言程序}
\longrightarrow
\text{汇编代码}
\longrightarrow
\text{机器代码}
\longrightarrow
\text{可执行文件}
$$

| 阶段 | 作用 |
| --- | --- |
| 编译 | 将高级语言翻译为汇编语言或目标代码 |
| 汇编 | 将汇编指令转换为机器指令 |
| 链接 | 合并多个目标模块，确定外部符号地址 |
| 装入 | 将程序装入内存并准备执行 |

机器级代码是指接近机器指令层次的程序表示。

## 3. 机器级代码中的基本对象

机器级代码通常围绕以下对象展开：

- 寄存器
- 主存单元
- 立即数
- 条件码或标志位
- 程序计数器 PC
- 栈指针 SP

| 对象 | 作用 |
| --- | --- |
| 寄存器 | 保存临时数据、地址或状态 |
| 主存 | 保存变量、数组、代码和栈 |
| 立即数 | 指令中直接给出的常量 |
| PC | 保存下一条指令地址 |
| SP | 指向栈顶位置 |

CPU 执行机器级代码时，本质上是在寄存器和主存之间搬运数据，并通过 ALU 完成运算。

## 4. 赋值语句的机器级表示

高级语言赋值：

```c
x = y;
```

在机器级代码中通常表示为：

1. 从变量 `y` 所在地址取出数据。
2. 将数据写入变量 `x` 所在地址。

如果采用寄存器作为中转，过程可以抽象为：

```txt
LOAD  R1, y
STORE x, R1
```

含义：

- `LOAD` 从主存读到寄存器。
- `STORE` 从寄存器写回主存。

在 load-store 型体系结构中，主存和寄存器之间的数据传送通常只能由专门的访存指令完成。

## 5. 算术逻辑表达式

高级语言表达式：

```c
z = x + y;
```

机器级代码可能表示为：

```txt
LOAD  R1, x
LOAD  R2, y
ADD   R3, R1, R2
STORE z, R3
```

基本过程：

1. 把操作数从主存取到寄存器。
2. 在寄存器中完成运算。
3. 把结果写回主存。

对于复杂表达式，编译器会使用寄存器保存中间结果。

## 6. 条件分支

高级语言条件语句：

```c
if (x == y) {
    z = 1;
} else {
    z = 0;
}
```

机器级代码通常包含：

- 比较指令。
- 条件转移指令。
- 无条件转移指令。

抽象形式：

```txt
LOAD  R1, x
LOAD  R2, y
CMP   R1, R2
JNE   ELSE
MOV   R3, #1
STORE z, R3
JMP   END
ELSE:
MOV   R3, #0
STORE z, R3
END:
```

其中：

- `CMP` 设置条件码或标志位。
- `JNE` 根据条件码判断是否跳转。
- `JMP` 无条件改变 PC。

条件分支的本质是根据条件修改 PC。

## 7. 循环语句

高级语言循环：

```c
while (i < n) {
    sum = sum + a[i];
    i = i + 1;
}
```

机器级代码通常由比较和转移构成：

```txt
LOOP:
CMP   i, n
JGE   END
LOAD  R1, sum
LOAD  R2, a[i]
ADD   R1, R1, R2
STORE sum, R1
ADD   i, i, #1
JMP   LOOP
END:
```

循环结构在机器级没有特殊魔法，本质上是：

$$
\text{比较条件}
\longrightarrow
\text{条件转移}
\longrightarrow
\text{循环体}
\longrightarrow
\text{无条件转移回开头}
$$

## 8. 数组访问

数组访问的核心是地址计算。

若数组首地址为 $base$，每个元素大小为 $s$ 字节，访问 $a[i]$ 的地址为：

$$
base + i \times s
$$

对于二维数组，若按行优先存储，数组 `a[m][n]` 中元素 `a[i][j]` 的地址为：

$$
base + (i \times n + j) \times s
$$

其中：

- $base$ 是数组首地址。
- $n$ 是每行元素个数。
- $s$ 是每个元素字节数。

机器级代码中，数组访问常使用变址寻址或基址加变址形式实现。

## 9. 指针访问

指针变量保存的是地址。

例如：

```c
y = *p;
```

机器级含义是：

1. 读取指针变量 `p` 的值。
2. 把该值作为地址。
3. 到该地址读取数据。
4. 将数据赋给 `y`。

抽象代码：

```txt
LOAD  R1, p
LOAD  R2, (R1)
STORE y, R2
```

其中 `(R1)` 表示以寄存器 $R1$ 中的内容作为地址访问主存。

指针访问与寄存器间接寻址、间接寻址关系密切。

## 10. 过程调用

过程调用也称函数调用。

机器级代码中，过程调用通常需要处理：

- 参数传递。
- 返回地址保存。
- 跳转到被调用过程。
- 保存和恢复现场。
- 返回调用点。

典型调用过程：

1. 调用者准备参数。
2. 保存返回地址。
3. 跳转到被调用函数入口。
4. 被调用函数建立栈帧。
5. 执行函数体。
6. 返回值放入约定位置。
7. 恢复现场并返回。

函数调用依赖栈保存临时信息。

常见栈中内容：

- 返回地址
- 参数
- 局部变量
- 保存的寄存器值

## 11. 栈帧

栈帧是一次函数调用在栈中占用的存储区域。

一个栈帧中可能包含：

- 返回地址。
- 上一个栈帧指针。
- 局部变量。
- 临时数据。
- 保存的寄存器。

不同体系结构和调用约定的栈帧布局不同。408 中通常只需要理解函数调用为什么需要栈，以及返回地址和局部变量如何保存。

## 12. 机器指令与二进制编码

汇编指令最终会被编码为二进制机器指令。

例如抽象指令：

```txt
ADD R1, R2, R3
```

机器码中可能包含：

- ADD 对应的操作码。
- 寄存器 $R1$ 的编号。
- 寄存器 $R2$ 的编号。
- 寄存器 $R3$ 的编号。

也就是说：

$$
\text{汇编助记符}
\longrightarrow
\text{操作码和字段编码}
\longrightarrow
\text{机器指令}
$$

标签在汇编阶段会被转换为地址或相对位移。

## 13. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 语句翻译 | 判断赋值、加法、条件语句需要哪些机器操作 |
| 分支执行 | 根据条件码判断是否跳转 |
| 循环结构 | 分析循环对应的比较和转移指令 |
| 数组地址 | 根据首地址、下标和元素大小计算地址 |
| 指针访问 | 判断需要几次访存 |
| 函数调用 | 判断返回地址、参数和局部变量与栈的关系 |

## 14. 易错点

- 高级语言中的一条语句可能对应多条机器指令。
- 条件语句和循环最终都依赖条件转移和无条件转移。
- 数组地址计算要乘以元素大小。
- 指针变量的值是地址，取指针指向的数据通常还要再访存。
- 标签不是机器执行时的文字名称，最终会变成地址或位移量。
- 函数调用会改变 PC，并需要保存返回地址。

## 15. 本节小结

机器级代码的本质是用指令序列描述数据传送、运算和控制流。

高级语言看起来有变量、数组、循环和函数；到了机器级，核心就变成了寄存器、主存地址、ALU 运算、条件码、PC 和栈。
