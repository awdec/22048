<h1><center>第五节 高速缓冲存储器</center></h1>

## 1. Cache 的作用

高速缓冲存储器简称 Cache，用于缓解 CPU 与主存之间的速度差异。

Cache 位于 CPU 和主存之间，保存主存中一部分最近或即将被访问的数据副本。

其基本思想是：

$$
\text{把常用数据放在离 CPU 更近、更快的地方}
$$

Cache 能有效工作的原因是程序具有局部性。

## 2. Cache 基本概念

| 概念 | 含义 |
| --- | --- |
| 主存块 | 主存与 Cache 之间交换的数据单位 |
| Cache 行 | Cache 中存放一个主存块的位置 |
| 标记 | 用于判断 Cache 行中保存的是哪个主存块 |
| 有效位 | 表示 Cache 行中的内容是否有效 |
| 命中 | CPU 访问的数据在 Cache 中 |
| 未命中 | CPU 访问的数据不在 Cache 中 |

主存和 Cache 之间通常按块交换数据，而 CPU 访问时通常按字节或字访问。

## 3. 命中率和平均访问时间

命中率表示访问 Cache 成功的比例。

$$
H
=
\frac{\text{命中次数}}{\text{总访问次数}}
$$

未命中率为：

$$
1 - H
$$

平均访问时间常写作：

$$
AMAT
=
\text{命中时间}
+
\text{未命中率}
\times
\text{未命中代价}
$$

也可以根据题目描述写成：

$$
T
=
H \times T_c
+
(1-H) \times T_m
$$

其中：

- $T_c$ 表示 Cache 访问时间。
- $T_m$ 表示未命中时访问主存相关时间。

::: warning 注意
不同题目对“未命中访问时间”是否包含已经访问 Cache 的时间，表述可能不同。做题时要按题目给出的含义列式。
:::

## 4. Cache 映射方式

主存块放入 Cache 的规则称为映射方式。

常见映射方式：

- 直接映射
- 全相联映射
- 组相联映射

### 4.1 直接映射

直接映射中，每个主存块只能映射到 Cache 中唯一一个行。

若 Cache 共有 $C$ 行，主存块号为 $B$，则映射到的 Cache 行号为：

$$
B \bmod C
$$

特点：

- 硬件实现简单。
- 查找速度快。
- 冲突概率较高。
- 替换策略简单，因为位置唯一。

### 4.2 全相联映射

全相联映射中，每个主存块可以放入 Cache 的任意一行。

特点：

- 冲突概率低。
- Cache 利用率高。
- 标记比较复杂。
- 硬件成本高。
- 需要替换策略。

### 4.3 组相联映射

组相联映射是直接映射和全相联映射的折中。

Cache 被分成若干组，每组有若干行。

主存块只能映射到指定组，但可以放在该组中的任意一行。

若 Cache 有 $G$ 组，主存块号为 $B$，则组号为：

$$
B \bmod G
$$

若每组有 $n$ 行，则称为 $n$ 路组相联。

特点：

- 冲突少于直接映射。
- 硬件复杂度低于全相联。
- 408 中非常常考。

## 5. Cache 地址划分

Cache 地址划分是 408 高频考点。

设：

- 主存地址共 $m$ 位。
- 主存块大小为 $B$ 字节。
- Cache 共有 $L$ 行。
- Cache 共有 $G$ 组。

块内地址位数为：

$$
\log_2 B
$$

### 5.1 直接映射地址格式

直接映射地址格式：

$$
\text{标记}
\quad
\text{Cache 行号}
\quad
\text{块内地址}
$$

Cache 行号位数为：

$$
\log_2 L
$$

标记位数为：

$$
m
-
\log_2 L
-
\log_2 B
$$

### 5.2 全相联映射地址格式

全相联映射地址格式：

$$
\text{标记}
\quad
\text{块内地址}
$$

标记位数为：

$$
m
-
\log_2 B
$$

全相联没有行号字段，因为主存块可以放入任意 Cache 行。

### 5.3 组相联映射地址格式

组相联映射地址格式：

$$
\text{标记}
\quad
\text{组号}
\quad
\text{块内地址}
$$

组号位数为：

$$
\log_2 G
$$

标记位数为：

$$
m
-
\log_2 G
-
\log_2 B
$$

## 6. 替换策略

当 Cache 未命中且目标位置已满时，需要选择一个旧块替换出去。

常见替换策略：

| 策略 | 含义 |
| --- | --- |
| FIFO | 替换最早进入 Cache 的块 |
| LRU | 替换最近最久未使用的块 |
| LFU | 替换使用次数最少的块 |
| Random | 随机替换 |

直接映射不需要复杂替换策略，因为每个主存块位置唯一。

全相联和组相联需要在可选范围内选择替换块。

## 7. 写策略

CPU 写 Cache 时，需要考虑主存中的副本是否同步更新。

### 7.1 写直达

写直达也称全写法。

写操作同时更新 Cache 和主存。

特点：

- 主存数据始终较新。
- 实现简单。
- 写操作开销较大。

### 7.2 写回

写回法只修改 Cache，不立即写主存。

当该 Cache 行被替换时，若内容被修改过，再写回主存。

特点：

- 减少主存写次数。
- 需要脏位标记是否被修改。
- 实现较复杂。

### 7.3 写分配和非写分配

写未命中时，还需要决定是否把对应主存块调入 Cache。

| 策略 | 含义 |
| --- | --- |
| 写分配 | 写未命中时，把主存块调入 Cache 再写 |
| 非写分配 | 写未命中时，直接写主存，不调入 Cache |

常见搭配：

- 写回法常与写分配搭配。
- 写直达常与非写分配搭配。

## 8. Cache 一致性简述

在多级 Cache 或多处理器系统中，可能出现同一数据有多个副本的问题。

如果某个副本被修改，而其他副本没有同步更新，就会产生一致性问题。

408 中通常不深入考查复杂一致性协议，但需要理解写策略会影响 Cache 与主存之间的一致性。

## 9. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 地址划分 | 求标记、组号、块内地址位数 |
| 映射判断 | 判断主存块映射到哪个 Cache 行或组 |
| 命中判断 | 根据标记和有效位判断是否命中 |
| 替换过程 | 按 FIFO 或 LRU 模拟替换 |
| 写策略 | 区分写直达、写回、写分配 |
| 平均访问时间 | 根据命中率计算 AMAT |

## 10. 易错点

- 块内地址位数由块大小决定。
- 组号位数由组数决定，不是由路数决定。
- 标记位数等于总地址位数减去索引字段和块内地址字段。
- 直接映射中不需要选择替换块。
- 全相联没有行号或组号字段。
- 写回法需要脏位。
- 写直达不等于写分配，二者是不同维度。
- Cache 命中率高不代表访问时间一定最短，还要看命中时间和未命中代价。

## 11. 本节小结

Cache 的核心考点是映射和地址划分。

做 Cache 题时建议按下面顺序：

1. 求主存地址位数。
2. 求块内地址位数。
3. 判断映射方式。
4. 求行号或组号位数。
5. 求标记位数。
6. 根据标记和有效位判断命中。
