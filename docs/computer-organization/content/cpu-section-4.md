<h1><center>第四节 控制器的功能和工作原理</center></h1>

## 1. 控制器的作用

控制器是 CPU 的指挥中心。

它根据指令、状态和时序信息产生控制信号，控制数据通路完成指令执行。

可以概括为：

$$
\text{控制器}
\longrightarrow
\text{控制信号}
\longrightarrow
\text{微操作}
$$

## 2. 控制器的输入和输出

控制器输入：

- 指令寄存器 IR 中的操作码
- 时钟和节拍信号
- 状态标志
- 外部中断请求
- 当前执行阶段

控制器输出：

- 寄存器写入控制
- 寄存器输出控制
- ALU 控制信号
- 主存读写信号
- PC 更新控制
- 总线控制信号

| 输入 | 作用 |
| --- | --- |
| 操作码 | 决定执行哪类指令 |
| 时序信号 | 决定微操作发生时刻 |
| 状态标志 | 决定条件转移等操作 |
| 中断请求 | 决定是否进入中断处理 |

## 3. 微操作和微命令

微操作是 CPU 中不可再分的基本操作。

例如：

$$
MAR \leftarrow PC
$$

$$
IR \leftarrow MDR
$$

$$
R1 \leftarrow R2 + R3
$$

微命令是控制器发出的、用于实现微操作的控制信号。

例如，要实现：

$$
R1 \leftarrow R2 + R3
$$

需要微命令控制：

- $R2$ 输出。
- $R3$ 输出。
- ALU 执行加法。
- $R1$ 写入。

## 4. 硬布线控制器

硬布线控制器使用组合逻辑电路直接产生控制信号。

其输入包括：

- 操作码
- 时序信号
- 状态标志

其输出是各类控制信号。

特点：

- 速度快。
- 适合指令格式规整的机器。
- 修改困难。
- 设计复杂度随指令系统复杂度增加而快速上升。

硬布线控制常用于 RISC 风格处理器。

## 5. 微程序控制器

微程序控制器用微程序产生控制信号。

基本思想：

- 一条机器指令由若干微指令解释执行。
- 每条微指令包含一组控制信号。
- 多条微指令组成微程序。
- 微程序存放在控制存储器中。

相关概念：

| 概念 | 含义 |
| --- | --- |
| 微命令 | 一个具体控制信号 |
| 微操作 | 微命令控制下完成的基本操作 |
| 微指令 | 一组微命令的编码 |
| 微程序 | 完成一条机器指令的一段微指令序列 |
| 控制存储器 | 存放微程序的存储器 |

## 6. 微程序控制器工作过程

微程序控制器执行一条机器指令的大致过程：

1. 根据机器指令操作码找到对应微程序入口。
2. 从控制存储器取出微指令。
3. 微指令产生控制信号。
4. 根据微地址形成方式得到下一条微指令地址。
5. 重复执行微指令，直到该机器指令完成。

可以概括为：

$$
\text{机器指令}
\longrightarrow
\text{微程序入口}
\longrightarrow
\text{微指令序列}
\longrightarrow
\text{控制信号}
$$

## 7. 微指令格式

微指令通常包含两部分：

- 操作控制字段
- 顺序控制字段

| 字段 | 作用 |
| --- | --- |
| 操作控制字段 | 指出本周期发出哪些微命令 |
| 顺序控制字段 | 指出下一条微指令地址如何形成 |

微指令可分为水平型和垂直型。

| 类型 | 特点 |
| --- | --- |
| 水平型微指令 | 微命令并行性强，字长较长，执行快 |
| 垂直型微指令 | 编码紧凑，字长较短，译码时间较长 |

## 8. 硬布线控制与微程序控制对比

| 项目 | 硬布线控制 | 微程序控制 |
| --- | --- | --- |
| 控制信号来源 | 组合逻辑电路 | 控制存储器中的微指令 |
| 速度 | 快 | 相对慢 |
| 修改难度 | 难 | 相对容易 |
| 设计复杂度 | 指令复杂时很高 | 结构较规整 |
| 适用场景 | RISC、流水线 | CISC、复杂指令 |

| 考法 | 示例 |

## 10. 易错点

- 微指令不是机器指令，微程序不是机器程序。
- 微命令是控制信号，微操作是控制信号引发的动作。
- 硬布线控制速度快，但修改困难。
- 微程序控制灵活，但通常需要访问控制存储器。
- 水平型微指令并行性强，但微指令字长较长。

## 11. 本节小结

控制器的核心任务是产生控制信号。

硬布线控制直接用逻辑电路产生控制信号，微程序控制通过微指令解释产生控制信号。408 中常考二者概念、特点和对比。
