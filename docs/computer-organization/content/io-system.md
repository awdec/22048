<h1><center>第七章 输入/输出系统</center></h1>

## 本章定位

输入/输出系统负责实现计算机主机与外部设备之间的信息交换。

CPU 和主存内部速度快、格式统一，而外部设备种类多、速度差异大、数据格式也不统一。因此，I/O 系统的核心任务不是简单“连上设备”，而是解决速度匹配、格式转换、状态控制和数据传送方式选择。

按照 2026 年考研 408 计算机组成原理的复习要求，本章重点放在两条主线：

1. I/O 接口如何连接主机和外设。
2. CPU、主存和外设之间的数据传送有哪些方式。

## 本章内容

### 第一节 I/O 接口

主要内容：

- I/O 接口的作用
- I/O 接口的基本结构
- 数据寄存器、状态寄存器和控制寄存器
- I/O 端口
- I/O 端口编址方式
- I/O 接口与总线、CPU、外设之间的关系

学习目标：

- 能说明为什么外设不能直接连接 CPU。
- 能区分数据端口、状态端口和控制端口。
- 能比较统一编址和独立编址。
- 能根据读写方向判断 CPU、接口和外设之间的数据流。

### 第二节 I/O 方式

主要内容：

- 程序查询方式
- 程序中断方式
- DMA 方式
- 三种方式的流程、特点和适用场景
- 中断和 DMA 的关系

学习目标：

- 能比较程序查询、中断和 DMA 对 CPU 的占用程度。
- 能描述中断响应、保护现场、中断服务和中断返回的基本过程。
- 能描述 DMA 传送的基本过程。
- 能判断不同场景适合采用哪种 I/O 方式。

## 学习主线

本章可以按下面的逻辑理解：

$$
\text{外设差异}
\rightarrow
\text{I/O 接口}
\rightarrow
\text{端口编址}
\rightarrow
\text{程序查询}
\rightarrow
\text{中断方式}
\rightarrow
\text{DMA 方式}
$$

一句话概括：

> I/O 接口解决“怎么连”，I/O 方式解决“怎么传”。

## 408 常见考点

| 考点 | 要求 |
| --- | --- |
| I/O 接口功能 | 理解数据缓冲、状态保存、控制命令、格式转换 |
| I/O 端口 | 区分数据端口、状态端口、控制端口 |
| 端口编址 | 比较统一编址和独立编址 |
| 程序查询 | 掌握 CPU 轮询状态位的流程和缺点 |

| 方式比较 | 能比较三种 I/O 方式的 CPU 参与程度和适用场景 |

## 易错点

- I/O 接口不是外设本身，而是主机和外设之间的控制部件。
- I/O 端口不是物理插口，而是 CPU 可访问的接口寄存器地址。
- 统一编址使用访存指令访问 I/O 端口，独立编址使用专门 I/O 指令。
- 中断方式减少 CPU 等待时间，但每次传送仍通常要 CPU 执行中断服务程序。
- DMA 不是不经过总线，而是由 DMA 控制器取得总线控制权。
- DMA 适合高速、大批量数据传送；低速少量数据不一定需要 DMA。
