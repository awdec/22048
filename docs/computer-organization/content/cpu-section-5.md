<h1><center>第五节 异常和中断机制</center></h1>

## 1. 异常和中断的作用

异常和中断机制允许 CPU 暂停当前程序，转去处理特殊事件。

处理完成后，CPU 可以返回原程序继续执行，或终止原程序。

常见事件：

- 外设完成 I/O
- 定时器到时
- 除数为 $0$
- 缺页
- 非法指令
- 系统调用

## 2. 异常和中断的区别

一般理解：

| 类型 | 来源 | 是否同步 | 例子 |
| --- | --- | --- | --- |
| 异常 | CPU 执行指令内部产生 | 同步 | 缺页、除零、非法指令、溢出 |
| 中断 | CPU 外部事件产生 | 异步 | I/O 中断、时钟中断 |

同步是指事件与当前执行的指令有确定关系。

异步是指事件可能在程序执行过程中任意时刻来自外部设备。

## 3. 异常的类型

常见异常：

| 异常 | 含义 |
| --- | --- |
| 故障 | 可恢复，处理后可重新执行相关指令 |
| 自陷 | 有意产生，用于系统调用或调试 |
| 终止 | 严重错误，通常无法恢复 |

例如缺页异常通常属于可恢复异常，操作系统调入页面后可重新执行引发缺页的指令。

## 4. 中断的分类

中断可以按不同方式分类。

按是否可屏蔽：

| 类型 | 含义 |
| --- | --- |
| 可屏蔽中断 | 可由中断允许位控制是否响应 |
| 不可屏蔽中断 | 通常用于严重事件，不能普通屏蔽 |

按来源：

- I/O 中断
- 时钟中断
- 电源故障中断
- 外部设备请求

## 5. 中断响应条件

CPU 响应中断通常需要满足：

1. 有中断请求。
2. CPU 当前允许中断。
3. 当前指令执行结束。
4. 中断优先级满足要求。

大多数中断在一条指令执行结束后响应。

这样可以保证当前指令执行状态相对完整，便于保存断点和恢复程序。

## 6. 中断处理过程

典型中断处理流程：

1. 关中断或调整中断允许位。
2. 保存断点。
3. 保存程序状态字。
4. 识别中断源。
5. 根据中断向量找到中断服务程序入口。
6. 执行中断服务程序。
7. 恢复现场。
8. 开中断。
9. 返回原程序。

可以概括为：

$$
\text{请求}
\longrightarrow
\text{响应}
\longrightarrow
\text{保存现场}
\longrightarrow
\text{执行服务程序}
\longrightarrow
\text{恢复现场}
\longrightarrow
\text{返回}
$$

## 7. 断点和现场

断点是被中断程序将要继续执行的位置，通常由 PC 保存。

现场是程序继续执行所需的状态，包括：

- 通用寄存器内容
- PC
- PSW
- 栈指针
- 必要的控制状态寄存器

保存现场通常由硬件和软件共同完成。

硬件常自动保存 PC 和 PSW，软件中断服务程序保存通用寄存器。

## 8. 中断向量

中断向量用于指出中断服务程序入口地址。

中断向量表保存各类中断对应的服务程序入口。

处理过程：

1. CPU 得到中断类型号。
2. 根据类型号查中断向量表。
3. 得到服务程序入口地址。
4. 将入口地址送入 PC。

$$
PC \leftarrow \text{中断服务程序入口}
$$

## 9. 中断优先级和嵌套

当多个中断同时发生时，需要按优先级响应。

高优先级中断可以优先得到处理。

中断嵌套是指 CPU 正在处理中断时，又响应更高优先级中断。

要支持中断嵌套，需要：

- 保存当前中断服务程序现场。
- 设置优先级屏蔽。
- 正确恢复多层现场。

## 10. 异常、中断和 DMA

DMA 传送数据时，主要由 DMA 控制器完成数据搬运。

CPU 通常在 DMA 传输结束后通过中断得知传输完成。

区别：

| 机制 | 主要作用 |
| --- | --- |
| 异常 | 处理 CPU 内部执行指令时的特殊情况 |
| 中断 | 响应外部事件 |
| DMA | 在外设和主存之间直接传送数据 |

DMA 不是中断，但 DMA 完成后常用中断通知 CPU。

## 11. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 概念区分 | 区分异常、中断、DMA |

| 响应条件 | 判断何时能响应中断 |
| 中断向量 | 根据中断类型号查服务程序入口 |
| 嵌套中断 | 判断优先级和屏蔽关系 |

## 12. 易错点

- 异常通常来自 CPU 内部，中断通常来自 CPU 外部。
- 中断一般在指令执行结束后响应。
- 保存断点通常保存 PC，保存现场不只保存 PC。
- 中断向量不是中断服务程序本身，而是入口地址或相关信息。
- DMA 传输过程不由 CPU 搬运每个数据，但完成后可能产生中断。

## 13. 本节小结

异常和中断让 CPU 能够处理正常顺序执行之外的事件。
