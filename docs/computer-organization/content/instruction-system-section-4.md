<h1><center>第四节 CISC 和 RISC 的基本概念</center></h1>

## 1. 指令系统设计思想

不同计算机可以采用不同指令系统设计思想。

常见两类：

- CISC
- RISC

CISC 强调提供功能丰富、复杂的指令。

RISC 强调指令简单、格式规整、便于流水线执行。

## 2. CISC

CISC 是 Complex Instruction Set Computer 的缩写，称为复杂指令系统计算机。

CISC 的基本思想是：

$$
\text{用功能较强的复杂指令完成较多工作}
$$

CISC 特点：

- 指令数量多。
- 指令功能复杂。
- 指令长度可能不固定。
- 寻址方式多。
- 一条指令可能完成较复杂操作。
- 指令执行周期差异较大。
- 硬件译码和控制较复杂。

早期计算机存储资源昂贵，复杂指令有利于减少程序指令条数和程序存储空间。

## 3. RISC

RISC 是 Reduced Instruction Set Computer 的缩写，称为精简指令系统计算机。

RISC 的基本思想是：

$$
\text{用简单、规整、高频的指令构造程序}
$$

RISC 特点：

- 指令数量相对少。
- 指令格式规整。
- 指令长度通常固定。
- 寻址方式较少。
- 大多数指令在一个或少数几个时钟周期内完成。
- 通用寄存器数量较多。
- 访存通常由专门的 load/store 指令完成。
- 便于流水线设计。

RISC 不表示指令条数绝对很少，而是强调指令功能简单、格式规整和执行效率高。

## 4. Load/Store 结构

RISC 常采用 load/store 结构。

其基本规则是：

- 只有 load 指令可以从主存读入寄存器。
- 只有 store 指令可以把寄存器内容写入主存。
- 算术逻辑运算通常只在寄存器之间进行。

例如：

```txt
LOAD  R1, x
LOAD  R2, y
ADD   R3, R1, R2
STORE z, R3
```

这种结构使指令格式更规整，也有利于流水线实现。

## 5. CISC 与 RISC 对比

| 项目 | CISC | RISC |
| --- | --- | --- |
| 指令数量 | 多 | 较少 |
| 指令功能 | 复杂 | 简单 |
| 指令长度 | 常可变 | 常固定 |
| 寻址方式 | 多 | 少 |
| 执行周期 | 差异较大 | 较规整 |
| 控制器实现 | 常较复杂 | 相对简单 |
| 通用寄存器 | 相对较少 | 通常较多 |
| 访存方式 | 多种指令可访存 | 通常 load/store 访存 |
| 流水线 | 较难规整实现 | 更适合流水线 |
| 编译器作用 | 相对较弱 | 更依赖编译器优化 |

## 6. RISC 的优点

RISC 的优点主要来自指令简单和格式规整。

常见优点：

- 译码简单。
- 控制逻辑简单。
- 更容易实现流水线。
- 指令执行时间更稳定。
- 编译器更容易进行优化。
- 硬件可以把更多资源用于寄存器和流水线。

RISC 的代价是复杂操作可能需要多条简单指令完成，因此程序指令条数可能增加。

## 7. CISC 的优点

CISC 的优点主要来自指令功能强。

常见优点：

- 一条复杂指令可以完成较多工作。
- 程序指令条数可能较少。
- 机器代码可能更紧凑。
- 对早期存储容量有限的系统较友好。

CISC 的代价是指令译码复杂、执行时间不规整，流水线实现难度较高。

## 8. 微程序控制与 CISC

CISC 中复杂指令较多，常使用微程序控制实现复杂指令功能。

微程序控制的基本思想是：

- 一条机器指令由若干微指令解释执行。
- 微指令存放在控制存储器中。
- 修改微程序可以较灵活地实现复杂控制。

RISC 更倾向于使用较简单、规整的硬布线控制，以提高速度和流水线效率。

这不是绝对划分，但可以作为 408 中理解两类风格的常见角度。

## 9. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 概念判断 | 判断 CISC 和 RISC 的基本特点 |
| 指令格式 | 判断哪类更倾向定长指令 |
| 寻址方式 | 判断哪类寻址方式更多 |
| 流水线 | 判断哪类更适合流水线 |
| 访存方式 | 判断 load/store 结构属于哪类 |
| 控制器 | 判断复杂控制和微程序控制的关系 |

## 10. 易错点

- RISC 不是“没有复杂功能”，而是用简单指令组合实现复杂功能。
- CISC 不是一定比 RISC 慢，实际性能还取决于实现、编译器和程序特征。
- RISC 指令条数可能更多，但单条指令执行更规整。
- CISC 指令功能强，但译码和流水线控制更复杂。
- load/store 结构是 RISC 的典型特点。
- “精简”主要指指令功能和格式规整，不是单纯指令总数越少越好。

## 11. 本节小结

CISC 和 RISC 是两种不同的指令系统设计取向。

CISC 倾向于用复杂指令缩短程序；RISC 倾向于用简单规整指令提升流水线和硬件执行效率。408 中重点是会比较二者特点，而不是背具体处理器型号。
