<h1><center>第一节 存储器概述</center></h1>

## 1. 存储器的作用

存储器用于存放程序和数据，是计算机系统的重要组成部分。

在冯·诺依曼计算机中，程序和数据都以二进制形式存放在存储器中，CPU 通过取指和访存操作访问它们。

存储器需要解决的核心矛盾是：

$$
\text{容量}
\quad
\text{速度}
\quad
\text{成本}
$$

容量越大，通常速度越慢、成本越低；速度越快，通常容量越小、成本越高。因此现代计算机采用多级存储体系。

## 2. 存储器分类

### 2.1 按存储介质分类

| 类型 | 特点 | 例子 |
| --- | --- | --- |
| 半导体存储器 | 速度快，常用于主存和 Cache | SRAM、DRAM、Flash |
| 磁表面存储器 | 容量大，成本低 | 磁盘 |
| 光存储器 | 利用光学方式读写 | 光盘 |

### 2.2 按存取方式分类

| 类型 | 访问特点 | 例子 |
| --- | --- | --- |
| 随机存取存储器 | 任意单元访问时间基本相同 | RAM |
| 顺序存取存储器 | 必须按顺序访问 | 磁带 |
| 直接存取存储器 | 先定位到附近区域，再顺序访问 | 磁盘 |
| 相联存储器 | 按内容访问 | Cache 标记匹配、TLB |

随机存取不等于随机存放，它强调的是任意地址访问时间基本相同。

### 2.3 按信息可保存性分类

| 类型 | 断电后信息是否保留 | 例子 |
| --- | --- | --- |
| 易失性存储器 | 断电后信息丢失 | SRAM、DRAM |
| 非易失性存储器 | 断电后信息仍可保留 | ROM、Flash、磁盘 |

### 2.4 按在系统中的作用分类

| 层次 | 作用 |
| --- | --- |
| 寄存器 | CPU 内部临时保存操作数、地址和状态 |
| Cache | 缓解 CPU 与主存速度差异 |
| 主存 | 存放当前正在运行的程序和数据 |
| 外存 | 长期保存大量程序和数据 |

## 3. 存储系统层次结构

现代计算机通常采用层次化存储系统：

$$
\text{寄存器}
\longrightarrow
\text{Cache}
\longrightarrow
\text{主存}
\longrightarrow
\text{外存}
$$

从左到右：

- 容量逐渐增大。
- 速度逐渐降低。
- 单位容量成本逐渐降低。
- CPU 访问频率逐渐降低。

| 层次 | 速度 | 容量 | 成本 |
| --- | --- | --- | --- |
| 寄存器 | 最快 | 最小 | 最高 |
| Cache | 很快 | 较小 | 较高 |
| 主存 | 中等 | 较大 | 中等 |
| 外存 | 慢 | 最大 | 最低 |

存储层次的目标是让系统看起来同时具有：

- 接近高速存储器的访问速度。
- 接近低速存储器的大容量。
- 接近低速存储器的低成本。

## 4. 局部性原理

存储层次能够有效工作，依赖程序访问的局部性原理。

局部性分为时间局部性和空间局部性。

| 类型 | 含义 | 例子 |
| --- | --- | --- |
| 时间局部性 | 最近访问过的数据或指令，将来可能再次访问 | 循环变量、多次调用同一段代码 |
| 空间局部性 | 访问某个地址后，附近地址也可能被访问 | 顺序扫描数组、顺序取指 |

Cache 利用局部性，把主存中最近或相邻的数据块复制到高速缓存中。

虚拟存储器也利用局部性，只把当前常用页面调入主存。

## 5. 存储器性能指标

### 5.1 存储容量

存储容量表示存储器能够保存的信息量。

若存储器有 $N$ 个存储单元，每个存储单元有 $W$ 位，则容量为：

$$
N \times W\text{ bit}
$$

若地址线有 $n$ 根，则最多可以寻址：

$$
2^n
$$

个不同地址。

若按字节编址，则 $n$ 根地址线可寻址容量为：

$$
2^n\text{ B}
$$

### 5.2 存取时间

存取时间是从发出读写请求到完成读写所需的时间。

读操作中，存取时间通常指从给出地址到数据稳定输出的时间。

写操作中，存取时间通常指从给出地址和数据到写入完成的时间。

### 5.3 存取周期

存取周期是连续两次独立访问存储器之间所需的最小时间间隔。

一般有：

$$
\text{存取周期} \geq \text{存取时间}
$$

因为一次访问结束后，存储器可能还需要恢复或刷新等额外时间。

### 5.4 存储器带宽

存储器带宽表示单位时间内能够传输的数据量。

$$
\text{带宽}
=
\frac{\text{传输数据量}}{\text{传输时间}}
$$

若每个存取周期传输 $W$ 位，存取周期为 $T$，则带宽为：

$$
\frac{W}{T}
$$

## 6. 地址、存储单元和编址

存储器由许多存储单元组成，每个存储单元都有唯一地址。

常见编址方式：

- 按字节编址
- 按字编址

按字节编址时，每个地址对应 $1$ 个字节。

按字编址时，每个地址对应 $1$ 个机器字或存储字。

408 中题目通常会明确说明按字节编址还是按字编址。若没有说明，现代计算机一般默认按字节编址。

## 7. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 存储层次判断 | 判断寄存器、Cache、主存、外存速度和容量关系 |
| 局部性分析 | 判断某段程序体现时间局部性还是空间局部性 |
| 容量计算 | 根据地址线和数据线计算可寻址空间 |
| 性能指标 | 区分存取时间、存取周期和带宽 |
| 存取方式 | 区分随机存取、顺序存取、直接存取和相联存取 |

## 8. 易错点

- 存取时间不一定等于存取周期。
- 随机存取强调访问时间与地址无关，不是随机选择地址。
- 地址线根数决定可寻址单元个数，数据线根数决定一次可传输的数据位数。
- 按字节编址和按字编址会影响容量计算。
- Cache 和虚拟存储器都利用局部性，但解决的问题不同。

## 9. 本节小结

存储系统的核心思想是用层次结构平衡速度、容量和成本。

本节最重要的是建立整体框架：CPU 直接频繁访问高速小容量存储器，低速大容量存储器负责保存更多内容，而局部性原理让这种层次结构能够有效工作。
