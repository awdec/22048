<h1><center>第一节 CPU 的功能和基本结构</center></h1>

## 1. CPU 的作用

CPU 是 Central Processing Unit 的缩写，即中央处理器。

CPU 是计算机系统的核心部件，负责解释并执行指令，控制其他部件协调工作。

从功能上看，CPU 主要完成：

- 取指令
- 分析指令
- 执行指令
- 处理中断和异常
- 控制数据在各部件之间传送

## 2. CPU 的基本功能

CPU 的基本功能可以概括为四类。

| 功能 | 含义 |
| --- | --- |
| 指令控制 | 控制程序中指令的执行顺序 |
| 操作控制 | 产生完成每条指令所需的控制信号 |
| 时间控制 | 控制各微操作在合适时刻发生 |
| 数据加工 | 对数据进行算术运算和逻辑运算 |

此外，现代 CPU 还需要支持异常、中断、特权级、虚拟存储和多任务切换等机制。

## 3. CPU 的基本组成

CPU 通常由运算器、控制器和寄存器组组成。

$$
\text{CPU}
=
\text{运算器}
+
\text{控制器}
+
\text{寄存器组}
$$

| 部件 | 主要作用 |
| --- | --- |
| 运算器 | 完成算术运算和逻辑运算 |
| 控制器 | 产生控制信号，协调各部件工作 |
| 寄存器组 | 暂存指令、地址、数据和状态信息 |

CPU 内部还包含连接这些部件的数据通路，例如内部总线、多路选择器和临时寄存器等。

## 4. 运算器

运算器负责数据加工，是执行算术逻辑运算的核心。

运算器通常包括：

- ALU
- 移位器
- 通用寄存器
- 暂存寄存器
- 状态标志产生电路

ALU 可以完成：

- 加法
- 减法
- 与
- 或
- 非
- 异或
- 比较

运算器执行操作后，通常会产生状态标志，例如：

- 零标志 ZF
- 符号标志 SF
- 进位标志 CF
- 溢出标志 OF

这些标志会影响条件转移指令和异常判断。

## 5. 控制器

控制器负责解释指令并产生控制信号。

控制器的主要任务：

1. 根据 PC 取出指令。
2. 对 IR 中的指令进行译码。
3. 根据操作码、寻址方式、状态标志和时序信号产生控制信号。
4. 控制寄存器、ALU、主存和 I/O 等部件协调工作。

控制器的输入通常包括：

- 指令操作码
- 时钟和节拍信号
- 状态标志
- 中断请求
- 总线请求

控制器的输出是各种控制信号，例如：

- 寄存器写使能
- ALU 操作控制
- 主存读写控制
- PC 更新控制
- 总线选择控制

## 6. CPU 内部寄存器

CPU 内部寄存器用于保存临时数据、地址、指令和状态信息。

常见寄存器如下。

| 寄存器 | 名称 | 作用 |
| --- | --- | --- |
| PC | 程序计数器 | 保存下一条指令地址 |
| IR | 指令寄存器 | 保存当前正在执行的指令 |
| MAR | 存储器地址寄存器 | 保存要访问的主存地址 |
| MDR | 存储器数据寄存器 | 保存从主存读出或将写入主存的数据 |
| ACC | 累加器 | 保存运算操作数或结果 |
| GPR | 通用寄存器 | 保存操作数、地址或中间结果 |
| PSW | 程序状态字 | 保存状态标志和控制信息 |

::: warning 注意
PC 中通常是下一条指令地址，IR 中是当前指令。MAR 面向地址，MDR 面向数据，二者不要混淆。
:::

## 7. 用户可见寄存器

用户可见寄存器是机器级程序可以直接或间接使用的寄存器。

常见用户可见寄存器：

- 通用寄存器
- 数据寄存器
- 地址寄存器
- 条件码寄存器
- 栈指针寄存器

通用寄存器可以减少访存次数，提高程序执行速度。

例如，计算：

```c
z = x + y;
```

机器级代码可能先把 `x` 和 `y` 装入寄存器，再由 ALU 相加，最后把结果写回主存。

## 8. 控制和状态寄存器

控制和状态寄存器主要供 CPU 和操作系统使用。

常见控制状态寄存器：

- PC
- IR
- MAR
- MDR
- PSW
- 中断相关寄存器
- 页表基址寄存器

其中 PSW 通常包含：

- 条件码
- 中断允许位
- CPU 工作模式
- 优先级信息

PSW 对异常、中断和操作系统切换非常重要。

## 9. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 部件功能 | 判断 ALU、控制器、寄存器的作用 |
| 寄存器区分 | 区分 PC、IR、MAR、MDR |
| 状态标志 | 判断 ZF、CF、OF 等标志含义 |
| CPU 功能 | 判断指令控制、操作控制、时间控制、数据加工 |
| 用户可见性 | 判断哪些寄存器可被程序直接使用 |

## 10. 易错点

- 运算器负责运算，控制器负责发控制信号。
- PC 保存下一条指令地址，不保存当前指令内容。
- IR 保存当前指令，不保存主存地址。
- MAR 保存主存地址，MDR 保存主存数据。
- PSW 不只是标志位，还可能包含中断允许、工作模式等控制信息。

## 11. 本节小结

CPU 的基本结构可以用“运算器 + 控制器 + 寄存器组”理解。

运算器负责算，控制器负责控，寄存器负责暂存。后续的数据通路和控制器章节，就是进一步说明这些部件如何协同完成指令执行。
