<h1><center>第三节 主存储器与 CPU 的连接</center></h1>

## 1. 连接目标

主存储器与 CPU 连接的目标是让 CPU 能够按照地址正确读写主存中的数据。

CPU 与主存之间通常通过三类总线连接：

- 地址总线
- 数据总线
- 控制总线

| 总线 | 作用 |
| --- | --- |
| 地址总线 | CPU 向主存发送要访问的地址 |
| 数据总线 | CPU 与主存之间传送数据 |
| 控制总线 | 传送读、写、片选等控制信号 |

## 2. 地址线和数据线

若 CPU 有 $n$ 根地址线，则最多可以产生：

$$
2^n
$$

个不同地址。

若按字节编址，则最大寻址空间为：

$$
2^n\text{ B}
$$

若数据总线宽度为 $m$ 位，则 CPU 一次可以和主存交换：

$$
m\text{ bit}
$$

的数据。

地址线决定“能找到多少位置”，数据线决定“一次能传多少位”。

## 3. 存储芯片连接基本信号

常见存储芯片通常包含：

- 地址输入端
- 数据输入输出端
- 片选信号
- 读控制信号
- 写控制信号

| 信号 | 含义 |
| --- | --- |
| 地址信号 | 选择芯片内部某个存储单元 |
| 数据信号 | 读出或写入数据 |
| 片选信号 | 决定当前芯片是否被选中 |
| 读信号 | 控制芯片输出数据 |
| 写信号 | 控制芯片写入数据 |

只有片选有效，并且读写控制信号有效时，存储芯片才会参与访问。

## 4. 位扩展

位扩展用于增加存储字长。

当单片存储芯片的数据位数小于 CPU 数据总线宽度时，需要多片芯片并联提供更多数据位。

例如，CPU 数据总线宽度为 $8$ 位，但芯片规格为：

$$
1K \times 4
$$

则需要 $2$ 片芯片并联，组成：

$$
1K \times 8
$$

位扩展特点：

- 多片芯片使用相同地址线。
- 多片芯片使用相同片选信号。
- 每片芯片连接数据总线的一部分。

## 5. 字扩展

字扩展用于增加存储单元个数，也就是扩大地址范围。

当单片存储芯片的字数不足时，需要多片芯片分别对应不同地址范围。

例如，需要组成：

$$
4K \times 8
$$

而单片芯片规格为：

$$
1K \times 8
$$

则需要 $4$ 片芯片。

字扩展特点：

- 各芯片数据线并接到数据总线。
- 各芯片低位地址线相同，用于片内寻址。
- 高位地址线经译码后产生不同片选信号。

## 6. 字位扩展

字位扩展同时增加存储单元个数和存储字长。

例如，需要组成：

$$
4K \times 8
$$

而单片芯片为：

$$
1K \times 4
$$

则：

- 字数方向需要 $4$ 组。
- 位数方向每组需要 $2$ 片。

所需芯片总数为：

$$
4 \times 2 = 8
$$

一般计算公式：

$$
\text{芯片数}
=
\frac{\text{目标字数}}{\text{单片字数}}
\times
\frac{\text{目标字长}}{\text{单片字长}}
$$

## 7. 地址译码与片选

CPU 发出的地址通常分为两部分：

- 低位地址：用于芯片内部寻址。
- 高位地址：用于片选。

若单片芯片有 $2^k$ 个存储单元，则需要 $k$ 根地址线进行片内寻址。

其余高位地址线可以通过译码器产生片选信号。

常见译码方式：

| 方式 | 特点 |
| --- | --- |
| 全译码 | 所有高位地址线都参与译码，地址唯一 |
| 部分译码 | 只有部分高位地址线参与译码，可能出现地址重叠 |
| 线选法 | 直接用某些地址线作为片选，电路简单但地址利用不充分 |

408 中通常重点掌握全译码和地址范围分析。

## 8. CPU 读主存过程

CPU 读主存的一般过程：

1. CPU 将要访问的地址送到地址总线。
2. 地址译码器选择对应存储单元或存储芯片。
3. CPU 发出读控制信号。
4. 主存将数据送到数据总线。
5. CPU 接收数据。

可以概括为：

$$
\text{给地址}
\longrightarrow
\text{片选}
\longrightarrow
\text{读控制}
\longrightarrow
\text{数据输出}
$$

## 9. CPU 写主存过程

CPU 写主存的一般过程：

1. CPU 将要访问的地址送到地址总线。
2. CPU 将要写入的数据送到数据总线。
3. 地址译码器选择对应存储单元或存储芯片。
4. CPU 发出写控制信号。
5. 主存把数据写入指定单元。

可以概括为：

$$
\text{给地址}
\longrightarrow
\text{给数据}
\longrightarrow
\text{片选}
\longrightarrow
\text{写控制}
\longrightarrow
\text{写入}
$$

## 10. 地址范围计算

若某芯片有 $k$ 根片内地址线，则每片容量为：

$$
2^k
$$

个地址单元。

若某片的起始地址为 $A$，容量为 $C$ 个地址单元，则结束地址为：

$$
A + C - 1
$$

做地址范围题时，建议按下面步骤：

1. 根据芯片容量确定片内地址线根数。
2. 找出参与片选的高位地址。
3. 根据片选条件确定起始地址。
4. 根据容量确定结束地址。

## 11. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 芯片数计算 | 根据目标容量和芯片规格求所需芯片数 |
| 位扩展 | 判断多片芯片如何连接数据线 |
| 字扩展 | 判断高位地址如何产生片选 |
| 地址译码 | 根据片选条件求地址范围 |
| 读写过程 | 判断地址线、数据线、控制线的作用 |

## 12. 易错点

- 地址线根数决定地址空间大小，不决定一次传输多少位。
- 数据线宽度决定一次传输位数，不决定能访问多少地址。
- 位扩展时多片芯片同时工作。
- 字扩展时通常一次只选中一组芯片。
- 片内地址线用于选择芯片内部单元，片选地址线用于选择芯片。
- 结束地址要用起始地址加容量再减 $1$。

## 13. 本节小结

主存与 CPU 的连接题本质上是地址线、数据线和控制线的配合问题。

看到芯片扩展题，先判断是扩展字长、扩展字数，还是二者同时扩展；看到地址范围题，先分清片内地址和片选地址。
