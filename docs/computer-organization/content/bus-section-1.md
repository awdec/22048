<h1><center>第一节 总线概述</center></h1>

## 1. 总线的基本概念

总线是一组能够为多个部件分时共享的信息传输线路。

在计算机系统中，CPU、主存、I/O 接口等部件不可能两两都直接连线，否则连线数量会迅速增加，系统结构也会变得复杂。总线把多个部件连接到同一组公共线路上，通过分时使用完成信息交换。

总线的本质特点：

- 共享性：多个部件共用同一组线路。
- 分时性：同一时刻通常只能有一个主设备使用总线传输。
- 规范性：连接总线的设备必须遵守统一的电气、时序和协议规则。

## 2. 总线中的主设备和从设备

一次总线传输通常由主设备发起，由从设备响应。

| 角色 | 含义 | 例子 |
| --- | --- | --- |
| 主设备 | 能申请并控制总线的设备 | CPU、DMA 控制器 |
| 从设备 | 被主设备访问或控制的设备 | 主存、I/O 接口 |

例如 CPU 从主存取指令时：

- CPU 是主设备。
- 主存是从设备。
- CPU 给出地址和读命令。
- 主存把指定单元的数据送到数据总线上。

DMA 传送时，DMA 控制器可以暂时成为总线主设备，直接控制主存和 I/O 设备之间的数据传送。

## 3. 总线的组成

系统总线通常由三类信号线组成：

| 组成 | 作用 | 典型信息 |
| --- | --- | --- |
| 数据总线 | 传送数据 | 指令、操作数、I/O 数据 |
| 地址总线 | 指明访问对象的位置 | 主存地址、I/O 端口地址 |
| 控制总线 | 传送控制和状态信号 | 读、写、中断、总线请求、总线响应 |

### 数据总线

数据总线用于传送真正的数据内容。

数据总线通常是双向的，因为 CPU 既要从主存或 I/O 设备读数据，也要向它们写数据。

数据总线宽度决定一次总线传输最多能传送多少位数据。

例如：

- 32 位数据总线一次最多传送 32 位，即 4 字节。
- 64 位数据总线一次最多传送 64 位，即 8 字节。

### 地址总线

地址总线用于指出数据传输的目标位置。

地址总线通常由主设备发出，因此在传统系统中常看作单向总线。

地址总线宽度决定最大可寻址空间。

若地址总线有 $n$ 位，则最多可形成：

$$
2^n
$$

个不同地址。

若按字节编址，则最大可寻址空间为：

$$
2^n \text{ B}
$$

例如 32 位地址总线按字节编址，最大可寻址空间为：

$$
2^{32}\text{ B} = 4\text{ GB}
$$

### 控制总线

控制总线用于传送控制命令和状态信息。

常见控制信号包括：

- 存储器读
- 存储器写
- I/O 读
- I/O 写
- 中断请求
- 中断响应
- 总线请求
- 总线允许
- 时钟
- 复位
- 就绪或等待

控制总线中的信号方向不一定相同，有些由 CPU 发出，有些由外部设备返回。

## 4. 总线的分类

### 按连接部件的位置分类

| 类型 | 作用 | 特点 |
| --- | --- | --- |
| 片内总线 | 连接 CPU 芯片内部部件 | 距离短、速度快 |
| 系统总线 | 连接 CPU、主存、I/O 接口 | 408 中最重要 |
| 通信总线 | 连接计算机系统与外部设备或其他系统 | 距离较长、协议复杂 |

片内总线通常连接寄存器、ALU、控制器等内部部件。

系统总线是本章重点，通常由地址总线、数据总线和控制总线组成。

通信总线用于系统外部通信，例如外设接口和计算机之间的数据交换。

### 按传输信息类型分类

| 类型 | 作用 |
| --- | --- |
| 地址总线 | 传送地址 |
| 数据总线 | 传送数据 |
| 控制总线 | 传送控制命令和状态信号 |

408 题目经常考查三类总线的作用，不要把数据总线宽度和地址总线宽度混淆。

### 按数据传输方式分类

| 类型 | 特点 |
| --- | --- |
| 并行总线 | 多位数据同时传输 |
| 串行总线 | 数据按位顺序传输 |

并行总线在短距离传输中效率较高，但线路多、成本高、长距离时容易受时钟偏斜影响。

串行总线线路少、抗干扰能力较强，现代高速外部接口大量采用串行传输。

### 按时序控制方式分类

| 类型 | 特点 |
| --- | --- |
| 同步总线 | 各部件按统一时钟工作 |
| 异步总线 | 通过请求和应答信号协调 |

同步总线控制简单，但对设备速度一致性要求较高。

异步总线适合连接速度差异较大的设备，但控制逻辑更复杂。

## 5. 总线结构

常见总线结构包括单总线结构、双总线结构和多总线结构。

### 单总线结构

所有部件都连接在一条系统总线上。

优点：

- 结构简单。
- 成本低。
- 易于扩展。

缺点：

- 所有传输共用同一条总线。
- 容易形成总线瓶颈。
- 总线负载较重。

### 双总线结构

常见形式是把存储总线和 I/O 总线分开。

优点：

- 减轻单条总线负载。
- CPU 和主存之间的数据传送可以更快。
- I/O 设备对主存访问的影响减小。

缺点：

- 结构比单总线复杂。
- 需要桥接或接口逻辑协调不同总线。

### 多总线结构

现代计算机常采用多层次总线或互连结构。

例如：

- CPU 与 Cache 或内存控制器之间使用高速总线。
- 主存和高速外设使用高速系统总线。
- 低速外设挂接在低速 I/O 总线上。

多总线结构可以兼顾高速部件和低速部件，但控制和协议更复杂。

## 6. 总线性能指标

### 总线宽度

总线宽度通常指数据总线的位数。

总线宽度越大，一次传输的数据量越大。

例如：

- 32 位数据总线一次传送 4 字节。
- 64 位数据总线一次传送 8 字节。

### 总线频率

总线频率表示总线时钟每秒振荡次数。

若总线频率为 $f$，则一个总线时钟周期为：

$$
T = \frac{1}{f}
$$

### 总线周期

总线周期是完成一次总线操作所需的时间。

一次总线操作可以是：

- 读一次主存
- 写一次主存
- 读一次 I/O 端口
- 写一次 I/O 端口

一个总线周期通常包含若干个时钟周期。

### 总线带宽

总线带宽表示单位时间内总线能够传输的数据量。

基本公式：

$$
B = \frac{W}{T}
$$

其中：

- $B$ 表示总线带宽。
- $W$ 表示一次总线传送的数据量。
- $T$ 表示完成一次传送所需时间。

若每个时钟周期传送 $k$ 次，每次传送 $W$ 字节，总线频率为 $f$，则：

$$
B = W \times f \times k
$$

注意：实际题目中还要看是否存在地址周期、等待周期、仲裁周期和协议开销。

## 7. 总线标准

总线标准规定了不同设备如何连接和通信。

一个完整的总线标准通常包括：

| 标准内容 | 含义 |
| --- | --- |
| 机械特性 | 插头、插槽、尺寸、引脚排列 |
| 电气特性 | 电平、负载能力、信号方向 |
| 功能特性 | 每根信号线的作用 |
| 过程特性 | 传输时序、握手方式、仲裁规则 |

408 一般不要求记忆大量具体总线标准细节，更重要的是理解总线标准为什么能让不同设备协同工作。

## 8. 总线仲裁

由于总线具有共享性，同一时刻不能让多个主设备同时驱动总线，否则会发生冲突。

总线仲裁用于决定多个主设备同时请求总线时，谁获得总线使用权。

可能请求总线的主设备包括：

- CPU
- DMA 控制器
- 具有总线主控能力的 I/O 控制器

### 集中式仲裁

集中式仲裁由一个总线控制器统一决定总线使用权。

常见方式：

| 方式 | 思路 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 链式查询 | 总线允许信号按设备顺序串行传递 | 结构简单 | 优先级固定，靠后的设备等待时间长 |
| 计数器查询 | 控制器用计数值轮询设备 | 优先级可调整 | 查询速度较慢 |
| 独立请求 | 每个设备有独立请求线和允许线 | 响应快 | 线路多，控制复杂 |

#### 链式查询方式

链式查询也叫菊花链方式。

设备按固定顺序连接，总线允许信号从高优先级设备向低优先级设备传递。

特点：

- 离总线控制器越近，优先级越高。
- 控制线少，结构简单。
- 一个设备故障可能影响后续设备。
- 低优先级设备可能长期得不到总线。

#### 计数器查询方式

总线控制器通过计数器产生设备编号，依次查询各设备是否请求总线。

特点：

- 可以通过改变计数初值调整优先级。
- 不会像链式查询那样完全固定。
- 查询过程需要时间。

#### 独立请求方式

每个主设备都有独立的总线请求线和总线允许线。

特点：

- 响应速度快。
- 可以灵活设置优先级。
- 控制线数量多。
- 硬件复杂度高。

### 分布式仲裁

分布式仲裁没有单独的集中控制器，各主设备通过某种规则共同决定谁获得总线。

特点：

- 不依赖单一仲裁器。
- 系统可靠性较高。
- 控制协议较复杂。

408 中更常考集中式仲裁，尤其是链式查询、计数器查询和独立请求方式的比较。

| 总线宽度 | 知道通常指数据总线宽度 |

## 10. 易错提醒

- 地址总线宽度决定地址范围，不决定一次传输多少数据。
- 数据总线宽度决定一次传输数据位数，不直接决定主存容量。
- 控制总线不是单向的，其中不同控制信号方向可能不同。
- 总线频率高不代表实际带宽一定高，还要看每次传输数据量和协议开销。
- 链式查询硬件简单，但优先级固定。
- 独立请求响应快，但控制线多。

## 11. 小结

总线概述部分要抓住三个关键词：共享、仲裁、带宽。

共享决定了总线需要仲裁，仲裁决定谁能使用总线，带宽决定总线的数据传输能力。408 中常把总线组成、总线分类、仲裁方式和带宽计算结合起来考查。
