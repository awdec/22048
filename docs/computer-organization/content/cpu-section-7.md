<h1><center>第七节 多处理器的基本概念</center></h1>

## 1. 多处理器的作用

单个 CPU 的性能提升会受到主频、功耗、散热和指令级并行度等限制。

多处理器系统通过多个处理单元并行工作，提高系统吞吐率和并行处理能力。

多处理器相关概念包括：

- 多核处理器
- 对称多处理器
- 多线程处理器
- SIMD
- MIMD
- 共享内存
- 分布式内存

## 2. Flynn 分类

Flynn 分类根据指令流和数据流划分计算机系统。

| 类型 | 含义 | 例子 |
| --- | --- | --- |
| SISD | 单指令流单数据流 | 传统单处理器 |
| SIMD | 单指令流多数据流 | 向量处理器、GPU 中的部分执行模式 |
| MISD | 多指令流单数据流 | 实际中少见 |
| MIMD | 多指令流多数据流 | 多核处理器、多处理器系统 |

408 中重点理解 SISD、SIMD 和 MIMD。

## 3. SIMD

SIMD 是 Single Instruction Multiple Data。

它用同一条指令同时处理多个数据元素。

适合：

- 向量运算
- 图像处理
- 矩阵运算
- 多媒体处理

例如，对两个数组逐元素相加：

```txt
C[i] = A[i] + B[i]
```

SIMD 可以让多个元素并行执行同一种加法操作。

特点：

- 控制简单。
- 数据并行性强。
- 不适合控制流差异很大的任务。

## 4. MIMD

MIMD 是 Multiple Instruction Multiple Data。

多个处理器可以执行不同指令，处理不同数据。

常见 MIMD 系统：

- 多核 CPU
- 对称多处理器
- 分布式集群

特点：

- 灵活性强。
- 适合任务并行。
- 硬件和软件管理复杂。

## 5. 多核处理器

多核处理器是在一个芯片上集成多个处理核心。

每个核心可以独立执行指令。

多核处理器通常共享：

- 末级 Cache
- 内存控制器
- 主存
- I/O 接口

也可能每个核心私有：

- 寄存器
- L1 Cache
- 部分 L2 Cache

多核处理器可以提高多任务和多线程程序的执行效率。

## 6. 对称多处理器

SMP 是 Symmetric Multiprocessing，即对称多处理。

SMP 系统中，多个处理器地位对等，共享同一主存和 I/O 系统。

特点：

- 所有处理器访问同一物理地址空间。
- 操作系统统一调度各处理器。
- 程序共享数据方便。
- 需要解决同步和 Cache 一致性问题。

## 7. 共享内存与分布式内存

多处理器系统按内存组织可分为共享内存和分布式内存。

| 类型 | 特点 |
| --- | --- |
| 共享内存 | 多个处理器共享同一地址空间 |
| 分布式内存 | 每个处理器有自己的局部内存 |

共享内存编程方便，但可扩展性受限。

分布式内存扩展性较好，但处理器之间需要通过消息通信交换数据。

## 8. UMA 和 NUMA

共享内存系统还可分为 UMA 和 NUMA。

| 类型 | 含义 |
| --- | --- |
| UMA | Uniform Memory Access，访问各内存位置时间基本相同 |
| NUMA | Non-Uniform Memory Access，访问本地内存和远程内存时间不同 |

NUMA 系统适合更多处理器扩展，但程序性能受数据放置位置影响。

## 9. 多线程处理器

多线程处理器允许一个处理器核心支持多个硬件线程。

当一个线程因为访存等原因等待时，处理器可以切换或同时执行其他线程，提高资源利用率。

常见思想：

- 细粒度多线程
- 粗粒度多线程
- 同时多线程

同时多线程可以在同一周期内从多个线程发射指令。

## 10. Cache 一致性

多核系统中，每个核心可能有自己的 Cache。

如果多个 Cache 中保存同一主存块的副本，当某个核心修改数据时，其他核心的副本可能变旧。

这就是 Cache 一致性问题。

例如：

1. 核心 1 和核心 2 都缓存变量 `x`。
2. 核心 1 修改 `x`。
3. 核心 2 若仍使用旧副本，就会读到错误数据。

解决 Cache 一致性需要硬件协议或软件配合。

408 通常只要求理解为什么会出现一致性问题，不要求深入掌握具体协议。

## 11. 并行性能和加速比

多处理器不一定让程序速度按处理器数量线性提升。

如果程序中只有一部分可以并行执行，加速比会受串行部分限制。

常见思想可用 Amdahl 定律理解：

$$
S
=
\frac{1}{(1-p) + \frac{p}{n}}
$$

其中：

- $p$ 是可并行部分比例。
- $n$ 是处理器数量。
- $S$ 是加速比。

当串行部分较大时，增加处理器数量收益有限。

## 12. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| Flynn 分类 | 判断 SISD、SIMD、MIMD |
| 多核概念 | 判断多核处理器共享和私有部件 |
| SMP | 判断对称多处理器特征 |

| Cache 一致性 | 判断多个核心缓存同一数据时的问题 |
| 加速比 | 根据并行比例和处理器数分析性能上限 |

## 13. 易错点

- 多核不等于每个程序都能自动加速。
- SIMD 适合数据并行，不适合复杂分支控制。
- MIMD 中不同处理器可以执行不同指令。
- SMP 中处理器地位对等，共享主存。
- Cache 一致性问题来自多个缓存副本。
- 加速比受串行部分限制，处理器越多不一定收益越大。

多处理器的核心是并行。
