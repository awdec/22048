<h1><center>第三章 存储系统</center></h1>

## 1. 本章定位

存储系统是计算机组成原理中的重点章节，主要研究计算机如何组织、访问和管理不同层次的存储器。

本章主要回答六个问题：

1. 为什么需要多级存储体系。
2. 主存储器如何存放程序和数据。
3. 主存储器如何与 CPU 连接。
4. 外部存储器如何组织和访问数据。
5. Cache 如何提高 CPU 访问速度。
6. 虚拟存储器如何扩大程序可用地址空间。

408 中，存储系统经常考查地址划分、容量计算、Cache 映射、平均访问时间、页式虚拟存储、TLB 和缺页等内容。

## 3. 学习主线

本章建议按下面顺序理解：

$$
\text{存储层次}
\longrightarrow
\text{主存}
\longrightarrow
\text{CPU 与主存连接}
\longrightarrow
\text{外存}
\longrightarrow
\text{Cache}
\longrightarrow
\text{虚拟存储器}
$$

::: tip 重点
第三章的核心不是记住存储器名称，而是理解“容量、速度、地址、映射、替换、命中率”之间的关系。Cache 和虚拟存储器是本章最容易出综合题的部分。
:::

## 4. 常见考点

| 考点 | 要求 |
| --- | --- |
| 存储层次 | 理解寄存器、Cache、主存、外存之间的速度和容量关系 |
| 局部性原理 | 会区分时间局部性和空间局部性 |
| 主存容量 | 会根据地址线、数据线、芯片规格计算容量 |
| DRAM 刷新 | 掌握集中刷新、分散刷新、异步刷新的特点 |
| 主存连接 | 会分析位扩展、字扩展和地址译码 |
| 磁盘访问时间 | 会计算寻道时间、旋转延迟和传输时间 |
| Cache 地址划分 | 会划分标记、组号、块内地址 |
| Cache 映射 | 会区分直接映射、全相联映射和组相联映射 |
| Cache 写策略 | 会区分写直达、写回、写分配和非写分配 |
| 虚拟存储器 | 会分析页号、页内偏移、页表和 TLB |

## 5. 易错点

- Cache 容量通常指数据容量，不一定包含标记位和有效位。
- 主存块大小等于 Cache 行中数据块大小。
- 直接映射中，一个主存块只能映射到一个 Cache 行。
- 全相联映射中，一个主存块可以映射到任意 Cache 行。
- 组相联映射中，一个主存块只能映射到某一组中的任意行。
- Cache 平均访问时间要看题目是否把未命中时的 Cache 访问时间计入。
- 虚拟地址由 CPU 产生，物理地址用于访问主存。
- 页内偏移在地址转换前后不变。
- TLB 是页表项的高速缓存，不是数据 Cache。
