<h1><center>第一节 I/O 接口</center></h1>

## 1. 为什么需要 I/O 接口

外部设备不能直接连接 CPU 和系统总线，原因主要有四个：

1. 外设种类多，信号格式不统一。
2. 外设速度通常远低于 CPU 和主存。
3. 外设工作方式各不相同，需要专门控制。
4. CPU 不能直接处理所有外设的电气细节和机械动作。

因此，主机和外设之间需要 I/O 接口。

I/O 接口位于主机和外设之间，负责完成数据缓冲、命令控制、状态反馈、格式转换和中断/DMA 请求等工作。

## 2. I/O 接口的位置

I/O 接口通常一端连接系统总线，另一端连接外部设备。

可以这样理解：

$$
\text{CPU/主存}
\leftrightarrow
\text{系统总线}
\leftrightarrow
\text{I/O 接口}
\leftrightarrow
\text{外部设备}
$$

CPU 访问外设时，实际访问的是 I/O 接口中的寄存器或端口。

外设通过接口向 CPU 报告状态、接收命令、传送数据。

## 3. I/O 接口的主要功能

### 数据缓冲

CPU 和主存速度快，外设速度慢。I/O 接口需要用数据缓冲寄存器暂存数据，缓解速度不匹配。

例如键盘输入一个字符后，可以先把字符放入接口的数据寄存器，等待 CPU 读取。

### 状态保存

接口需要保存外设状态，供 CPU 查询。

常见状态包括：

- 设备是否就绪
- 数据是否准备好
- 是否忙
- 是否出错
- 是否允许中断

CPU 可以读取状态寄存器，判断下一步操作。

### 命令控制

CPU 通过接口向外设发送控制命令。

例如：

- 启动设备
- 停止设备
- 设置工作方式
- 清除错误标志
- 允许或禁止中断

这些命令通常写入控制寄存器。

### 数据格式转换

CPU 和主存内部通常按字节或字传送数据，而外设可能按位、字符、块或特定协议传送。

接口需要进行必要的格式转换。

例如串行设备按位传输，接口需要完成串并转换。

### 地址译码

接口需要识别 CPU 发出的地址是否属于本接口。

当地址匹配时，接口响应对应的读写操作。

### 中断和 DMA 请求

接口可以在设备准备好或发生异常时向 CPU 发出中断请求。

对于高速设备，接口还可以配合 DMA 控制器完成大批量数据传送。

## 4. I/O 接口的基本结构

典型 I/O 接口包括：

| 部件 | 作用 |
| --- | --- |
| 数据寄存器 | 暂存 CPU 与外设之间传送的数据 |
| 状态寄存器 | 保存外设和接口当前状态 |
| 控制寄存器 | 保存 CPU 发出的控制命令 |
| 地址译码逻辑 | 判断 CPU 访问的是哪个端口 |
| 控制逻辑 | 产生接口内部控制信号 |
| 数据缓冲器 | 连接系统总线和接口内部数据通路 |
| 中断/DMA 逻辑 | 发出中断请求或 DMA 请求 |

接口不一定每种寄存器都物理上完全独立，但在逻辑功能上通常可以这样划分。

## 5. I/O 端口

I/O 端口是 CPU 可以访问的 I/O 接口寄存器。

端口不是物理插孔，而是具有地址的寄存器或寄存器集合。

常见端口类型：

| 端口 | 对应寄存器 | 访问方向 | 作用 |
| --- | --- | --- | --- |
| 数据端口 | 数据寄存器 | 可读或可写 | 传送数据 |
| 状态端口 | 状态寄存器 | 通常只读 | 读取设备状态 |
| 控制端口 | 控制寄存器 | 通常只写 | 发送控制命令 |

### 数据端口

数据端口用于 CPU 和外设之间交换数据。

输入设备的数据端口通常由 CPU 读。

输出设备的数据端口通常由 CPU 写。

### 状态端口

状态端口用于 CPU 查询设备当前状态。

常见状态位：

| 状态位 | 含义 |
| --- | --- |
| Ready | 设备就绪 |
| Busy | 设备忙 |
| Done | 操作完成 |
| Error | 发生错误 |
| Interrupt Enable | 中断允许 |

程序查询方式的核心就是反复读取状态端口。

### 控制端口

控制端口用于 CPU 向设备发送命令。

例如：

- 启动输入
- 启动输出
- 设置工作模式
- 清除中断请求
- 复位设备

## 6. I/O 端口编址方式

CPU 访问 I/O 端口时，必须能确定端口地址。

常见编址方式有统一编址和独立编址。

### 统一编址

统一编址也称存储器映射 I/O。

它把 I/O 端口和主存单元放在同一个地址空间中。

特点：

- CPU 用访存指令访问 I/O 端口。
- I/O 端口占用一部分主存地址空间。
- 指令系统不需要专门 I/O 指令。
- 可使用丰富的访存寻址方式。

例如某接口的数据端口被映射到地址 `0xFFFF0000`，CPU 对这个地址执行读写，就相当于访问该 I/O 端口。

优点：

- 指令统一。
- 编程方便。
- 可使用访存相关寻址方式。

缺点：

- 占用主存地址空间。
- 需要避免普通内存地址和 I/O 地址冲突。

### 独立编址

独立编址也称 I/O 映射编址或端口映射 I/O。

它为 I/O 端口设置独立的地址空间，与主存地址空间分开。

特点：

- CPU 使用专门 I/O 指令访问端口。
- I/O 地址空间和主存地址空间互不占用。
- 指令系统需要提供输入/输出指令。

例如某些体系结构中，CPU 使用 `IN` 指令从端口读数据，使用 `OUT` 指令向端口写数据。

优点：

- 不占用主存地址空间。
- I/O 操作和存储器操作区分明确。

缺点：

- 需要专门 I/O 指令。
- 寻址方式通常不如访存指令灵活。

### 两种编址方式对比

| 比较项 | 统一编址 | 独立编址 |
| --- | --- | --- |
| 地址空间 | 与主存共用 | 独立于主存 |
| 指令 | 访存指令 | 专门 I/O 指令 |
| 是否占用主存地址 | 占用 | 不占用 |
| 编程灵活性 | 较高 | 较低 |
| 硬件区分 | 通过地址范围区分 | 通过 I/O 控制信号区分 |

## 7. I/O 接口和总线信号

CPU 访问 I/O 接口时，通常需要以下信息：

| 信息 | 来源 | 作用 |
| --- | --- | --- |
| 地址 | CPU | 指定访问哪个端口 |
| 控制信号 | CPU | 指明读、写、I/O 或存储器操作 |
| 数据 | CPU 或接口 | 完成输入或输出 |
| 状态 | 接口 | 告诉 CPU 设备是否就绪 |

读 I/O 端口时：

1. CPU 发出端口地址。
2. CPU 发出读控制信号。
3. 接口译码地址。
4. 接口把指定寄存器内容送上数据总线。
5. CPU 接收数据。

写 I/O 端口时：

1. CPU 发出端口地址。
2. CPU 把数据送上数据总线。
3. CPU 发出写控制信号。
4. 接口译码地址。
5. 接口把数据写入对应寄存器。

## 8. I/O 接口和中断

当外设准备好数据或完成操作时，接口可以向 CPU 发出中断请求。

常见过程：

1. 外设完成某项操作。
2. 接口设置状态位。
3. 若中断允许，接口发出中断请求。
4. CPU 响应中断后执行中断服务程序。
5. CPU 在服务程序中访问接口端口。
6. 服务结束后清除中断请求并返回原程序。

中断方式减少了 CPU 无意义轮询，提高了 CPU 利用率。

## 9. I/O 接口和 DMA

对于高速设备或大批量数据传输，仅靠 CPU 一次次读写端口效率较低。

DMA 方式下，接口或设备向 DMA 控制器提出请求，由 DMA 控制器控制主存和外设之间的数据传送。

I/O 接口在 DMA 中可能负责：

- 发出 DMA 请求。
- 接收 DMA 响应。
- 提供设备状态。
- 配合数据传送。
- 在传送结束后发出中断。

## 10. 例题：判断端口类型

某接口有三个端口：

- 端口 A：CPU 读取后得到键盘输入字符。
- 端口 B：CPU 读取后可判断键盘是否有新字符。
- 端口 C：CPU 写入后可允许或禁止键盘中断。

判断三个端口类型。

分析：

- 端口 A 传送输入数据，是数据端口。
- 端口 B 反映设备状态，是状态端口。
- 端口 C 接收 CPU 控制命令，是控制端口。

## 12. 易错提醒

- I/O 接口不是简单电缆，而是含有寄存器和控制逻辑的硬件部件。
- CPU 通常访问接口寄存器，而不是直接访问外设内部机械部件。
- 状态端口一般用于读，控制端口一般用于写。
- 统一编址不是把外设变成主存，而是把 I/O 端口映射到主存地址空间。
- 独立编址需要专门 I/O 指令或专门 I/O 控制信号。

## 13. 小结

I/O 接口是主机和外设之间的桥梁。

它把外设复杂、低速、不统一的特性封装起来，让 CPU 能以读写端口的方式完成输入、输出、控制和状态查询。
