<h1><center>第一节 计算机系统层次结构</center></h1>

## 1. 计算机系统的组成

完整的计算机系统由硬件系统和软件系统组成。

$$
\text{计算机系统}
=
\text{硬件系统}
+
\text{软件系统}
$$

硬件系统是计算机中看得见、摸得着的物理设备，软件系统是运行在硬件之上的程序和相关文档。

| 组成 | 含义 | 例子 |
| --- | --- | --- |
| 硬件 | 计算机系统的物理实体 | CPU、主存、硬盘、键盘、显示器 |
| 软件 | 指挥硬件工作的程序及相关资料 | 操作系统、编译器、应用程序 |

硬件是软件运行的基础，软件是硬件功能发挥的手段。没有硬件，软件无法运行；没有软件，硬件难以完成具体任务。

## 2. 硬件系统

从功能角度看，计算机硬件通常包括：

- 运算器
- 控制器
- 存储器
- 输入设备
- 输出设备

其中，运算器和控制器合称为中央处理器，也就是 CPU。

$$
\text{CPU}
=
\text{运算器}
+
\text{控制器}
$$

各部分作用如下：

| 部件 | 主要作用 |
| --- | --- |
| 运算器 | 完成算术运算和逻辑运算 |
| 控制器 | 解释指令并控制各部件协调工作 |
| 存储器 | 存放程序和数据 |
| 输入设备 | 将外部信息送入计算机 |
| 输出设备 | 将计算机处理结果输出给外部 |

现代计算机中，CPU、主存储器和 I/O 设备通常通过总线或互连结构连接。

## 3. 软件系统

软件系统通常分为系统软件和应用软件。

| 类型 | 作用 | 例子 |
| --- | --- | --- |
| 系统软件 | 管理计算机资源，为应用软件提供运行环境 | 操作系统、编译程序、数据库管理系统 |
| 应用软件 | 为用户完成特定任务 | 浏览器、办公软件、图像处理软件 |

在 408 中，需要重点理解操作系统、编译器和硬件之间的关系：

- 高级语言程序不能被硬件直接执行。
- 编译器将高级语言程序转换为机器能够识别的目标代码。
- 操作系统负责管理 CPU、内存、文件和 I/O 设备等资源。
- CPU 最终执行的是机器指令。

## 4. 计算机系统的层次结构

计算机系统可以看作由多个层次构成。

从用户到硬件，常见层次可以概括为：

$$
\text{用户}
\longrightarrow
\text{应用程序}
\longrightarrow
\text{操作系统}
\longrightarrow
\text{机器语言}
\longrightarrow
\text{微体系结构}
\longrightarrow
\text{硬件电路}
$$

每一层都建立在下一层基础上，并向上一层提供服务。

| 层次 | 说明 |
| --- | --- |
| 应用程序层 | 面向用户需求，完成具体任务 |
| 操作系统层 | 管理硬件资源，提供系统调用接口 |
| 指令系统层 | 定义 CPU 能识别和执行的机器指令 |
| 微体系结构层 | 实现指令执行的数据通路和控制逻辑 |
| 硬件电路层 | 由数字逻辑电路和电子器件构成 |

这种层次结构体现了计算机系统中的抽象思想。上层不必关心下层全部实现细节，只需要使用下层提供的接口。

## 5. 程序语言的层次与转换

从程序员编写程序到 CPU 真正执行，中间通常要经历多个语言层次。

常见层次可以概括为：

$$
\text{高级语言}
\longrightarrow
\text{汇编语言}
\longrightarrow
\text{机器语言}
\longrightarrow
\text{硬件执行}
$$

### 5.1 高级语言

高级语言更接近人的思维方式，通常与具体机器结构关系较弱。

常见高级语言包括：

- C
- C++
- Java
- Python

高级语言的特点：

| 特点 | 说明 |
| --- | --- |
| 抽象程度高 | 可以使用变量、函数、循环、数组、对象等概念 |
| 可读性强 | 程序形式更接近数学表达和人的描述 |
| 可移植性较好 | 同一程序经过重新编译或解释，可能在不同机器上运行 |
| 不能被 CPU 直接执行 | 必须转换为机器指令后，CPU 才能执行 |

例如，高级语言中的语句：

```c
a = b + c;
```

从程序员角度看只是一次加法赋值；但从硬件角度看，可能需要取数、加法、存回等多条机器指令完成。

### 5.2 汇编语言

汇编语言是机器语言的符号化表示。

它用助记符、寄存器名和标号来表示机器指令中的二进制编码。例如：

```asm
ADD R1, R2, R3
```

可以理解为：

$$
R1 \leftarrow R2 + R3
$$

汇编语言的特点：

| 特点 | 说明 |
| --- | --- |
| 与指令系统密切相关 | 不同 CPU 指令系统通常对应不同汇编语言 |
| 可读性强于机器语言 | 用助记符代替二进制操作码 |
| 可移植性较差 | 程序依赖具体机器的寄存器、指令和寻址方式 |
| 不能被 CPU 直接执行 | 需要汇编程序转换为机器语言 |

汇编语言通常比高级语言更接近硬件，能直接体现寄存器、主存地址、寻址方式和指令执行顺序。

### 5.3 机器语言

机器语言是由二进制机器指令组成的语言，是 CPU 能够直接识别并执行的语言。

一条机器指令通常包含：

$$
\text{操作码}
+
\text{地址码}
$$

| 字段 | 含义 |
| --- | --- |
| 操作码 | 指出执行什么操作 |
| 地址码 | 指出操作数或操作数地址在哪里 |

机器语言的特点：

| 特点 | 说明 |
| --- | --- |
| 二进制形式 | 程序和数据在存储器中都以二进制形式保存 |
| 硬件可直接执行 | CPU 取出机器指令后进行译码和执行 |
| 与机器密切相关 | 不同指令系统的机器指令格式可能不同 |
| 人工编写困难 | 可读性差，容易出错 |

408 中要特别注意：CPU 最终执行的是机器指令，而不是高级语言语句，也不是汇编语言文本。

### 5.4 翻译程序

高级语言和汇编语言要变成机器语言，需要依靠翻译程序。

| 翻译程序 | 作用 | 输入 | 输出 |
| --- | --- | --- | --- |
| 编译程序 | 将高级语言源程序翻译成目标程序 | 高级语言程序 | 汇编代码或机器代码 |
| 汇编程序 | 将汇编语言程序翻译成机器语言目标程序 | 汇编语言程序 | 机器语言目标程序 |
| 解释程序 | 边翻译边执行高级语言语句 | 高级语言程序 | 执行结果 |
| 链接程序 | 将多个目标模块和库函数连接起来 | 目标模块、库文件 | 可执行文件 |
| 装入程序 | 将可执行程序装入主存并准备运行 | 可执行文件 | 主存中的进程映像 |

典型的编译执行过程可以表示为：

$$
\text{高级语言源程序}
\longrightarrow
\text{编译}
\longrightarrow
\text{汇编语言程序}
\longrightarrow
\text{汇编}
\longrightarrow
\text{目标程序}
\longrightarrow
\text{链接}
\longrightarrow
\text{可执行程序}
\longrightarrow
\text{装入内存}
\longrightarrow
\text{CPU 执行机器指令}
$$

不同编译系统的中间结果不完全相同，有的编译器可以直接生成机器代码；但无论过程怎样变化，最终都必须形成 CPU 能执行的机器指令。

### 5.5 三类语言的对比

| 语言层次 | 接近对象 | 表示形式 | 是否能被 CPU 直接执行 | 可移植性 |
| --- | --- | --- | --- | --- |
| 高级语言 | 人和问题 | 变量、表达式、函数、语句 | 不能 | 较好 |
| 汇编语言 | 指令系统 | 助记符、寄存器名、标号 | 不能 | 较差 |
| 机器语言 | 硬件 | 二进制机器指令 | 能 | 很差 |

三者关系可以简单理解为：

- 高级语言解决“要做什么”。
- 汇编语言描述“用哪些机器级操作做”。
- 机器语言给出“CPU 实际执行的二进制指令”。

## 6. 冯·诺依曼计算机

冯·诺依曼计算机的核心思想是存储程序。

存储程序思想是指：

- 程序和数据都以二进制形式存放在存储器中。
- 程序由一条条指令组成。
- 计算机按地址顺序取出指令并执行。
- 程序可以控制计算机自动完成任务。

冯·诺依曼计算机的五大组成部分：

| 部分 | 作用 |
| --- | --- |
| 运算器 | 进行算术运算和逻辑运算 |
| 控制器 | 控制程序执行过程 |
| 存储器 | 存放程序和数据 |
| 输入设备 | 输入程序和数据 |
| 输出设备 | 输出处理结果 |

其基本工作过程可以概括为：

$$
\text{取指}
\longrightarrow
\text{译码}
\longrightarrow
\text{执行}
\longrightarrow
\text{访存}
\longrightarrow
\text{写回}
$$

在第一章中，只需要理解大致执行流程；具体数据通路和控制过程会在 CPU 章节详细学习。

## 7. 计算机系统结构与计算机组成

408 中容易混淆“计算机系统结构”和“计算机组成”。

| 概念 | 关注点 | 例子 |
| --- | --- | --- |
| 计算机系统结构 | 程序员可见的机器属性 | 指令系统、数据类型、寻址方式 |
| 计算机组成 | 如何实现这些机器属性 | 数据通路、控制器、Cache 结构 |
| 计算机实现 | 具体物理实现 | 器件、工艺、电路布局 |

简单理解：

- 系统结构回答“机器对程序员表现成什么样”。
- 计算机组成回答“机器内部怎样实现这些功能”。
- 计算机实现回答“用什么器件和工艺做出来”。

## 8. 408 常见考法

| 考法 | 示例 |
| --- | --- |
| 概念判断 | 判断硬件、软件、系统软件、应用软件 |
| 层次关系 | 判断高级语言、操作系统、指令系统、硬件之间的关系 |
| 语言层次 | 区分高级语言、汇编语言和机器语言 |
| 翻译过程 | 判断编译、汇编、链接、装入各自的作用 |
| 冯·诺依曼特点 | 判断程序和数据是否都存放在存储器中 |
| 部件功能 | 判断控制器、运算器、存储器的作用 |
| 概念区分 | 区分计算机系统结构、计算机组成和计算机实现 |

## 9. 易错点

- 程序和数据在计算机中本质上都以二进制形式存放。
- CPU 不能直接执行高级语言程序，最终执行的是机器指令。
- 汇编语言不是机器语言，CPU 不能直接执行汇编语言源程序。
- 汇编语言通常与具体指令系统相关，因此可移植性比高级语言差。
- 编译程序、汇编程序、链接程序和装入程序的作用不同，不能混为一谈。
- 控制器负责控制各部件协调工作，不直接完成算术运算。
- 运算器负责运算，但不能单独控制整机运行。
- 指令系统是软件和硬件之间的重要接口。
- 计算机系统结构不等于计算机组成，前者偏“对程序员可见”，后者偏“内部如何实现”。

## 10. 本节小结

计算机系统层次结构的核心是理解“抽象”和“接口”。

从用户角度看，计算机可以运行应用程序；从硬件角度看，计算机只是在执行一条条机器指令。中间的操作系统、编译器、汇编程序、链接程序和指令系统把高层需求逐步转换成底层硬件可以执行的操作。
