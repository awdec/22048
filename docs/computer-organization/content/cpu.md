<h1><center>第五章 中央处理器</center></h1>

## 1. 本章定位

中央处理器是计算机组成原理中连接“指令系统”和“硬件实现”的核心章节。

前面学习了指令如何表示操作，本章要进一步理解 CPU 如何取出指令、解释指令、产生控制信号，并通过数据通路完成运算和访存。

本章主要回答七个问题：

1. CPU 有哪些功能，由哪些基本部件组成。
2. 一条指令从取出到执行结束经历哪些阶段。
3. 数据通路如何让数据在寄存器、ALU、主存之间流动。
4. 控制器如何产生控制信号。
5. 异常和中断如何打断正常执行流程。
6. 指令流水线如何提高吞吐率，以及会遇到哪些冲突。
7. 多处理器系统有哪些基本组织形式。

## 3. 学习主线

本章建议按下面顺序理解：

$$
\text{CPU 结构}
\longrightarrow
\text{指令执行}
\longrightarrow
\text{数据通路}
\longrightarrow
\text{控制器}
\longrightarrow
\text{异常和中断}
\longrightarrow
\text{流水线}
\longrightarrow
\text{多处理器}
$$

::: tip 重点
第五章不要只背部件名称。408 更常考“某条指令执行时，数据经过哪些部件、控制器发出哪些信号、流水线为何停顿、异常中断如何改变执行流程”。
:::

## 4. 常见考点

| 考点 | 要求 |
| --- | --- |
| CPU 基本结构 | 掌握运算器、控制器和寄存器的作用 |
| 指令周期 | 会区分取指、间址、执行和中断周期 |
| 微操作序列 | 能写出常见指令的基本数据流 |
| 数据通路 | 理解单总线、多总线和专用通路差异 |
| 控制器 | 会比较硬布线控制和微程序控制 |
| 异常中断 | 掌握响应、保存现场、转入处理程序、返回的过程 |
| 流水线 | 会计算流水线执行时间、吞吐率和加速比 |
| 相关冲突 | 会区分结构相关、数据相关和控制相关 |
| 多处理器 | 理解多核、SMP、SIMD、MIMD、Cache 一致性等基本概念 |

## 5. 易错点

- 指令周期不等于机器周期，也不等于时钟周期。
- PC 保存下一条要执行指令的地址，IR 保存当前正在执行的指令。
- MAR 保存主存地址，MDR 保存主存读写数据。
- 数据通路负责“数据怎么走”，控制器负责“什么时候让数据走”。
- 硬布线控制速度快但修改困难，微程序控制灵活但速度通常较慢。
- 异常多来自 CPU 内部，外部中断多来自外设或时钟等外部事件。
- 流水线提高吞吐率，不会缩短单条指令本身的执行延迟。
- 流水线最大加速比受阶段数、阶段均衡性和冲突影响。
