<h1><center>第三章 存储系统</center></h1>

## 全章题目简要总结

本章练习主要围绕“数据在多级存储系统中如何存放、访问、映射和调度”展开。做题时要先判断题目属于哪一层存储结构，再选择对应的地址、容量、时间或替换规则。

| 小节 | 题目主线 | 核心关注点 |
| --- | --- | --- |
| 第一节 存储器概述 | 存储器分类、存取方式、性能指标、多级存储 | 分清 RAM、ROM、SAM、DAM 和相联存储器，理解存取时间与存取周期 |
| 第二节 主存储器 | SRAM、DRAM、ROM、刷新、多模块交叉存储 | 先算容量、地址线和数据线，再看刷新方式、编址方式和访存周期 |
| 第三节 主存储器与 CPU 的连接 | 存储芯片扩展、片选译码、地址范围、MAR 位数 | 先判定位扩展、字扩展或字位扩展，再用高位地址做片选 |
| 第四节 外部存储器 | 磁盘结构、访问时间、传输速率、SSD 特性 | 磁盘题拆成寻道、旋转等待和传输时间，SSD 题分清读写页和擦除块 |
| 第五节 高速缓冲存储器 | Cache 映射、地址字段、替换算法、缺失损失 | 先定块大小、行数、组数和路数，再拆标记、组号和块内地址 |
| 第六节 虚拟存储器 | 页表、TLB、缺页处理、MMU 地址转换 | 先拆虚页号和页内偏移，再查 TLB 或页表，判断命中、缺页或越权 |

本章题目的共同特点是：同一个地址或容量条件，在不同存储层次中要按不同规则解释。因此不要一上来就套公式，而应先确定：

1. 题目讨论的是主存、Cache、外存，还是虚拟存储器。
2. 地址是按字节编址还是按字编址。
3. 容量问的是数据区容量、芯片容量，还是包含标记位和控制位的总容量。
4. 访问过程是否涉及映射、替换、缺页、中断或异常。
5. 时间计算是否需要拆成多个阶段，例如访存周期、总线传送、磁盘寻道和旋转等待。

可以把本章题目统一成一句话：

$$
\text{先定位存储层次，再拆地址和容量，最后按访问流程计算}
$$

常见易错点包括：

- 把存取时间和存取周期混为一谈。
- 把字扩展、位扩展和字位扩展混淆。
- 地址范围题漏掉首末地址的闭区间计算。
- 磁盘题只算传输时间，漏掉平均寻道时间和平均旋转等待时间。
- Cache 题把行数、组数和路数混用。
- Cache 总容量题漏算标记位、有效位或脏位。
- TLB 缺失、页面缺失和 Cache 缺失的发生阶段混淆。
- 缺页处理后误以为执行下一条指令，而不是重新执行原指令。
