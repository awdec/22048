<h1><center>第一章 计算机系统概述</center></h1>

## 全章题目简要总结

本章练习主要围绕“计算机系统由哪些层次组成，以及如何评价计算机性能”展开。题目一类考概念边界，一类考性能公式，做题时要先判断是概念辨析题还是计算比较题。

| 小节 | 题目主线 | 核心关注点 |
| --- | --- | --- |
| 第一节 计算机系统层次结构 | 冯·诺依曼机、存储程序、虚拟机层次、ISA、翻译程序 | 分清机器语言、汇编语言、高级语言，以及编译、解释、汇编、链接的作用 |
| 第二节 计算机的性能指标 | CPI、IPC、MIPS、FLOPS、主频、时钟周期、执行时间 | 围绕 $T=\frac{IC\times CPI}{f}$ 分析，不只看单一指标 |

本章题目的共同特点是：很多选项看起来都“有点对”，但题目往往问的是最准确的概念或最完整的性能条件。因此不要只凭关键词判断，而应先确定：

1. 题目讨论的是系统层次、语言翻译、ISA，还是性能指标。
2. CPU 能否“直接执行”，要看对象是不是机器语言程序。
3. 比较性能时，是否同时给出了指令条数、平均 CPI 和主频。
4. MIPS、FLOPS、CPI、IPC 等指标是否依赖具体程序和指令系统。
5. 加速题中，哪些时间被加速，哪些时间没有变化。

可以把本章题目统一成一句话：

$$
\text{概念题先划边界，性能题先写公式}
$$

常见易错点包括：

- 把“存储程序”误解成程序从外存直接被 CPU 执行。
- 把汇编语言和机器语言混为一谈。
- 把编译、解释、汇编、链接、装入的作用混淆。
- 把 ISA 和微体系结构混淆。
- 只根据主频、CPI 或 MIPS 中的单一指标判断机器快慢。
- 平均 CPI 题忘记按指令比例加权。
- 加速比题把未被加速的 I/O 时间也一起缩短。
- FLOPS、MIPS、GIPS、PFLOPS 等单位换算时漏掉数量级。
