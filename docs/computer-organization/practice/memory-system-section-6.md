<h1><center>第六节 虚拟存储器</center></h1>

::: info 例 1.
为使虚拟存储系统有效地发挥其预期的作用，所运行程序应具有的特性是（ ）。<br>
A. 不应含有过多的 I/O 操作<br>
B. 大小不应小于实际的内存容量<br>
C. 应具有较好的局部性<br>
D. 顺序执行的指令不应过多
:::

::: details 解析
答案：C。

虚拟存储器能工作的核心前提是程序访问具有局部性。

程序在一段时间内通常集中访问少量代码和数据，因此只需把当前活跃的页或段装入主存，就能让较大的虚拟地址空间在较小的实际主存上运行。

A 说的是 I/O 操作多少，B 和 D 都不是虚拟存储器发挥作用的关键条件。

所以选 C。

:::

::: info 例 2.
虚拟存储管理系统的基础是程序访问的局部性原理，此理论的基本含义是（ ）。<br>
A. 在程序的执行过程中，程序对主存的访问是不均匀的<br>
B. 空间局部性<br>
C. 时间局部性<br>
D. 代码的顺序执行

:::

::: details 解析
答案：A。

局部性原理的基本含义是：程序在执行过程中，对主存的访问不是均匀分布的，而是集中在某些局部区域。

它进一步表现为：

- 时间局部性：刚访问过的内容，短时间内可能再次访问。
- 空间局部性：刚访问过的位置附近的内容，短时间内可能被访问。

B、C 只是局部性的两个具体方面；D 是产生空间局部性的一个常见原因，但不是局部性原理的完整含义。

所以选 A。

:::

::: info 例 3.
在虚拟存储器中，当程序正在执行时，由（ ）完成地址映射。<br>
A. 程序员<br>
B. 编译器<br>
C. 装入程序<br>
D. 操作系统

:::

::: details 解析
答案：D。

虚拟存储器中，程序执行时产生的是逻辑地址或虚拟地址，需要转换成主存物理地址。

运行时的地址映射由操作系统管理相关表项，并依靠硬件地址转换机构完成。选项中没有单独给出硬件或 MMU，因此应选操作系统。

程序员、编译器和装入程序都不能在程序运行过程中动态完成虚拟地址到物理地址的映射。

所以选 D。

:::

::: info 例 4.
下列有关虚拟存储管理机制中地址转换的叙述，错误的是（ ）。<br>
A. 地址转换是指把逻辑地址转换为物理地址<br>
B. 通常逻辑地址的位数比物理地址的位数少<br>
C. 地址转换过程中会发现是否“缺页”<br>
D. 内存管理单元（MMU）在地址转换过程中要访问页表项

:::

::: details 解析
答案：B。

地址转换是把逻辑地址或虚拟地址转换为主存物理地址，A 正确。

在虚拟存储系统中，虚拟地址空间通常可以大于实际主存空间，因此逻辑地址位数通常不少于物理地址位数，B 的说法错误。

地址转换过程中需要查页表项，页表项中的有效位可以反映该页是否已在主存中，因此可能发现缺页，C、D 正确。

所以选 B。

:::

::: info 例 5.
下列有关虚拟存储管理机制的页表的叙述中，错误的是（ ）。<br>
A. 系统中每个进程有一个页表<br>
B. 页表中每个表项与一个虚页对应<br>
C. 每个页表项中都包含装入位（有效位）<br>
D. 所有进程都可以访问页表
:::

::: details 解析
答案：D。

页式虚拟存储管理中，每个进程通常有自己的页表，页表项与虚页一一对应。

页表项中至少要包含页框号以及存在位或有效位等控制信息，用来判断该页是否已装入主存。

页表属于操作系统管理的关键数据结构，不能让所有进程任意访问，否则会破坏地址隔离和保护机制。

所以 D 错，选 D。

:::

::: info 例 6.
下列有关缺页处理的叙述中，错误的是（ ）。<br>
A. 若对应页表项中的有效位为 0，则发生缺页<br>
B. 缺页是一种外部中断，需要调用操作系统提供的中断服务程序来处理<br>
C. 缺页处理过程中需根据页表中给出的磁盘地址去读磁盘数据<br>
D. 缺页处理完后要重新执行发生缺页的指令
:::

::: details 解析
答案：B。

若页表项中的有效位为 $0$，说明对应页面不在主存中，访问该页会发生缺页，A 正确。

缺页属于程序执行过程中由地址转换引发的异常或故障，不是来自外部设备的外部中断，因此 B 错。

缺页处理通常由操作系统的缺页处理程序完成，需要从外存读入所缺页面；处理完成后，应重新执行导致缺页的那条指令，C、D 正确。

所以选 B。

:::

::: info 例 7.
下列关于段式虚拟存储管理的叙述中，错误的是（ ）。<br>
A. 段是逻辑结构上相对独立的程序块，因此段是可变长的<br>
B. 按程序中实际的段来分配主存，所以分配后的存储块是可变长的<br>
C. 每个段表项必须记录对应段在主存的起始位置和段的长度<br>
D. 分段方式对低级语言程序员和编译器来说是透明的

:::

::: details 解析
答案：D。

段式虚拟存储管理按程序的逻辑结构划分段，如代码段、数据段、栈段等，因此段长可以不同。

由于段长可变，分配到主存中的存储块也通常是可变长的。

段表项需要记录该段在主存中的起始地址和段长，以便完成地址转换和越界检查。

分段反映程序的逻辑结构，对汇编语言程序员、编译器等并不完全透明，D 的说法错误。

所以选 D。

:::

::: info 例 8.
【2013 统考真题】某计算机主存地址空间大小为 256 MB，虚拟地址空间大小为 4 GB，采用页式存储管理，页面大小为 4 KB，TLB（快表）采用全相联映射，有 4 个页表项，内容如下表所示。

| 有效位 | 标记    | 页框号 | … |
| ------ | ------- | ------ | - |
| 0      | FF180H  | 0002H  | … |
| 1      | 3FFF1H  | 0035H  | … |
| 0      | 02FF3H  | 0351H  | … |
| 1      | 03FFFH  | 0153H  | … |

则对虚拟地址 03FFF180H 进行虚实地址变换的结果是（ ）。<br>
A. 0153180H<br>
B. 0035180H<br>
C. TLB 缺失<br>
D. 缺页
:::

::: details 解析
答案：A。

页面大小为 $4\text{ KB}=2^{12}\text{ B}$，因此虚拟地址低 $12$ 位是页内偏移。

虚拟地址为：

$$
03FFF180\text{H}
$$

所以：

$$
\text{虚页号}=03FFF\text{H},\quad \text{页内偏移}=180\text{H}
$$

TLB 采用全相联映射，只需在所有有效表项中查找标记。表中有效位为 $1$ 且标记为 $03FFF\text{H}$ 的表项存在，其页框号为 $0153\text{H}$。

物理地址为：

$$
0153\text{H} \parallel 180\text{H}=0153180\text{H}
$$

所以选 A。

:::

::: info 例 9.
【2019 统考真题】下列关于缺页处理的叙述中，错误的是（ ）。<br>
A. 缺页是在地址转换时 CPU 检测到的一种异常<br>
B. 缺页处理由操作系统提供的缺页处理程序来完成<br>
C. 缺页处理程序根据页故障地址从外存读入所缺失的页<br>
D. 缺页处理完成后回到发生缺页的指令的下一条指令执行
:::

::: details 解析
答案：D。

缺页是在地址转换过程中由 CPU 或 MMU 检测到的一种异常，A 正确。

缺页处理由操作系统提供的缺页处理程序完成，B 正确。

缺页处理程序根据发生缺页的虚拟地址，找到所缺页面在外存中的位置并调入主存，C 正确。

缺页处理完成后，应重新执行发生缺页的那条指令，而不是执行下一条指令。

所以 D 错，选 D。

:::

::: info 例 10.
【2020 统考真题】下列关于 TLB 和 Cache 的叙述中，错误的是（ ）。<br>
A. 命中率都与程序局部性有关<br>
B. 缺失后都需要去访问主存<br>
C. 缺失处理都可以由硬件实现<br>
D. 都由 DRAM 存储器组成

:::

::: details 解析
答案：D。

TLB 和 Cache 都利用程序局部性提高访问速度，A 正确。

TLB 缺失后通常需要访问主存中的页表；Cache 缺失后也需要访问下一级存储层次，常见情况下要访问主存，B 正确。

Cache 缺失通常由硬件处理；TLB 缺失可以由硬件处理，也可以由软件处理，因此“都可以由硬件实现”在机制上成立，C 正确。

TLB 和 Cache 为了高速访问，通常采用 SRAM 或相联存储结构，而不是 DRAM。

所以 D 错，选 D。

:::

::: info 例 11.
【2022 统考真题】某计算机主存地址为 24 位，采用分页虚拟存储管理方式，虚拟地址空间大小为 4 GB，页大小为 4 KB，按字节编址。某个进程的页表部分内容如下表所示。

| 虚页号 | 实页号（页框号） | 存在位 |
| ------ | ---------------- | ------ |
| 82     | 024H             | 0      |
| …      | …                | …      |
| 129    | 180H             | 1      |
| 130    | 018H             | 1      |

当 CPU 访问虚拟地址 00082840H 时，虚-实地址转换的结果是（ ）。<br>
A. 得到主存地址 024840H<br>
B. 得到主存地址 180840H<br>
C. 得到主存地址 018840H<br>
D. 检测到缺页异常
:::

::: details 解析
答案：C。

页大小为 $4\text{ KB}=2^{12}\text{ B}$，因此虚拟地址低 $12$ 位为页内偏移。

虚拟地址：

$$
00082840\text{H}
$$

可分为：

$$
\text{虚页号}=82\text{H}=130,\quad \text{页内偏移}=840\text{H}
$$

查页表可知虚页号 $130$ 的存在位为 $1$，对应页框号为 $018\text{H}$。

因此主存地址为：

$$
018\text{H} \parallel 840\text{H}=018840\text{H}
$$

所以选 C。

:::

::: info 例 12.
【2024 统考真题】某计算机按字节编址，采用页式虚拟存储管理方式，虚拟地址为 32 位，主存地址为 30 位，页大小为 1 KB。若 TLB 共有 32 个表项，采用 4 路组相联映射方式，则 TLB 表项中标记字段的位数至少是（ ）。<br>
A. 17<br>
B. 18<br>
C. 19<br>
D. 20

:::

::: details 解析
答案：C。

页大小为 $1\text{ KB}=2^{10}\text{ B}$，因此页内偏移为 $10$ 位。

虚拟地址为 $32$ 位，所以虚页号位数为：

$$
32-10=22
$$

TLB 共有 $32$ 个表项，采用 $4$ 路组相联映射，因此组数为：

$$
\frac{32}{4}=8=2^3
$$

组号需要 $3$ 位。TLB 标记字段来自虚页号，去掉组号后得到：

$$
22-3=19
$$

位。

所以选 C。

:::

::: info 例 13.
【2024 统考真题】下列事件中，不是在 MMU 地址转换过程中检测的是（ ）。<br>
A. 访问越权<br>
B. Cache 缺失<br>
C. 页面缺失<br>
D. TLB 缺失
:::

::: details 解析
答案：B。

MMU 负责虚拟地址到物理地址的转换。在转换过程中，它会查 TLB 或页表，并根据页表项检测访问权限、页面是否存在等信息。

因此访问越权、页面缺失和 TLB 缺失都与 MMU 地址转换过程有关。

Cache 缺失发生在得到地址后访问 Cache 的过程中，不属于 MMU 地址转换过程中检测的事件。

所以选 B。

:::

| 例 5 | 页表作用 | 页表属于操作系统管理的数据结构，不能被所有进程任意访问 |
| 例 6 | 缺页处理 | 缺页是异常或故障，不是外部中断 |

| 例 8 | TLB 地址转换 | 虚页号查 TLB，命中后用页框号和页内偏移拼接物理地址 |

这些题共同强调：

$$
\text{虚拟存储题先拆虚拟地址，再查表项，最后判断是否命中、缺页或越权}
$$

做题时最容易混淆的是：

- 虚拟地址、逻辑地址和物理地址的关系。
- 页号、页框号和页内偏移的含义。
- TLB 缺失、页面缺失和 Cache 缺失的发生位置。
- 缺页属于异常或故障，不是外部中断。
- 缺页处理完成后重新执行原指令，而不是执行下一条指令。
- TLB 标记位来自虚页号，而不是完整虚拟地址。

## 答题技巧总结

### 1. 地址转换题

页式虚拟存储器的虚拟地址通常分为：

$$
\text{虚页号}
\quad
\text{页内偏移}
$$

物理地址通常分为：

$$
\text{页框号}
\quad
\text{页内偏移}
$$

页内偏移位数由页大小决定：

$$
\text{页内偏移位数}
=
\log_2 \text{页大小}
$$

解题步骤：

1. 由页大小求页内偏移位数。
2. 将虚拟地址拆成虚页号和页内偏移。
3. 查 TLB 或页表，判断是否命中、有效或缺页。
4. 若有效，则用页框号和页内偏移拼成物理地址。

### 2. 页表题

页表的基本作用是记录虚页到页框的映射关系。

页表项中常见字段包括：

- 页框号。
- 有效位或存在位。
- 访问权限位。
- 修改位。
- 访问位。

看到有效位或存在位为 $0$，通常表示该页不在主存中，访问时会发生缺页。

### 3. 缺页处理题

缺页处理的基本流程是：

1. 地址转换时发现页不在主存。
2. 产生缺页异常。
3. 操作系统调用缺页处理程序。
4. 从外存调入所缺页面。
5. 修改页表和相关状态位。
6. 重新执行发生缺页的指令。

注意两个常见陷阱：

- 缺页不是外部中断，而是异常或故障。
- 缺页处理后不是转到下一条指令，而是重新执行原指令。

### 4. TLB 题

TLB 是页表项的高速缓存，用于加速虚拟地址到物理地址的转换。

若 TLB 采用全相联映射，则用虚页号作为标记，与所有有效表项比较。

若 TLB 采用组相联映射，则：

$$
\text{组数}
=
\frac{\text{TLB 表项数}}{\text{路数}}
$$

$$
\text{组号位数}
=
\log_2 \text{组数}
$$

TLB 标记位数为：

$$
\text{TLB 标记位数}
=
\text{虚页号位数}
-
\text{组号位数}
$$

其中：

$$
\text{虚页号位数}
=
\text{虚拟地址位数}
-
\text{页内偏移位数}
$$

### 5. TLB、页表和 Cache 的区别

TLB、页表和 Cache 都可能出现“缺失”，但含义不同。

| 项目 | 发生阶段 | 含义 |
| --- | --- | --- |
| TLB 缺失 | 地址转换时 | TLB 中没有对应页表项，需要查页表 |
| 页面缺失 | 地址转换时 | 页面不在主存，需要操作系统调页 |
| Cache 缺失 | 访问 Cache 时 | 所需数据块不在 Cache 中，需要访问下一级存储 |

判断题中看到 “MMU 地址转换过程中检测 Cache 缺失” 这类说法，一般是错误的。

### 6. 段式管理题

段式管理按照程序的逻辑结构划分地址空间。

段表项通常至少包含：

- 段基址。
- 段长。
- 访问控制信息。

段式管理的特点：

- 段长可变。
- 便于共享和保护。
- 地址转换时需要进行段界限检查。
- 分段反映程序逻辑结构，对低级语言程序员和编译器不完全透明。
