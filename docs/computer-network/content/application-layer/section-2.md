<h1><center>第二节 域名系统 DNS</center></h1>

## 1. DNS 功能

DNS 是分布式、层次化的命名系统，主要把域名解析为 IP 地址，也支持别名、邮件服务器和负载分配等信息。

## 2. 域名层次

从右向左层次升高，例如：

```text
www.example.com
```

- `com`：顶级域。
- `example`：二级域。
- `www`：主机或子域标签。

根域位于最高层，完整域名末尾逻辑上还有一个点。

## 3. DNS 服务器

- 根域名服务器。
- 顶级域服务器。
- 权威域名服务器。
- 本地域名服务器/递归解析器。

本地域名服务器通常不属于全球域名层次本身，而是代表客户端执行查询并缓存结果。

## 4. 递归与迭代查询

- 递归查询：被查询服务器负责返回最终答案或错误。
- 迭代查询：服务器返回自己知道的最佳下一步转介，查询方继续询问。

主机通常向本地 DNS 发递归查询，本地 DNS 再对根、顶级域和权威服务器执行迭代查询。

## 5. 资源记录

| 类型 | 作用 |
| --- | --- |
| A | 域名到 IPv4 地址 |
| AAAA | 域名到 IPv6 地址 |
| NS | 域的权威服务器 |
| CNAME | 别名到规范名 |
| MX | 邮件服务器 |
| PTR | 反向解析 |

## 6. DNS 传输

DNS 通常使用端口 53：普通查询多使用 UDP；响应过大、区域传送或需要可靠传输时使用 TCP。

缓存记录受 TTL 控制，可减少时延和上级服务器负载，但短期内可能返回旧数据。

::: warning 易错点
DNS 不是只使用 UDP；本地 DNS 缓存命中时，无需访问根、顶级域和权威服务器。
:::
