<h1><center>第一节 网络应用模型与套接字</center></h1>

## 1. 客户—服务器模型

服务器通常长期运行，使用相对稳定的地址和熟知端口；客户端主动发起请求，可以间歇上线和使用动态地址。

优点是管理集中、服务质量易保证；缺点是服务器可能成为瓶颈和单点故障。

## 2. P2P 模型

对等方既可请求服务也可提供服务，具有自扩展性和较强分布性，但管理、安全和资源发现更复杂。

## 3. 应用层协议

应用层协议定义：

- 交换的报文类型。
- 报文语法和字段语义。
- 进程发送、响应报文的时序规则。

应用程序不是应用层协议本身；协议只是应用通信规则的一部分。

## 4. 套接字接口

典型 TCP 服务器流程：

```text
socket → bind → listen → accept → read/write → close
```

典型 TCP 客户端流程：

```text
socket → connect → read/write → close
```

accept 返回一个新的已连接套接字，监听套接字仍可继续接收其他连接。

UDP 无需 listen、accept 和连接建立，可用 sendto/recvfrom 按数据报通信。

## 5. 应用选择传输服务

应用需考虑：

- 可靠性。
- 吞吐量。
- 时延和抖动。
- 安全性。

例如文件传输重视可靠性，实时语音更重视低时延并能容忍少量丢包。

::: warning 易错点
服务器端口可相同而并发连接仍可区分，因为 TCP 用四元组标识连接；accept 创建的是连接套接字，不会替换监听套接字。
:::
