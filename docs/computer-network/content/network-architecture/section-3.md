<h1><center>第三节 分层、协议与体系结构</center></h1>

## 1. 分层思想

每层向上一层提供服务，并使用下一层提供的服务。分层可降低系统复杂度、促进标准化，但会产生首部开销和功能重复。

## 2. 协议、接口与服务

- **协议**：对等实体之间通信的规则集合。
- **接口**：相邻层之间交换信息的位置和规则。
- **服务**：下层通过接口向上层提供的功能。

协议三要素：

- 语法：数据格式。
- 语义：字段含义和动作。
- 同步：事件顺序和速度匹配。

## 3. 实体与数据单元

同一层中实现该层功能的活动元素称为实体，对等实体使用协议通信。

数据逐层封装：上层协议数据单元作为下层的数据部分，下层添加首部，部分链路层协议还添加尾部。

## 4. OSI 七层模型

| 层次 | 主要功能 | 典型数据单元 |
| --- | --- | --- |
| 应用层 | 面向用户的网络服务 | 报文 |
| 表示层 | 数据表示、压缩、加密 | 报文 |
| 会话层 | 会话建立与同步 | 报文 |
| 传输层 | 端到端进程通信 | 报文段/用户数据报 |
| 网络层 | 路由与分组转发 | 数据报/分组 |
| 数据链路层 | 成帧、差错控制与介质访问，部分协议提供可靠传输 | 帧 |
| 物理层 | 透明传输比特 | 比特 |

## 5. TCP/IP 与五层模型

TCP/IP 通常分为应用层、传输层、网际层和网络接口层。教学中的五层模型进一步把网络接口层拆为数据链路层和物理层。

## 6. 各层寻址

- 数据链路层：MAC 地址，标识同一链路上的接口。
- 网络层：IP 地址，标识网络中的接口和路由位置。
- 传输层：端口号，标识主机上的进程。

端到端通信通常需要“IP 地址 + 端口号”共同定位套接字端点。

::: tip 横向记忆
物理层传比特，链路层传帧，网络层选路径，传输层找进程，应用层定义具体业务。
:::
