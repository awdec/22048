<h1><center>第一节 传输层服务、端口与 UDP</center></h1>

## 1. 传输层功能

传输层为不同主机上的应用进程提供端到端逻辑通信，主要功能包括：

- 复用与分用。
- 差错检测。
- 可选的可靠传输、流量控制和拥塞控制。

网络层面向主机或接口，传输层通过端口号进一步交付到进程。

## 2. 端口号

TCP 和 UDP 端口号均为 16 位，范围 $0$～$65535$。

- 熟知端口：$0$～$1023$。
- 注册端口：$1024$～$49151$。
- 动态/私有端口：$49152$～$65535$。

常见端口：HTTP 80、DNS 53、SMTP 25、POP3 110、FTP 控制 21。

## 3. 套接字

一个通信端点通常用“IP 地址 + 端口号”标识。TCP 连接由四元组唯一标识：

$$
(\text{源 IP, 源端口, 目的 IP, 目的端口}).
$$

因此同一服务器端口可同时服务多个客户端连接。

## 4. UDP 特点

- 无连接。
- 尽最大努力交付，不保证可靠、按序或无重复。
- 面向报文，保留应用报文边界。
- 首部只有 8 字节，开销小。
- 支持一对一、一对多、多对一和多对多通信。

## 5. UDP 首部与校验和

UDP 首部包含源端口、目的端口、长度和校验和。校验和计算还使用包含源/目的 IP、协议号和 UDP 长度的伪首部，以检查误投递。

伪首部只参与校验和计算，不实际传输，也不是 UDP 首部的一部分。

## 6. UDP 适用场景

适合重视低时延、应用可自行控制可靠性的场景，如 DNS 查询、实时音视频和部分网络管理协议。

::: warning 易错点
UDP“无连接”不等于没有端口和校验；UDP 可以实现可靠应用，但可靠性必须由应用层另行设计。
:::
