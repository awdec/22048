<h1><center>第二节 TCP 报文段与可靠传输</center></h1>

## 1. TCP 特点

- 面向连接。
- 每条连接只有两个端点。
- 提供可靠、按序、无重复的字节流。
- 全双工通信。
- 面向字节流，不保留应用报文边界。

## 2. TCP 首部

关键字段包括：

- 源端口、目的端口。
- 序号、确认号。
- 数据偏移。
- 控制位 SYN、ACK、FIN、RST、PSH、URG。
- 接收窗口。
- 校验和、紧急指针和选项。

无选项的 TCP 首部为 20 字节。

## 3. 序号与确认号

TCP 按字节编号。报文段序号是本段数据第一个字节的序号；确认号表示期望收到的下一个字节序号，是累计确认。

若报文段序号为 $x$，携带 $L$ 字节数据，则正确接收后确认号为：

$$
x+L.
$$

SYN 和 FIN 虽不携带普通数据，也各消耗一个序号。

## 4. 可靠传输机制

- 校验和检测差错。
- 序号检测丢失、重复与失序。
- ACK 累计确认。
- 超时重传。
- 快速重传。
- 接收端缓存和按序交付。

## 5. 超时重传

TCP 根据测得 RTT 动态估计重传超时 RTO。RTO 应略大于加权平均 RTT，并随 RTT 偏差调整。

Karn 思想：重传报文段收到 ACK 时难以判断对应原报文还是重传报文，通常不使用该样本直接更新 RTT。

## 6. 快速重传

发送方连续收到 3 个对同一数据的重复 ACK 时，不等待超时，立即重传推测丢失的报文段。

::: warning 易错点
确认号确认的是“此前所有字节已收到，并期待下一个字节”，不是“当前报文段编号”；TCP 序号按字节而不是按报文段递增。
:::
