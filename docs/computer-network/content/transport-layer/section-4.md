<h1><center>第四节 TCP 连接管理与计时器</center></h1>

## 1. 三次握手

假设客户端初始序号为 $x$，服务器初始序号为 $y$：

1. 客户端发送 `SYN=1, seq=x`。
2. 服务器发送 `SYN=1, ACK=1, seq=y, ack=x+1`。
3. 客户端发送 `ACK=1, seq=x+1, ack=y+1`。

第三次握手可以携带数据。

三次握手可确认双方收发能力并防止过期连接请求突然到达导致错误建立连接。

## 2. 四次挥手

TCP 全双工的两个方向需分别关闭：

1. 主动关闭方发送 FIN。
2. 被动方确认 ACK，连接进入半关闭状态。
3. 被动方数据发送完后发送 FIN。
4. 主动方确认，并进入 TIME-WAIT。

中间 ACK 和 FIN 有时可以合并，因此报文数量不一定机械地总是四个。

## 3. TIME-WAIT

主动关闭方等待 $2\times MSL$，目的包括：

- 保证最后 ACK 丢失时可以重发。
- 让旧连接中滞留报文在网络中消失，避免影响同四元组新连接。

## 4. SYN 攻击

攻击者发送大量 SYN 而不完成握手，使服务器半连接队列耗尽。防御可使用 SYN cookies、限制速率、缩短超时和扩大队列等。

## 5. TCP 计时器

- 重传计时器：超时重传未确认数据。
- 持续计时器：处理零窗口死锁。
- 保活计时器：探测长期空闲连接的对端状态。
- TIME-WAIT 计时器：等待 $2MSL$。

## 6. MSS 与 MTU

MSS 是 TCP 报文段中最大数据载荷，不含 TCP/IP 首部。典型关系：

$$
MSS=MTU-IP\text{ 首部}-TCP\text{ 首部}.
$$

以以太网 MTU 1500、无选项 IPv4/TCP 首部各 20 字节为例，MSS 通常为 1460 字节。

::: warning 易错点
SYN 和 FIN 各消耗一个序号；纯 ACK 且不携带数据时通常不消耗序号。
:::
