<h1><center>第四节 局域网、无线局域网、广域网与 PPP</center></h1>

## 1. MAC 地址

以太网 MAC 地址通常为 48 位，标识网络接口。全 `1` 地址是广播地址：

```text
FF-FF-FF-FF-FF-FF
```

MAC 地址用于同一链路或局域网内逐帧交付，跨路由器转发时每一跳的链路层首部都会重写。

## 2. 以太网帧

以太网帧主要包含：

- 目的 MAC 地址。
- 源 MAC 地址。
- 类型/长度。
- 数据。
- 帧校验序列 FCS。

经典以太网帧从目的地址到 FCS 的长度通常为 64～1518 字节（不含前导码和帧开始定界符）。

## 3. 交换机自学习

交换机收到帧时：

1. 根据源 MAC 地址学习“源地址 → 入端口”。
2. 目的地址已知且端口不同：定向转发。
3. 目的地址未知：向除入端口外的端口泛洪。
4. 目的端口与入端口相同：过滤。
5. 广播帧：向除入端口外的同 VLAN 端口泛洪。

MAC 表项会老化，以适应拓扑变化。

## 4. 冲突域与广播域

- 集线器所有端口属于一个冲突域和一个广播域。
- 交换机每个端口独立冲突域，但默认仍属于同一广播域。
- 路由器每个接口隔离广播域。

## 5. VLAN

虚拟局域网在交换网络中逻辑划分广播域。不同 VLAN 之间通信需要三层设备转发。

IEEE 802.1Q 在以太网帧中加入 VLAN 标记；交换机之间的 trunk 链路可承载多个 VLAN 的帧。

## 6. IEEE 802.11 无线局域网

IEEE 802.11 无线局域网由站点、基本服务集 BSS 和分配系统组成。基础设施模式下，站点先与接入点 AP 关联，再通过 AP 与同一 BSS 或其他网络通信；一个 AP 及其覆盖范围内的站点构成基础设施 BSS。

802.11 使用 CSMA/CA、帧间间隔、随机退避和确认帧降低碰撞影响，必要时用 RTS/CTS 缓解隐蔽站问题。无线帧可能同时携带发送站、接收站、源站和目的站相关地址，以适应经 AP 转发的场景。

## 7. 广域网

广域网覆盖范围大，通常利用运营商提供的点到点链路或交换网络连接不同地区的局域网。局域网侧重共享介质和站点接入，广域网侧重远距离链路上的分组交付。

## 8. PPP 协议

PPP 用于点到点链路，提供成帧、链路控制和网络层协议复用。PPP 帧包含标志字段、地址字段、控制字段、协议字段、信息字段和 FCS。

- 标志字节为 `0x7E`，异步链路使用字节填充实现透明传输。
- LCP 用于建立、配置、测试和终止数据链路。
- NCP 为不同网络层协议配置相关参数。
- PPP 使用 FCS 检错，但不提供序号、确认、重传和流量控制。

::: warning 易错点
交换机能隔离冲突域，但普通二层交换机不能隔离同一 VLAN 内的广播域；VLAN 的主要作用正是逻辑划分广播域。
:::
