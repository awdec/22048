<h1><center>第二节 流量控制与可靠传输</center></h1>

## 1. 可靠传输机制

- 差错检测。
- 接收方确认 ACK 或否认 NAK。
- 发送方超时计时器。
- 出错或超时后的重传。
- 序号，用于区分新帧和重复帧。

## 2. 停止—等待协议

发送方每发送一个数据帧就等待确认，收到 ACK 后才发送下一帧。序号用 1 位即可区分相邻帧。

若数据帧发送时延为 $T_d$，确认帧发送时延忽略，单程传播时延为 $T_p$：

$$
U=\frac{T_d}{T_d+2T_p}.
$$

传播时延较大时利用率很低。

## 3. 滑动窗口

允许发送方在未收到确认前连续发送多个帧，从而实现流水线传输。

若序号字段有 $n$ 位：

- 停止—等待：发送窗口 $W_T=1$，接收窗口 $W_R=1$。
- 后退 N 帧 GBN：$W_T\le2^n-1$，$W_R=1$。
- 选择重传 SR：通常 $W_T=W_R\le2^{n-1}$。

## 4. 后退 N 帧 GBN

接收方只按序接收，丢弃失序帧，并累计确认最后一个连续正确帧。某帧超时后，发送方重传该帧及其后的所有未确认帧。

实现简单，但一个错误可能造成多个正确帧重传。

## 5. 选择重传 SR

接收方可缓存窗口内失序帧并分别确认。某帧出错或超时时只重传该帧，效率高，但缓存和控制复杂。

## 6. 窗口利用率

发送窗口为 $W$，帧发送时延为 $T_d$，往返等待近似为 $2T_p$，若不考虑确认发送时间：

$$
U=\min\left(1,\frac{WT_d}{T_d+2T_p}\right).
$$

::: warning 易错点
GBN 接收窗口为 1，不缓存失序帧；SR 接收窗口大于 1，可缓存并单独确认。序号空间必须足够大，避免新旧帧混淆。
:::
