<h1><center>第五节 NAT、IPv6 与组播</center></h1>

## 1. NAT

网络地址转换在边界路由器修改 IP 地址，允许私有网络共享一个或少量公网地址。

NAPT/PAT 还转换传输层端口，通过“内部地址:端口 ↔ 公网地址:端口”映射区分不同连接。

优点是节约公网 IPv4 地址、隐藏内部结构；缺点是破坏端到端透明性，对某些协议和入站连接不友好。

## 2. IPv6 地址

IPv6 地址为 128 位，使用十六进制冒号表示。连续全零段可压缩为 `::`，但一个地址中只能使用一次。

IPv6 基本首部固定为 40 字节，主要特点：

- 地址空间更大。
- 首部简化，无首部校验和。
- 扩展首部承载可选功能。
- 路由器不对普通 IPv6 分组分片，分片由源主机完成。
- 不使用广播，改用组播和任播。

## 3. IPv4 向 IPv6 过渡

- 双协议栈：设备同时运行 IPv4 和 IPv6。
- 隧道：把 IPv6 分组封装在 IPv4 中穿越 IPv4 网络。
- 翻译：在 IPv4 与 IPv6 报文之间转换。

## 4. IP 组播

组播把同一数据高效发送给一组接收者，网络在适当分支复制分组。IPv4 组播地址范围为 `224.0.0.0/4`。

IGMP 用于 IPv4 主机与本地路由器之间的组成员关系管理，不负责计算跨网络组播路由。

## 5. 移动 IP

移动节点保留归属地址，通过转交地址反映当前位置；归属代理把发往归属地址的分组隧道转发到当前位置。

典型通信过程为：

1. 移动节点进入外地网络后获得转交地址，并向归属代理注册当前位置。
2. 通信对端仍把分组发送到移动节点的归属地址。
3. 归属代理截获分组，通过隧道封装转发到转交地址。
4. 外地网络或移动节点解封装并接收分组。
5. 移动节点返回归属网络后注销转交地址，恢复直接交付。

移动 IP 保持上层使用的归属地址不变；经归属代理绕行可能形成三角路由。

::: warning 易错点
NAT 改写 IP 地址，NAPT 还改写端口；IPv6 路由器不执行中途分片，IPv4 路由器在条件允许时可以分片。
:::
