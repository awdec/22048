<h1><center>第四章总结：网络层</center></h1>

## 地址与转发

- 地址块大小：$2^{32-n}$。
- 传统可用主机数：$2^{32-n}-2$。
- 多条路由匹配时使用最长前缀匹配。
- 跨网发送时，IP 目的地址是最终主机，帧目的 MAC 是下一跳。

## 协议速查

| 协议 | 功能 | 典型封装/算法 |
| --- | --- | --- |
| ARP | IPv4 到 MAC 解析 | 链路层广播请求 |
| DHCP | 动态配置网络参数 | UDP 67/68 |
| ICMP | 差错报告与探测 | 封装在 IP 中 |
| RIP | AS 内距离向量路由 | UDP、跳数 |
| OSPF | AS 内链路状态路由 | 直接封装 IP、Dijkstra |
| BGP | AS 间路径向量路由 | TCP、AS 路径 |

## 控制与转发

- SDN 将控制平面与数据平面分离，由控制器统一生成并下发转发规则。
- 路由表保存控制平面的网络可达信息，转发表供数据平面逐分组查找下一跳。
- 移动 IP 由归属代理把发往归属地址的分组隧道转发到已注册的转交地址。

::: tip 一句话总结
网络层以 IP 地址实现跨网络交付：转发表决定下一跳，ARP 找到下一跳链路地址，路由协议负责生成和维护可达路径。
:::
