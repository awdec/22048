<h1><center>第三节 转发、ARP、DHCP 与 ICMP</center></h1>

## 1. IP 转发

主机发送数据前用掩码判断目的 IP 是否与自己同一网络：

- 同一网络：直接交付，ARP 查询目的主机 MAC。
- 不同网络：间接交付，帧的目的 MAC 是默认网关接口 MAC，IP 目的地址仍是最终主机。

每经过一个路由器，链路层帧都被解封装并重新封装，源/目的 MAC 随下一跳变化；源/目的 IP 通常保持不变（NAT 除外）。

## 2. ARP

ARP 在同一广播域内把 IPv4 地址解析为 MAC 地址：

1. 查询 ARP 缓存。
2. 未命中则广播 ARP 请求。
3. 目标接口单播回复自己的 MAC 地址。
4. 双方可缓存映射。

ARP 广播不能跨越路由器。

## 3. DHCP

DHCP 动态分配 IP 地址、掩码、默认网关和 DNS 服务器等参数。典型过程：

```text
DHCP Discover → DHCP Offer → DHCP Request → DHCP ACK
```

客户端尚无地址时使用 UDP，客户端端口 68，服务器端口 67。跨网段时可由 DHCP 中继代理转发。

## 4. ICMP

ICMP 用于差错报告和网络探测，封装在 IP 数据报中。常见类型：

- 目的不可达。
- 时间超过。
- 参数问题。
- 回显请求与回显应答。

ping 使用 ICMP 回显，traceroute 利用逐渐增大的 TTL 和 ICMP 时间超过报文发现路径上的路由器。

## 5. 不发送 ICMP 差错报文的典型情况

通常不对以下报文再发送 ICMP 差错：

- ICMP 差错报文本身。
- 非首片分片。
- 广播或组播数据报。
- 源地址为特殊地址的数据报。

## 6. 路由器的组成与功能

路由器主要由输入端口、交换结构、输出端口和路由处理器组成：

- 输入端口完成物理层和链路层处理，并查询转发表。
- 交换结构把分组从输入端口送到选定的输出端口。
- 输出端口负责排队、链路层封装和发送。
- 路由处理器运行路由协议、维护路由信息并生成转发规则。

路由表主要供控制平面保存路由协议得到的网络可达信息；转发表供数据平面按目的地址高速查找下一跳和输出接口。路由表生成转发表，二者不能简单视为同一张表。

::: warning 易错点
ARP 解析的是“下一跳 IP 对应的 MAC”，不一定是最终目的主机的 MAC；跨网通信时主机首先解析默认网关。
:::
