<h1><center>第四节 路由算法与路由协议</center></h1>

## 1. 静态与动态路由

- 静态路由由管理员配置，简单可控，适合小型稳定网络。
- 动态路由通过协议交换信息，自动适应拓扑变化。

## 2. 距离向量算法

每个路由器周期性向邻居发送自己的距离向量，根据 Bellman-Ford 思想更新：

$$
D_x(y)=\min_v\{c(x,v)+D_v(y)\}.
$$

缺点包括慢收敛和“好消息传得快，坏消息传得慢”，可用毒性逆转等方法缓解计数到无穷。

## 3. 链路状态算法

每个路由器发现邻居和链路代价，把链路状态分组洪泛到整个区域，建立完整拓扑数据库，再用 Dijkstra 算法计算最短路径树。

## 4. 层次路由

互联网把路由域划分为自治系统 AS。AS 内使用内部网关协议，AS 间使用外部网关协议。层次划分可减小路由信息规模并允许不同管理域采用不同策略；OSPF 还可把一个 AS 划分为多个区域。

## 5. RIP

RIP 是基于距离向量的内部网关协议：

- 度量为跳数。
- 最大有效跳数 15，16 表示不可达。
- 周期性向邻居交换整个路由表。
- 适合小型网络。

## 6. OSPF

OSPF 是基于链路状态的内部网关协议：

- 洪泛链路状态信息。
- 使用 Dijkstra 算法。
- 支持分层区域和等价负载分担。
- 直接封装在 IP 中。

## 7. BGP

BGP 是自治系统之间的外部网关协议，属于路径向量协议。它交换网络可达性和 AS 路径，强调策略与可扩展性，不只追求最短距离。BGP 使用 TCP 建立可靠连接。

## 8. 自治系统

自治系统 AS 是由同一管理机构控制、采用统一路由策略的一组网络和路由器。AS 内使用 IGP（如 RIP、OSPF），AS 间使用 BGP。

::: warning 易错点
OSPF 不是传输层上的 TCP/UDP 应用，它直接封装在 IP 中；BGP 则使用 TCP。RIP 通常使用 UDP。
:::
