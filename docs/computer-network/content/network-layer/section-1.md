<h1><center>第一节 网络层服务与 IPv4 数据报</center></h1>

## 1. 网络层功能

- 路由选择：计算端到端路径。
- 分组转发：根据转发表把分组送到输出接口。
- 异构网络互联。
- 拥塞控制与差错报告。

互联网网络层提供无连接、尽最大努力交付的数据报服务，不保证可靠、按序或无重复。

## 2. SDN 基本概念

软件定义网络 SDN 把控制平面与数据平面分离：控制平面计算路径并生成转发规则，数据平面根据规则高速转发分组。逻辑集中的控制器通过南向接口管理转发设备，并通过北向接口向网络应用提供能力。

传统路由器通常在每台设备内分别运行控制协议；SDN 通过控制器统一掌握网络状态，便于集中配置和策略控制。

## 3. 网络层拥塞控制

当进入网络的流量超过链路和路由器的处理、缓存能力时，会出现排队时延增加、缓冲区溢出和分组丢失。拥塞控制面向整个网络，不同于保护接收端的流量控制。

- 开环控制在拥塞发生前通过接纳控制、流量整形和资源预留等策略预防拥塞。
- 闭环控制根据丢包、时延或显式通知等反馈降低发送负载。

## 4. 数据报与虚电路

| 项目 | 数据报 | 虚电路 |
| --- | --- | --- |
| 建立连接 | 不需要 | 需要 |
| 路由 | 每个分组独立选择 | 建立时确定路径 |
| 首部 | 完整目的地址 | 短虚电路号 |
| 故障适应 | 较灵活 | 需重建虚电路 |

## 5. IPv4 首部

关键字段包括：

- 版本、首部长度和总长度。
- 标识、标志、片偏移。
- 生存时间 TTL。
- 协议字段。
- 首部校验和。
- 源、目的 IPv4 地址。

首部长度字段以 4 字节为单位，总长度以字节为单位。无选项的 IPv4 首部为 20 字节。

## 6. 分片

当数据报长度超过链路 MTU 时，IPv4 路由器可分片（若 DF 位未置 1）。

- 同一原始数据报的各分片标识字段相同。
- MF=1 表示后面还有分片，最后一片 MF=0。
- 片偏移以 8 字节为单位，因此除最后一片外，数据长度通常为 8 的倍数。
- 分片只在最终目的主机重组。

## 7. TTL 与协议字段

每经过一个路由器，TTL 至少减 1；减为 0 时丢弃并通常返回 ICMP 超时报文，防止分组无限循环。

协议字段指明 IP 数据部分交给 TCP、UDP、ICMP 等哪个上层协议。

::: warning 易错点
IPv4 校验和只校验首部，不校验数据；路由器修改 TTL 后必须重新计算首部校验和。
:::
