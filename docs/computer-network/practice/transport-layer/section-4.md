<h1><center>第四节 TCP 连接管理</center></h1>

::: info 例 1
客户端第一次握手序号为 100，服务器第二次握手序号为 500，写出第二、三次握手的确认号。
:::

::: details 解析
SYN 消耗一个序号：

- 第二次握手：`ack=101`。
- 第三次握手：`ack=501`，客户端自身序号从 101 开始。
:::

::: info 例 2
为什么 TCP 关闭通常需要四次交互？
:::

::: details 解析
TCP 是全双工，每个方向的数据流要独立关闭。收到对方 FIN 只能确认该方向关闭，本方可能仍有数据，因此 ACK 和本方 FIN 常分开发送。
:::

::: info 例 3
以太网 MTU 为 1500 字节，IPv4 首部和 TCP 首部均无选项，求 MSS。
:::

::: details 解析
$$
MSS=1500-20-20=1460\text{ B}.
$$
:::

::: info 例 4
主动关闭方为什么等待 $2MSL$？
:::

::: details 解析
保证最后 ACK 丢失时能重发，并让旧连接中滞留的重复报文在网络中消失，避免影响同四元组的新连接。
:::
