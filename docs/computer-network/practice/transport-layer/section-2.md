<h1><center>第二节 TCP 可靠传输</center></h1>

::: info 例 1
TCP 报文段序号为 1001，携带 500 字节数据。接收方正确收到后确认号是多少？
:::

::: details 解析
$$
ACK=1001+500=1501.
$$
表示期待下一个字节序号 1501。
:::

::: info 例 2
发送方收到三个确认号均为 2001 的重复 ACK，通常采取什么动作？
:::

::: details 解析
执行快速重传，不等待超时，立即重传从序号 2001 开始的推测丢失报文段，并按具体 TCP 版本进入快恢复或相应拥塞处理。
:::

::: info 例 3
TCP 为什么需要序号，即使已有校验和？
:::

::: details 解析
校验和只能检测比特错误；序号还用于识别丢失、重复与乱序，并让接收方按序重组字节流。
:::
