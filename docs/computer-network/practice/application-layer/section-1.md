<h1><center>第一节 应用模型与套接字</center></h1>

::: info 例 1
TCP 服务器调用 accept 后，原监听套接字是否被关闭或替换？
:::

::: details 解析
不会。accept 返回一个用于该客户端通信的新连接套接字，原监听套接字继续等待其他连接。
:::

::: info 例 2
P2P 系统是否完全不存在服务器？
:::

::: details 解析
不一定。P2P 的核心是对等方直接提供和请求资源，实际系统仍可能使用中心服务器完成登录、索引或协调，但主要数据交换可在对等方之间进行。
:::

::: info 例 3
实时语音和文件下载对传输服务的主要要求有何差别？
:::

::: details 解析
实时语音更重视低时延和稳定抖动，可容忍少量丢包；文件下载重视完整、按序和可靠交付，通常使用 TCP。
:::
