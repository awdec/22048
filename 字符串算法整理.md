## 基础概念与工具

### Border

Border 指字符串的公共前后缀。根据语境不同，它可以表示字符串本身、长度、最长公共前后缀，且有时允许等于原串，有时不允许，需要结合题意判断。

周期与 Border 等价：若 $p$ 是字符串 $s$ 的周期，则 $|s|-p$ 是 $s$ 的 Border；反过来也成立。因此求最小周期可以转化为求最大 Border。

Border 具有传递性：一个 Border 的 Border 仍然是原串的 Border。基于这个性质，可以用类似 KMP 的方式在线性时间求出所有前缀的最长 Border，即 `border[i]`。时间复杂度 $O(n)$。

### 字符串处理

`docs/string/string.md` 主要记录 C++ `string` 的常用接口：

- `append`：追加整个字符串、子串、若干相同字符或迭代器范围。
- `substr`：截取子串。
- `find/rfind`：查找子串；C++20 起有 `starts_with/ends_with`，C++23 起有 `contains`。
- `replace`：替换子串。
- `stoi/stol/stoll/stof/stod/stold`：字符串转数字；`to_string`：数字转字符串。
- `isupper/islower/isalpha/isdigit/isalnum/isspace/ispunct/isxdigit`：字符分类。
- `toupper/tolower`：字符大小写转换。

这部分不是竞赛意义上的数据结构，但属于字符串题常用基础工具。

## 单串匹配与 LCP

### KMP

KMP 用模式串的 Border 处理失配。当文本串 $a$ 与模式串 $b$ 匹配到 $a_i$ 和 $b_j$ 且失配时，模式串不从头开始，而是跳到 $border_{j-1}+1$ 继续匹配。

它适合解决“在文本串中查找模式串”的单模式匹配问题，时间复杂度 $O(\max(|a|, |b|))$。

### KMP 自动机与 Border 树

对一个字符串 $s$，可以建立 Border 树，也称失配树。它有 $n+1$ 个点 $0,1,\dots,n$，点 $i$ 的父亲为 $border_i$。

常见性质：

- 前缀 `pre_i` 的所有 Border：点 $i$ 到根的链。
- 有长度为 $x$ 的 Border 的前缀：点 $x$ 的子树。
- 两个前缀的公共 Border：对应点的 LCA。

也可以把它看作只有一个模式串的 AC 自动机。

### Z 函数与扩展 KMP

Z 函数定义为 $z(i)=LCP(s, suf_i)$，也就是原串与第 $i$ 个后缀的最长公共前缀。

算法维护右端点最靠右的匹配段 $[l,r]$。若 $i$ 在当前匹配段内，可利用对称位置的信息得到初值，再继续暴力扩展；由于右端点只会向右移动 $O(n)$ 次，总时间复杂度 $O(n)$。

扩展 KMP 用模式串的 Z 信息，求模式串与文本串每个后缀的 LCP。它适合批量比较“模式串前缀”和“文本串后缀”的最长匹配长度。

## 字典匹配结构

### Trie

Trie，又称字典树或前缀树，用于维护字符串集合并支持字符串查询。

每条边表示一个字符，每个节点表示一个前缀。插入字符串时，从根开始按字符走边，不存在则新建节点；查询时沿字符转移即可。空间通常与所有字符串总长度乘字符集维护方式有关。

若字符集只有 `0/1`，则是 01-Trie；在字符串目录中主要用于 01 串字典序或计数问题，而整数异或维护通常归到数据结构目录的 01-Trie。

### AC 自动机

AC 自动机是 KMP 与 Trie 的结合，用于“一个文本串匹配多个模式串”的字典匹配问题。

Trie 上一个节点表示某个模式串前缀，AC 自动机为每个节点维护 `fail/border` 指针：失配后应跳到的最长可匹配后缀节点。由于求一个节点的 fail 依赖其父亲的 fail，通常用 BFS 构建。

复杂度：

- 建立 Trie 与 fail：$O(\sum |T|)$，其中 $T$ 为所有模式串。
- 匹配文本串：朴素跳 fail 的分析同 KMP，文本匹配 $O(|S|)$。

对于文本串匹配到 Trie 节点 $x$ 时，所有匹配成功的模式串对应 $x$ 到 fail 树根路径上的标记节点。

### Trie 图优化

Trie 图优化会补全每个节点在每个字符上的转移 `trans[x][c]`：

- 若原 Trie 中有 `c` 后继，则直接转移到儿子。
- 否则转移到 `trans[fail[x]][c]`。

补全后，匹配时每读入一个字符只需一次转移，常用于自动机 DP。代价是需要为所有节点和所有字符预处理转移，时空复杂度 $O(\sum |T| \cdot |c|)$。

## 子串结构

### 后缀数组 SA

后缀数组把所有后缀按字典序排序，用数组表示排序结果：

- `rk[i]`：后缀 `suf[i]` 的排名。
- `sa[i]`：排名第 $i$ 的后缀起点。
- 二者满足 `rk[sa[i]] = i`。

SA 将任意子串表示为某个后缀的前缀，因此适合处理子串排序、LCP、不同子串统计等问题。后缀排序最优可 $O(n)$，竞赛中常用倍增 $O(n\log n)$；空间复杂度 $O(n)$。

### height 数组

`height` 数组记录相邻排名后缀的 LCP：

$$h[i] = LCP(suf[sa[i-1]], suf[sa[i]])$$

任意两个后缀 `suf[sa[i]]` 和 `suf[sa[j]]` 的 LCP 等于区间 $h[i+1..j]$ 的最小值。因此 SA + height + RMQ 可以快速求任意后缀 LCP。height 数组可 $O(n)$ 求出。

### 后缀自动机 SAM

SAM，即后缀自动机，是识别一个字符串所有子串的最小确定有限状态自动机。它适合处理子串存在性、不同子串数量、子串出现次数、最长公共子串等问题。

SAM 中有两类关系：

- 转移边：形成一个 DAG，从起点出发的任意路径表示原串的某个子串。
- fail/slink 边：每个节点指向其不能表示的最长后缀所在状态，所有 fail 边形成 fail 树。

SAM 的每个节点表示一个 `endpos` 等价类。一个节点中所有子串是该等价类最长串的一系列连续后缀；构建时维护 `maxlen`，而该状态可表示的最短长度由 fail 父亲确定。

增量构造单串 SAM 的时间复杂度 $O(n)$。若用数组维护字符转移，空间与状态数和字符集有关；状态数为 $O(n)$。

### fail 树与广义 SAM

SAM 的 fail 树上，一个节点的 `endpos` 集合可以由其子树中节点的 `endpos` 合并得到，实际统计时要注意克隆节点与原节点的信息归属。

广义 SAM 是对多个字符串建立的 SAM，常用于多串子串问题。

## 回文结构

### Manacher

Manacher 用“回文中心 + 回文半径”描述所有回文子串。通常在相邻字符间插入特殊字符，把奇偶回文统一成奇回文处理。

算法维护右端点最靠右的回文区间 $[l,r]$。若当前位置在区间内，可由关于中心的对称位置给出初值，再继续暴力扩展。右端点总共只右移 $O(n)$ 次，因此时间复杂度 $O(n)$。

它适合快速求每个位置为中心的最长回文半径，以及进一步统计回文数量、判断某个区间是否回文等。

### 回文自动机 PAM

PAM，又称回文树或回文自动机，用节点表示本质不同的回文子串。与 Manacher 偏静态求半径不同，PAM 用“最长回文后缀”在线增量维护回文子串。

PAM 通常有两个根，用来区分奇数长度和偶数长度回文。插入新字符时，从上一次的最长回文后缀开始沿 Border/fail 跳，找到能被当前字符扩展的最长回文后缀，再创建或转移到对应节点。

复杂度：

- 朴素跳 Border 构建：$O(|S|)$。
- 若补全 Trie 图，时间复杂度 $O(|S|\cdot |c|)$。
- PAM 节点数为 $O(n)$，因此本质不同回文子串数量也是 $O(n)$。

如果题目不强制在线、只关心本质不同回文子串且与 endpos 无关，也可用 Manacher 半径枚举 + Hash 判重替代；使用 `map` 为 $O(n\log n)$，哈希表期望 $O(n)$，但常数不小。

### Palindrome Series

Palindrome Series 用于优化枚举回文后缀转移的 DP。朴素枚举每个位置结尾的所有回文后缀可能为 $O(n^2)$；利用 PAM 上 fail 链中相同 `diff = len[x] - len[fail[x]]` 的连续段，可把部分转移整体维护。

原文描述的目标是把“枚举回文后缀转移”的 DP 优化到 $O(n\log n)$ 级别，并在每个连续段内用辅助值 `g` 做 $O(1)$ 转移。

## 快速判等

### 字符串 Hash

字符串 Hash 用于快速判断两个字符串是否相同，常用多项式哈希：

$$H(s)=\sum s_i \times base^i$$

常见形式：

- 单哈：对一个模数取模。实现简单，但在随机不同数据约 $\sqrt p$ 次比较后冲突概率会明显上升。
- 自然溢出：使用 `unsigned long long`，相当于模 $2^{64}$，速度快且无需 `int128`，但模数特殊，容易被构造冲突。
- 双哈：使用两个模数，把哈希值视为二元组。根据 CRT 角度，冲突概率近似对应更大的组合模数，竞赛中较稳。

为了降低被卡风险，可以引入随机化，例如随机字符权值、随机底数或随机模数。实际存储时，可把两个 `int` 哈希压成一个 `uint64_t`，比 `pair<int,int>` 在 `map` 等结构里常数更小。

## 选型速查

| 需求 | 优先考虑 |
| --- | --- |
| 单模式串匹配 | KMP |
| 求原串与每个后缀的 LCP | Z 函数 |

| 维护字符串集合、查前缀 | Trie |
| 多模式串匹配文本串 | AC 自动机 |
| 自动机 DP，需要每字符快速转移 | Trie 图优化 |
| 后缀排序、任意后缀 LCP | SA + height + RMQ |
| 动态识别所有子串、统计子串信息 | SAM |
| 多串子串问题 | 广义 SAM |
| 每个中心最长回文半径 | Manacher |
| 在线维护本质不同回文子串 | PAM |
| 回文后缀 DP 优化 | PAM + Palindrome Series |
| 快速判断两段字符串是否相等 | 字符串 Hash |
| 周期、循环节、前后缀关系 | Border / KMP fail |
